기본 URL

Audience API의 기본 URL은 다음과 같습니다.

https://audience.api.brightcove.com/v1

계정 경로

모든 경우에 특정 Video Cloud 계정에 대한 요청이 이루어집니다. 항상 기본 URL에 "계정"이라는 용어와 계정 ID를 추가해야 합니다.

https://audience.api.brightcove.com/v1/accounts/{account_id}

입증

Audience API는 Brightcove를 사용합니다. OAuth 서비스통화를 인증합니다.

먼저 클라이언트 자격 증명(a client_id그리고client_secret ). 이것은 다음을 사용하여 수행할 수 있는 일회성 작업입니다. OAuth 자격 증명 UI . 청중/읽기 작업에 대한 권한이 필요합니다.

다음을 사용하여 Brightcove OAuth 서비스에서 직접 클라이언트 자격 증명을 얻을 수 있습니다. 곱슬 곱슬하다또는우편 집배원 .

Authorization: Bearer {access_token}

오류 처리

오류가 발생하면 API는 다음 상태 코드 중 하나로 응답하고 응답 본문에 해당 오류 코드를 표시합니다.

상태 코드 에러 코드 설명
400 BAD_REQUEST_ERROR 쿼리 매개변수가 잘못되었습니다.
401 UNAUTHORIZED_ERROR 액세스 토큰이 없거나 만료되었거나 유효하지 않습니다.
404 RESOURCE_NOT_FOUND URL이 존재하지 않습니다
429 REQUEST_THROTTLED_ERROR 사용자가 속도 제한 정책을 초과했습니다.
500 INTERNAL_ERROR 내부 오류가 발생했습니다
504 GATEWAY_TIMEOUT_ERROR 요청을 수행하는 동안 서버 시간이 초과되었습니다.

다음은 오류에 대한 샘플 응답 본문입니다.

[
	{
	"error_code": "UNAUTHORIZED_ERROR",
	"message": "Permission denied"
	}
]

매개변수

검색된 데이터를 제한하고 필터링하기 위해 요청에 추가할 수 있는 몇 가지 매개변수가 있습니다. 이는 다음 섹션에서 설명하는 모든 요청 유형에 적용됩니다.

필터링 결과

다음을 사용하여 결과를 필터링할 수 있습니다. where매개변수. 필터 구문은 다음과 같습니다.

where=field1==value1;field2==value2

예를 들면 다음과 같습니다.

where=video_id==123456789;video_name==test

쉼표는 논리적 OR로 처리되고 세미콜론은 논리적 AND로 처리됩니다. 예를 들어, where=video_id==1234,5678;video_name=test "여기서 video_id = 1234 OR 5678, AND video_name = test"로 해석됩니다.

반환할 필드 선택

결과를 해당 필드 하위 집합으로 제한하기 위해 요청에 필드 목록을 지정할 수 있습니다. 필드 구문은 다음과 같습니다.

fields=field1,field4

예를 들면 다음과 같습니다.

fields=video_id,video_name

필터링 및 정렬할 수 있는 필드는 다음 섹션에서 각 요청 유형에 대해 자세히 설명합니다.

기간

날짜 범위는 다음에서 지정할 수 있습니다. from그리고to매개변수는 뷰 이벤트가 마지막으로 업데이트된 날짜(Updated_at 필드)에 적용됩니다. 날짜 범위는 다음 형식으로 표시할 수 있습니다.

  • 텍스트 값now현재 시간을 나타내는
  • 밀리초 단위의 Epoch 시간 값(예: 1377047323000
  • ISO 8601 표준 국제 날짜 형식으로 표시된 날짜: YYYY-MM-DD와 같은 형식2013-09-12 . 이 형식으로 표시된 날짜의 경우:
    • 지정된 모든 날짜 범위는 UTC로 해석됩니다.
    • 날짜 제공 시간은 자정으로 해석됩니다( 00:00:00 ) 지정된 날짜에
  • 상대 날짜: 다음 중 하나를 표현할 수 있습니다. to그리고from다른 값에 상대적인 값d (날), h (시간), m (분) 또는s (비서). 예를 들면 다음과 같습니다.
    • from=2015-01-01&to=31d
    • from=-48h&to=now
    • from=-2d&to=now (이전 예제와 동일한 결과를 제공합니다)
    • from=-365d&to=2015-12-31
    • from=-10m&to=now

페이징 결과

NS limit반환할 항목의 수입니다(기본값: 25, 최대값: 200). offset건너뛸 항목의 수입니다(기본값: 0). 당신이 사용할 수있는limit그리고offset결과를 살펴보는 앱을 함께 만들 수 있습니다. 각각 포함limit , offset , 그리고count. , 총 결과에 대한 반복을 설정하는 데 사용할 수 있습니다. 예를 들어 JavaScript에서 다음과 같이 필요한 총 반복을 얻을 수 있습니다.

// response is the JSON-parsed response from the first request
var totalRequests = Math.ceil(response.count / response.limit)
