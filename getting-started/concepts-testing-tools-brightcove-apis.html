쉘 스크립트(cURL)

명령줄 도구는 다음과 같은 쉘 스크립트입니다. API 테스터.sh액세스 토큰을 얻고 API를 호출합니다. api-tester 스크립트는 cURL을 사용하여 http 호출을 수행합니다. cURL은 Mac MacOS 및 모든 Unix/Linux 시스템에 내장되어 있습니다. Windows에서 cURL을 얻으려면 다음을 참조하십시오. cURL 설정 . 그런 다음 다운로드 api-tester.sh 스크립트

쉘 스크립트의 전체 코드는 다음에서 볼 수 있습니다. 액세스 토큰 받기 .

쉘 스크립트 실행

셸 스크립트를 다운로드한 후 원하는 위치에 압축을 풀고 동일한 폴더의 명령줄로 이동합니다. 스크립트를 실행하려면 다음을 입력하십시오.

	bash api-tester.sh

스크립트를 실행할 때마다 5개의 변수를 입력하라는 메시지가 표시됩니다.

  • 클라이언트 ID( 아래 섹션클라이언트 자격 증명을 얻을 때)
  • 클라이언트 암호( 아래 섹션클라이언트 자격 증명을 얻을 때)
  • 전체 API 호출(예: https://cms.api.brightcove.com/v1/accounts/12345678/videos
  • 요청 동사 - 다음 중 하나: GET | 포스트 | 패치 | 넣어 | DELETE(기본값은 GET임)
  • 요청 데이터 - POST, PATCH 및 PUT에 사용됩니다. GET 및 DELETE 요청의 경우 이 항목을 공백으로 두고 다음을 누르기만 하면 됩니다. 반품

명령줄 도구는 다음과 같은 응답을 반환합니다. 온라인 도구 - 그러나 어떤 이유로 요청이 실패할 경우 헤더와 추가 정보를 볼 수 있는 전체 응답도 표시합니다. 또한 응답에 대한 원시 JSON과 예쁜 인쇄 버전을 에코합니다.

온라인 도구

온라인 테스트 도구를 찾을 수 있습니다여기 . 명령줄 도구와 마찬가지로 다음을 요구합니다.

  • 클라이언트 ID
  • 클라이언트 비밀
  • 전체 API 요청 URL
  • 요청 유형
  • 제출할 JSON 데이터(있는 경우)

자격 증명 얻기

Brightcove API를 사용하려면 다음으로 구성된 적절한 클라이언트 자격 증명이 필요합니다. client_id그리고client_secret . 참조 OAuth 개요클라이언트 자격 증명 흐름을 더 잘 이해할 수 있습니다.
