소개

Brightcove 플랫폼 API는 데이터를 가져오고 Brightcove 플랫폼의 다양한 부분으로 보낼 수 있는 RESTful API입니다. 사용 예는 다음과 같습니다.

  • 비디오 클라우드로 비디오 수집
  • Video Cloud에서 비디오 관리
  • Video Cloud 계정에 대한 맞춤형 분석 보고서 생성
  • Brightcove 플레이어 생성 및 관리
  • 라이브 스트리밍을 위한 Brightcove 라이브 작업 생성
  • Zencoder에서 비디오용 트랜스코딩 작업 만들기

RESTful API

RESTful API에 익숙하지 않은 경우 다음과 같은 몇 가지 공통적인 특성이 있습니다.

  • 요청은 다음과 같은 다양한 방법을 사용하여 HTTP 요청을 전송하여 이루어집니다. GET , POST , PATCH , 그리고DELETE , 수행된 작업의 종류에 따라 다릅니다.
  • 응답은 일반적으로 다음과 같은 형식으로 이루어집니다. JSON모든 프로그래밍 언어가 기본 데이터 유형으로 구문 분석할 수 있는 문자열입니다.

REST의 장점은 RESTful API가 특정 프로그래밍 언어에 의존하지 않는다는 것입니다. HTTP 요청을 만들고 JSON 응답을 구문 분석할 수 있는 한 API는 사용 중인 언어를 신경 쓰지 않습니다.

CORS

한 가지 알아야 할 점은 대부분의 Brightcove API는 CORS활성화. 링크된 문서에서 전체 설명을 읽을 수 있지만 결론은 거의 모든 경우에 API 요청이 웹 클라이언트(예: 웹 앱)에서 이루어질 수 없다는 것입니다.

입증

대부분의 플랫폼 API에 대한 인증은 임시 OAuth2 액세스 토큰을 통해 이루어집니다. 일부 API는 대신 영구 API 키 또는 정책 키를 사용합니다. 자세한 내용은 개별 API에 대한 설명서를 참조하십시오.
