소개

Dynamic Delivery의 작동 방식과 사용 이점을 이해하려면 다음을 참조하십시오. Dynamic Delivery 개요 . 이 항목에서는 특히 Dynamic Ingest 사용과 관련하여 Dynamic Delivery로 전환한 후 수행해야 하는 몇 가지 작업을 다룹니다.

비디오 재변환

기존 비디오는 자동으로 변환되지 않습니다. 이전 동영상을 이전하는 방법에 대한 자세한 내용은 계정 관리자에게 문의하세요.

기존 비디오를 변환했거나 직접 재코드 변환하여 Dynamic Delivery로 이동하는 경우 비디오에 대한 다음 메타데이터가 변경됩니다.

  • NS update_at데이트
  • 썸네일 및 포스터 이미지 경로(권장 ~ 아니다일반적으로 페이지 및 앱의 비디오 자산 URL 하드코딩)

DRM

DRM으로 비디오를 보호하는 경우 Dynamic Delivery의 장점 중 하나는 ~ 아니다 DRM을 적용하려면 사용자 정의 프로필이 필요합니다. 대신 지원팀에 연락하여 동영상에 적용할 DRM 유형에 맞게 계정을 구성하기만 하면 됩니다. 그 후에는 Dynamic Delivery 프로필을 사용하여 수집되거나 다시 코드화된 모든 비디오가 자동으로 보호됩니다. 보다 DRM으로 콘텐츠 수집자세한 사항은.

미디어 공유

Dynamic Delivery의 미디어 공유에 대해 청구가 다르게 작동한다는 점에 유의하십시오. 공유 미디어와 관련된 비용은 미디어가 속한 마스터 계정으로 청구됩니다. 에서 공유, 미디어가 공유되는 계정이 아닙니다.

알림

Dynamic Delivery 수집에 대한 알림은 레거시 수집에 대한 알림과 유사하지만 몇 가지 차이점이 있습니다. 후속 작업을 트리거하기 위해 Dynamic Ingest 알림에 의존하는 앱 또는 통합이 있는 경우 약간 수정해야 할 수 있습니다.

다음은 차이점을 강조하기 위해 Dynamic Delivery가 아닌 Dynamic Delivery 수집에 대한 성공적인 변환 완료에 대한 샘플 알림입니다.

선수

좋은 소식입니다. 표준 플레이어 내장 코드 또는 기본 플레이어 SDK를 사용하는 경우 Dynamic Delivery를 위해 기존 플레이어를 변경할 필요가 없습니다. 동영상은 이전과 같이 계속 작동합니다.
