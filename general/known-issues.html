Analytics API

비디오 또는 플레이어 필드 요청fields치수로 요청하지 않고

사용하는 경우video 또는player 필드(예: video , video_name , player 또는player_name ) 반환할 필드에 있지만 차원에는 포함하지 않습니다. dimensions , 응답 항목에는 해당 항목과 관련이 없는 값이 포함되며 무시해야 합니다. 항목의 다른 값이 유효합니다.

해결 방법 : 비디오 또는 플레이어 필드도 응답 필드로 포함하지 않는 한 응답 필드에 포함하지 마십시오. dimensions .

에 대한 "기타" 결과의 높은 수device_manufacturer
경우에 따라 "기타" 결과가 많이 나타날 수 있습니다. device_manufacturer들. 데이터 수집기가 인식하지 못할 때 발생합니다. user_agent분석 데이터와 함께 제공됩니다. 이에 대해 식별한 가장 일반적인 원인은 Instagram에서 만든 사용자 지정 user_agent 문자열이지만 다른 것도 있을 수 있습니다.

CMS API

특정 경우에는original_filename 손상되다
어떤 경우에는original_filename 비디오 필드가 손상되고 실제 파일 이름이 포함되지 않습니다.
태그는 쉼표를 포함할 수 없습니다.
쉼표(예: "SomeCo, Inc") "불법 태그" 오류와 함께 요청이 실패합니다.
예약된 비디오
Playback API 및 카탈로그는 최대 10-15분 동안 비디오를 캐시하기 때문에 다음 몇 분(최대 20분) 동안 사용할 수 있도록 예약된 비디오를 요청하는 플레이어는 캐시를 새로 고칠 때까지 재생 가능한 비디오를 얻지 못할 수 있습니다.
has_digital_master

이 필드를 사용하여 비디오를 재변환할 수 있는지 여부를 테스트하는 경우 원본 비디오에 마스터가 있는 경우에도 마찬가지이므로 공유 비디오에 대해서는 신뢰할 수 없지만 여전히 제휴 계정으로 재변환할 수는 없습니다.

해결 방법 : 동영상이 계정에 공유된 경우 두 가지 모두에 대해 테스트해야 합니다. has_digital_master = true 그리고sharing.by_external_account = false . 샘플 코드(JavaScript):

var video={some_video_object}
if (video.sharing !== null) {
  if (video.sharing.by_external_account === false) {
    if (video.has_digital_master) {
    // video can be retranscoded
    }
  }
}
업데이트/패치 사용 시 재생 목록에 중복 참조 ID 허용
재생 목록을 업데이트할 때reference_id를 사용하여 CMS API (또는 동일한 API 요청을 사용하기 때문에 미디어 모듈) 다른 재생 목록에 이미 사용 중인 참조 ID가 허용됩니다.
