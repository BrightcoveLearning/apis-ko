사용자 기능

다음은 관련 기능입니다. 비디오 클라우드 CMS 사용자에게 다음을 제공할 수 있습니다.

  • 에 새 비디오 추가비디오 클라우드
  • 교체비디오 클라우드새 버전의 비디오
  • 제목, 설명 및 태그와 같은 동영상의 메타데이터 업데이트
  • 동영상 삭제
  • 재생 목록 만들기
  • 재생 목록의 비디오 변경
  • 재생목록 삭제
  • 비디오 플레이어 만들기
  • 크기 또는 스타일과 같은 동영상 플레이어 속성 수정
  • 플러그인을 통해 비디오 플레이어에 특수 기능 추가
  • 단일 비디오 또는 재생 목록 게시
  • 비디오 로드, 조회수, 재생률, 참여 등에 대한 분석 데이터를 제공합니다.

이 모든 기능을 최종 사용자에게 노출하고 싶지 않을 수 있습니다. 예를 들어 최종 사용자가 비디오를 삭제하도록 허용하지 않을 수 있습니다. 통합의 장점 중 하나비디오 클라우드사용자가비디오 클라우드 Studio는 Brightcove API를 통해 사용자에게 노출할 기능을 정확히 선택할 수 있다는 것입니다.

입증

모든 Brightcove API 요청에 대해 인증은 OAuth2 액세스 토큰을 기반으로 합니다. 액세스 토큰을 얻는 2단계 프로세스가 있습니다.

  1. 필요한 API 작업에 대한 권한이 있는 클라이언트 자격 증명 생성
  2. 클라이언트 자격 증명을 사용하여 API 요청을 인증하기 위한 임시 액세스 토큰 생성

클라이언트 자격 증명 생성

클라이언트 자격 증명 생성은 다음을 통해 수행할 수 있는 일회성 작업입니다. 비디오 클라우드사진관아니면 그 OAuth API . 어떻게 하든, client_id그리고client_secret액세스 토큰을 요청하려면 저장해야 합니다.

NS client_secret클라이언트 자격 증명이 생성된 경우에만 표시되며 다시 검색할 수 없습니다. 클라이언트 암호를 분실한 경우 새 클라이언트 자격 증명을 생성해야 합니다.

액세스 토큰 만들기

임시 액세스 토큰은 다음을 사용하여 생성됩니다. OAuth API . NS client_id그리고client_secret BASE64로 인코딩되어야 하며Basic권한 부여 문자열.

NS access_token반환된 것은 API 호출과 함께 Authorization 헤더에 차례로 전달됩니다.

    Authorization: Bearer your_access_token

액세스 토큰은 5분 동안 유효합니다. 수백 개의 연속적인 API 호출을 수행하는 일종의 배치 작업을 수행하지 않는 한 시간 초과를 추적하려고 하는 것보다 각 API 호출에 대해 새로운 것을 요청하는 것이 합리적입니다.

비디오 추가

사용자가 비디오를 추가할 수 있도록 하려면비디오 클라우드 CMS에서 다음을 사용하여 수행할 수 있습니다. Dynamic Ingest API . 사용자가 S3 버킷 또는 공개 서버일 수 있는 리포지토리에 비디오를 업로드하도록 하는 것이 좋습니다. Dynamic Ingest 시스템은 비디오를 가져와서 비디오에 추가할 수 있습니다. 비디오 클라우드아래에 설명된 2단계 프로세스를 통해 시스템.

비디오 개체 추가비디오 클라우드

첫 번째 단계는 비디오 개체를 만드는 것입니다. 비디오 클라우드를 만들어 시스템우편에 요청 CMS API :

    https://cms.api.brightcove.com/v1/accounts/:account_id/videos

요청 본문에는 기본 비디오 속성이 포함됩니다. JSON개체 - 최소한 비디오name하지만 다음과 같은 추가 메타데이터를 포함할 수도 있습니다. description그리고tags :

    {
    "name": "Woodpecker",
    "description": "A bird that hunts insects inside wood",
    "reference_id": "Bird_Woodpecker.mp4",
    "tags": ["bird", "air", "nature"]
    }
NS reference_id모든 비디오에 추가할 수 있습니다. 참조 ID는 계정 내에서 고유해야 합니다. 동영상에 CMS 시스템에 ID가 있는 경우 해당 ID를 참조 ID로 사용하는 것이 좋습니다. 비디오 클라우드체계.

비디오 관리

NS CMS API계정에 대한 비디오 데이터를 다시 가져올 수 있습니다. (위에서 볼 수 있듯이 비디오 수집 프로세스의 일부로 비디오 개체를 만드는 데에도 사용됩니다.) 가장 기본적인 요청은 다음과 같습니다.

    https://cms.api.brightcove.com/v1/accounts/account_id/videos

기본적으로 이 요청은 JSON이름, 설명, 태그, 사용자 정의 필드, 생성 및 마지막 수정 날짜, 포스터 및 썸네일의 URL 등을 포함한 풍부한 메타데이터를 포함하는 20개의 비디오 개체 배열.

다음 매개변수 중 하나 이상을 요청에 추가하여 요청 결과를 구체화할 수 있습니다.

limit
이것은 반환할 비디오 개체의 수를 결정하며 최대 100까지 설정할 수 있습니다. 기본값은 20입니다.
offset
이것은 건너뛸 항목의 수를 결정하므로 다음과 함께 사용됩니다. limit비디오 카탈로그를 통해 페이지로 이동 - 기본값은 0입니다.
sort
결과를 정렬할 비디오 메타데이터 필드를 결정합니다. 기본적으로 결과는 다음 기준으로 정렬됩니다. updated_at (내림차순, 가장 최근에 업데이트된 비디오를 먼저 표시)

동영상 검색

또한 다음을 사용하여 다양한 기준으로 비디오를 검색할 수 있습니다. q매개변수. 이름, 설명, 태그, 날짜 및 동영상 상태와 같은 특정 필드로 검색할 수 있습니다.

    https://cms.api.brightcove.com/v1/accounts/account_id/videos?q=tags:sea,mammal

자세한 내용 및 검색에 대한 모든 옵션은 다음을 참조하십시오. 동영상 검색 .

특정 비디오 가져오기 및 업데이트

ID 또는 참조 ID로 특정 동영상을 검색하려면:

    https://cms.api.brightcove.com/v1/accounts/account_id/videos/id
    or
    https://cms.api.brightcove.com/v1/accounts/account_id/videos/ref:reference_id

NS 가져 오기요청은 비디오 객체를 반환합니다. 업데이트하려면 수정하십시오. JSON사용하여 반환반점동일한 URL로 요청합니다.

사용자에게 비디오를 표시할 때 게시하거나 업데이트할 잘못된 비디오를 선택할 가능성을 줄이기 위해 디스플레이에 축소판 이미지를 포함하는 것이 좋습니다.

재생 목록

재생 목록 정보도 다음을 사용하여 관리됩니다. CMS API비디오 정보와 거의 같은 방식으로. 참고비디오 클라우드두 가지 범주에서 8가지 유형의 재생 목록을 지원합니다.

수동(또는EXPLICIT ) 재생 목록
지정된 비디오 세트 포함 - 최대 100개의 비디오가 포함될 수 있습니다.
스마트 재생 목록
검색 기준에 따라 런타임에 동적으로 구축됨 - 목록 내에서 동영상이 정렬되는 방식에 해당하는 7가지 종류의 스마트 재생 목록이 있습니다.
  • ACTIVATEDOLDESTTONEWEST
  • ACTIVATEDNEWESTTOOLDEST
  • ALPHABETICAL
  • PLAYSTOTAL
  • PLAYSTRAILINGWEEK
  • STARTDATEOLDESTTONEWEST
  • STARTDATENEWESTTO_OLDEST
할 수 있습니다. ~ 아니다재생 목록이 생성된 후 수동 재생 목록을 스마트 재생 목록으로(또는 그 반대로) 변경합니다.

동영상 수 제한은 최대 100개까지 설정할 수 있습니다.

비디오와 마찬가지로 다음을 사용하여 모든 재생 목록을 검색할 수 있습니다. limit그리고offset계정에 많은 수의 재생 목록이 있는 경우 결과 페이지로 이동:

    https://cms.api.brightcove.com/v1/accounts/account_id/playlists

반환된 재생 목록 개체 배열에는 다음을 포함하여 재생 목록에 대한 메타데이터가 포함됩니다. type위에서 설명한 유형 중 하나에 해당합니다. 유형이EXPLICIT , 또한 있을 것입니다video_ids포함된 비디오의 ID를 포함하는 배열입니다. 유형이 스마트 재생 목록 유형 중 하나인 경우search다음과 같이 동영상을 검색하는 검색 문자열을 포함하는 속성:

    q=tags:fish,birds

다음으로 단일 재생 목록을 검색할 수도 있습니다. id :

    https://cms.api.brightcove.com/v1/accounts/account_id/playlists/playlist_id

재생 목록의 전체 비디오 개체를 검색해야 하는 경우(페이지에 비디오에 대한 정보를 표시하기 위해) 다음을 추가하기만 하면 됩니다. /videos해당 URL:

    https://cms.api.brightcove.com/v1/accounts/account_id/playlists/playlist_id/videos

스마트 재생 목록의 경우 요청은 현재 검색 기준과 일치하지만 변경될 수 있는 비디오를 반환합니다.

플레이어 생성

Brightcove 플레이어는 다음을 통해 만들 수 있습니다. Player Management API . API를 사용하면 플레이어를 만들고, 속성을 업데이트하고, URL 형식으로 포함 코드를 가져올 수 있습니다. iframe태그 또는 페이지에 포함할 HTML 블록입니다.

계정당 최대 200명의 플레이어가 있을 수 있지만 일반적으로 절대적으로 필요한 만큼 적은 수의 플레이어를 보유하는 것이 사용자에게 덜 혼란스럽습니다. 단일 비디오 또는 재생 목록을 재생하려면 별도의 플레이어가 있어야 하지만 그렇지 않으면 스타일이 다르거나 플러그인을 통해 다른 기능이 추가되는 경우에만 다른 플레이어가 필요합니다.

플레이어를 만들려면 간단히우편에 요청Player Management API :

    https://players.api.brightcove.com/v2/accounts/account_id/players

요청 본문에 다음을 포함하십시오. 플레이어 구성 - 필요한 유일한 것은name :

    {
    "name": "Single video player for blog posts"
    }

응답은 플레이어 ID와 여러 형식의 포함 코드를 제공합니다.

    {
    "embed_code": "<iframe src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>",
    "embed_in_page": "http://players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/in_page.embed",
    "id": "de055fa4-4f09-45af-8531-419c6794ad04",
    "preview_embed_code": "<iframe src='//preview-players.brightcove.net/v1/accounts/57838016001/players/de055fa4-4f09-45af-8531-419c6794ad04/preview/embeds/default/master/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>",
    "preview_url": "http://preview-players.brightcove.net/v1/accounts/57838016001/players/de055fa4-4f09-45af-8531-419c6794ad04/preview/embeds/default/master/index.html",
    "url": "http://players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html"
    }

전체 플레이어 구성을 얻으려면/players끝점이지만 위의 응답에서 반환된 플레이어 ID를 추가합니다.

    https://players.api.brightcove.com/v2/accounts/account_id/players/de055fa4-4f09-45af-8531-419c6794ad04

당신은 만들 수 있습니다반점동일한 엔드포인트에 요청하여 플레이어 구성을 업데이트합니다.

위의 응답에서 알 수 있습니다. preview_embed_code그리고preview_url . 새로운 플레이어나 플레이어 업데이트를 테스트할 수 있도록 새로 생성되거나 업데이트된 플레이어는 미리 보기 모드로 설정되어 기존 플레이어에게 변경 사항을 푸시하기 전에 볼 수 있습니다. 변경 사항을 프로덕션으로 푸시하려면 다음을 수행해야 합니다. 출판하다이 요청이 있는 플레이어:

    https://players.api.brightcove.com/v2/accounts/account_id/players/de055fa4-4f09-45af-8531-419c6794ad04/publish

플레이어 커스터마이징

NS 브라이트코브 플레이어표준 웹 기술로 구축: HTML, CSS 및자바스크립트 . 동일한 기술을 사용하여 플레이어를 사용자 정의할 수 있습니다. 이것은 플레이어가 게시된 페이지에서 수행할 수 있지만 가장 좋은 방법은 플레이어를 통해 플레이어 자체에 사용자 정의를 추가하는 것입니다. 구성 , 다음을 통해 플레이어 업데이트반점에 요청Player Management API이전 섹션에서 설명한 대로.

다음을 통해 플레이어에 추가 기능을 추가할 수도 있습니다. 자바스크립트플러그인 , 그리고 광범위한플레이어 API코드를 플레이어와 통합하는 데 도움이 됩니다. Brightcove는 광고 활성화, 최종 화면 사용자 지정 및 오버레이 추가와 같은 작업을 위해 미리 만들어진 여러 플러그인을 제공합니다.

동영상 게시

에서플레이어 만들기 섹션위에서 우리는 다음을 사용하여 플레이어 구성 개체를 가져올 때Player Management API , 반환된 데이터에는 HTML 페이지에 플레이어를 포함하기 위한 iframe 태그와 플레이어를 페이지에 직접 포함하려는 경우 전체 HTML에 대한 URL이 포함됩니다.

플레이어의 CSS와자바스크립트페이지에서 사용되는 기타 스타일시트 및 스크립트. 보다이 문서차이점에 대한 자세한 내용과 다양한 상황에서 사용할 임베딩에 대한 보다 구체적인 권장 사항은

어떤 임베드를 선택하든 다음을 추가해야 합니다. 비디오 클라우드플레이어에 콘텐츠를 추가하려면 비디오 ID 또는 재생 목록 ID를 소스 코드에 추가하십시오. iframe 임베드 코드는 다음과 같습니다.

    <iframe
    src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html'
    allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>

플레이어의 URL에 매개변수를 추가해야 합니다. videoId={}video_id , 전체 내장 코드가 다음과 같이 표시되도록 합니다.

    <iframe
    src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html?videoId=4483119716001'
    allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>

재생 목록 플레이어인 경우 매개변수를 사용합니다. playlistId={playlist_id}대신에. 인페이지 임베드 코드의 수정도 유사합니다.

플레이어 구성에서 플레이어 크기가 고정되어 있지 않은 경우에는 너비와 높이를 추가하여 플레이어 크기를 조정해야 합니다. style기인하다:

    <iframe
    src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html?videoId=4483119716001'
    allowfullscreen webkitallowfullscreen mozallowfullscreen
    style=width:640px;height:360px;></iframe>
