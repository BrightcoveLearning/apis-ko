알림: 소스 변경

2019년 3월 동안 Brightcove는 Dynamic Delivery용 인프라를 업데이트하여 CMS/Playback API에서 반환된 소스를 약간 변경하여 Brightcove Player 및 기본 SDK 플레이어에 전달합니다.

대다수의 Video Cloud 사용자에게 이 변경 사항은 완전히 투명합니다. 당신이 할 필요가 없습니다. 기존 소스의 정확한 구조를 기반으로 통합을 구축한 일부 고객의 경우 이러한 통합을 변경해야 합니다.

무엇이 바뀌고 있습니까?

현재 Dynamic Delivery는 Playback API 요청이 들어올 때 HLS의 2가지 버전(v4 및 v5)을 반환합니다. 일부 특별한 경우(일반적으로 특정 장치로 전달) 계정은 v3 전달을 위해 활성화됩니다.

각각에 대해 다음 DRM 유형도 반환합니다.

  • V3: Clear / AES-128(계정이 v3에 대해 활성화된 경우)
  • V4: 클리어 / AES-128
  • V5: 클리어 / AES-128 / 페어플레이

HLSv5를 제공하는 유일한 이유는 그것이 Fairplay를 지원하기 때문입니다. Clear 및 AES-128의 경우 반환하는 비디오와 매니페스트가 동일합니다.

그러나 URL이 다르기 때문에("v4" 또는 "v5"가 포함됨) 전달하고 캐시해야 하는 콘텐츠의 양을 두 배로 늘리고 있으며 이는 낭비이며 Dynamic Delivery와 SSAI의 성능에 영향을 미칩니다.

따라서 v5에 대해 Fairplay 매니페스트만 반환되도록 인프라를 변경하여 소스 세트가 이제 다음과 같이 표시되도록 합니다.

  • V3: Clear / AES-128(계정이 v3에 대해 활성화된 경우)
  • V4: 클리어 / AES-128
  • V5: 페어플레이

이것이 내 플레이어에게 영향을 미칩니까?

  1. 만약 너라면 ~하지 않다 Clear DRM 또는 AES-128 암호화 사용, 이 변경차이가 없다
  2. 만약 너라면 ~하다 Clear DRM 또는 AES-128 암호화를 사용하지만 플레이어가 소스를 자동으로 선택하게 하면 이 변경차이가 없다 (플레이어가 v4 소스를 선택합니다)
  3. 만약 너라면 ~하다 Clear DRM 또는 AES-128 암호화를 사용하고 플레이어가 v5 소스 URL을 사용하도록 사용자 정의 코드를 작성했습니다. 플레이어에 재생 가능한 소스가 없는 결과 - 해결책은 플레이어가 소스를 자동으로 선택하도록 하거나 대신 v4 소스를 선택하도록 코드를 수정하는 것입니다.
