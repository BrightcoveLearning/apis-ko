소개

NS Dynamic Ingest API강력하고 유연하지만 가장 효과적으로 사용하려면 아래에 자세히 설명된 몇 가지 모범 사례를 따라야 합니다.

모범 사례

다음은 Dynamic Ingest에 대한 규칙 및 지침입니다.

  • 당신은해야 ~ 아니다동일한 작업에서 비디오와 함께 이미지 및 텍스트 트랙과 같은 다른 자산을 수집하는 것은 괜찮지만 동일한 비디오에 대해 중복 작업을 처리해야 합니다.
    비디오의 이미지, 텍스트 트랙 또는 기타 자산을 수집하는 경우 ~ 전에 동영상 처리가 완료되면 ~ 아니다 동영상에 추가됩니다.

    동일한 비디오 ID에 여러 작업을 생성할 수 있는 수집 작업 생성을 위해 자동화된 프로세스를 사용하는 경우 다음을 확인해야 합니다. 알림새 작업을 만들기 전에 동일한 비디오에서 처리 중인 다른 작업이 없는지 확인합니다.

  • NS profile수집 요청 필드는선택 과목 . 하드코딩 피하기profile 앱 및 통합에서 - 대신 가장 일반적으로 사용되는 프로필을기본계정에 대해 필드를 생략하거나계정에 사용 가능한 프로필 가져오기사용자가 하나를 선택하도록 합니다.
  • 전체 요청에 대한 각 계정의 초당 요청 20개 제한동적 수집그리고 CMS API .
  • 계정당 동시 일반 우선순위 DI 작업은 100개로 제한됩니다.
  • 대용량 파일 배치를 수집하는 경우 다음을 사용하는 것이 좋습니다. 낮은 우선 순위 대기열 100개 이상의 작업을 대기열에 넣을 수 있습니다.
  • 수집 콘텐츠는 일반적으로 파일당 100GB로 제한됩니다(더 큰 파일을 수집해야 하는 경우 계정 관리자에게 문의하십시오).
  • 일괄 처리로 파일을 수집하는 경우 동시 일반 우선 순위 작업을 100으로 제한하고 다른 작업을 추가하기 전에 한 작업이 처리를 완료할 때까지 기다리십시오. 대체/재시도 전략자세한 내용은. 또는 다음을 설정할 수 있습니다. "우선순위": "낮음" - 낮은 우선 순위를 사용하면 처리 시간이 조금 더 오래 걸릴 수 있지만 수집 대기열의 크기에 대해 걱정할 필요가 없습니다.
  • 전체 배치를 수집하기 전에 원하는 결과를 얻을 수 있도록 수집 프로필 변경 사항을 확인합니다.
