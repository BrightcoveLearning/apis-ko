배경

Dynamic Ingest는 두 가지 방식으로 비율 제한(계정당)을 적용합니다.

  • 초당 20개 이하의 요청(CMS API 및/또는 Ingest API 요청)이 허용됩니다.
  • 100개 이상의 동시 일반 우선 순위 작업이 허용되지 않습니다.

첫 번째는 앱에서 관리하기 어렵지 않습니다. 요청 사이에 3초 이상의 지연을 부과할 수 있습니다. 두 번째는 현재 처리 중인 작업 수를 확인하기 위해 시스템에 직접 쿼리할 수 있는 방법이 없기 때문에 더 복잡합니다. 또는 다음을 사용하는 것이 좋습니다. 낮은 우선 순위 대기열 100개 이상의 작업을 대기열에 넣을 수 있습니다.

일정 시간 동안 기다렸다가 성공할 때까지 요청을 재시도할 수 있지만 다음을 수신하여 보다 합리적인 대체/재시도 시스템을 구현할 수 있습니다. 알림 Dynamic Ingest 시스템에서 정보를 사용하여 진행 중인 작업을 직접 추적합니다.

이를 구현하는 한 가지 방법은 수집 요청을 제출하고 알림을 수신하는 트랜시버 앱을 구축하는 것입니다. 아래 다이어그램은 이러한 앱의 상위 수준 논리를 보여줍니다.
