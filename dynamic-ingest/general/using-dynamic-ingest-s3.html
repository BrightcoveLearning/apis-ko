개요

Dynamic Ingest는 Amazon S3 버킷에서 비디오를 다운로드할 수 있습니다. S3 콘텐츠에 액세스할 수 있는 경우모든 사람 , 당신이 할 필요가 없습니다. 그러나 콘텐츠가 보호되는 경우 Dynamic Ingest가 동영상에 액세스할 수 있도록 허용하는 버킷 정책을 추가해야 합니다. 이 문서에서는 그 방법을 설명합니다.

Dynamic Ingest는 이제 V4 서명을 사용할 때 이미지 및 VTT 파일에 대해 V4 AWS 리전을 지원합니다. AWS 리전이 V4 서명만 허용하는 경우 V4 서명을 사용해야 합니다.

S3 정책 설정

Dynamic Ingest가 동영상에 액세스할 수 있도록 정책을 설정하려면 AWS 콘솔 . 버킷을 선택한 다음 "권한" 탭을 선택한 다음 "버킷 정책" 버튼을 클릭합니다. 마지막 행에서 클릭버킷 정책 추가 . 다음은 권장되는 정책입니다.

    {
     "Version": "2012-10-17",
     "Id": "VideoCloudBucketPolicy",
     "Statement": [
       {
         "Sid": "Stmt1295042087538",
         "Effect": "Allow",
         "Principal": {
             "AWS": "arn:aws:iam::395540211253:root"
         },
         "Action": [
         "s3:GetObjectAcl",
         "s3:GetObject"
         ],
         "Resource": "arn:aws:s3:::MY-BUCKET/*"
       },
       {
         "Sid": "Stmt1295042087538",
         "Effect": "Allow",
         "Principal": {
             "AWS": "arn:aws:iam::395540211253:root"
         },
         "Action": [
         "s3:GetBucketLocation"
         ],
         "Resource": "arn:aws:s3:::MY-BUCKET"
       }
     ]
     }

만 있다한 가지 변경해야 할 사항준비가 되기 전에 이 버킷 정책에:

  1. 바꾸다MY-BUCKET양동이 이름과 함께Resource윤곽. 떠나다/*그 명령문이 버킷 내의 모든 파일에 적용되기 때문에 첫 번째 명령문이 끝날 때.

    Video Cloud에 S3 버킷에 대한 액세스를 제공하기 때문에 계정 ID 또는 사용자 ID 값을 변경할 필요가 없습니다.
  2. 선택적으로 Action 배열을 허용하려는 작업으로 바꿉니다. Video Cloud에 필요한 유효한 조치는 다음과 같습니다.

    개체 권한 - 위의 예에서 첫 번째 작업 배열

    • s3:GetObject - Video Cloud가 버킷에서 파일을 다운로드하도록 허용
    • s3:GetObjectAcl - Video Cloud가 파일에 대한 ACL 권한을 얻도록 허용

    버킷 권한 - 위의 예에서 두 번째 작업 배열

    • s3:GetBucketLocation - Video Cloud가 버킷의 위치를 얻도록 허용합니다(미국, EU, 아시아 등).

    Video Cloud에는 쓰기 권한이 필요하지 않습니다.

정책에는 두 가지 설명이 있습니다. 첫 번째 문은 버킷 내의 키에 영향을 미치고 두 번째 부분은 버킷 자체에 영향을 줍니다.

나머지 정책은 그대로 둘 수 있으며 잘 작동합니다. Amazon 설명서에는 다음과 같은 추가 정보가 있습니다. 버킷 정책의 요소.
