소개

소스 파일 업로드를 통한 수집의 경우 Brightcove는 비디오 및 자산 파일을 업로드할 수 있는 S3 버킷을 제공하고 Dynamic Ingest는 자체 S3 버킷 또는 URL에서와 동일한 방식으로 S3 버킷에서 비디오를 가져옵니다. 아래 다이어그램은 기본 동적 수집 워크플로와 소스 파일 업로드를 통한 수집 워크플로의 차이점을 보여줍니다.

소스 파일 이름

Brightcove Player에서 비디오 및 자산 액세스 문제를 방지하려면 비디오, 이미지 또는 텍스트 트랙(WebVTT 파일)에 관계없이 소스 파일 이름에 특수 문자를 사용하지 않아야 합니다. 이는 원격 자산에도 적용됩니다. 파일 이름에는 다음만 포함되어야 합니다.

  • 싱글바이트문자(대문자 또는 소문자)
  • 숫자
  • 대시(-) 및 밑줄(_)
  • 스페이스 URL로 인코딩된 경우

입증

Dynamic Ingest에 대한 클라이언트 자격 증명을 얻는 가장 쉬운 방법은 API 인증을 위한 Studio 관리 페이지 . API 권한의 경우 최소한 다음이 필요합니다.

  • CMS > 비디오 읽기
  • 동적 수집 > 생성
  • 동적 수집 > 푸시 파일(새로운 소스 파일 업로드 API)

푸시 기반 수집에 대한 Brightcove API 요청에 대한 인증에는 요청에 대한 추가 권한이 필요합니다. 기타 동적 수집 요청 :

      video-cloud/upload-urls/read

소스 파일 업로드에 필요한 전체 권한 세트는 다음과 같습니다.

  • 비디오 클라우드/비디오/만들기
  • 비디오 클라우드/비디오/읽기
  • 비디오 클라우드/비디오/업데이트
  • 비디오 클라우드/업로드 URL/읽기

이러한 권한은 다음에서 사용할 수 있습니다. 사진관 . 또는 다음과 같이 POST 요청을 수행하여 OAuth API에서 직접 소스 파일 업로드 API를 사용하기 위한 클라이언트 자격 증명을 얻을 수 있습니다.

API 요청

푸시 기반 수집과 관련된 네 가지 API 요청이 있습니다.

  1. Video Cloud에서 비디오 객체를 생성하기 위한 CMS API POST 요청(풀 기반 수집과 동일)
  2. Brightcove S3 버킷 URL을 가져오기 위한 Dynamic Ingest GET 요청
  3. 소스 파일을 Brightcove S3 버킷에 업로드하기 위한 PUT 요청
  4. 소스 파일을 수집하기 위한 Dynamic Ingest POST 요청(풀 기반 수집과 동일)

NS CMS API그리고Dynamic Ingest API요청은 서버 측 앱에서 이루어져야 합니다. 대리 . S3 버킷에 파일을 업로드하기 위한 PUT 요청은 클라이언트 앱 또는 서버 앱에서 직접 수행할 수 있습니다.

웹사이트(클라이언트 측)에서 비디오를 업로드하는 경우 이 라이브러리는 원활한 사용자 경험을 위해 필요한 단계를 단순화합니다. https://github.com/BrightcoveOS/evaporate-brightcove . 이 라이브러리는 오픈 소스이며 Brightcove에서 공식적으로 지원하지 않습니다.

이러한 요청은 다음 섹션에 자세히 설명되어 있습니다.

CMS API 요청

NS CMS API요청은 새 비디오를 추가하기 위한 Dynamic Ingest 작업과 동일합니다. 이 요청은 새 동영상을 수집하는 데 필요합니다. 기존 비디오에 자산을 교체하거나 추가하는 경우 이 단계가 필요하지 않습니다. 대신 다른 요청에서 기존 비디오 ID를 사용합니다.

소스 파일을 S3에 PUT

S3 URL을 가져온 후 다음을 사용하여 비디오 파일을 업로드하기 위해 PUT 요청을 합니다. signed_url목적지로.

다음을 사용할 수 있습니다. 곱슬 곱슬하다 PUT 작업을 테스트하는 명령:

      curl -X PUT "SIGNED_URL_GOES_HERE" --upload-file FILE_PATH_FOR_LOCAL_ASSET_GOES_HERE

단일 및 다중 파트 업로드

AWS는 최대 5GB 크기의 파일에 대한 단일 부분 업로드를 허용합니다(파일 크기에 대한 다른 제한 없음). 더 큰 파일의 경우 멀티파트 업로드를 사용해야 합니다. 단일 부분 업로드가 설정하기가 다소 쉽지만 가능하면 다중 부분 업로드를 사용하는 것이 좋습니다. 둘의 차이점은 다음과 같습니다.

  • 단일 부분 업로드는 비디오를 모두 하나의 파일로 업로드합니다. 단일 부분 업로드는 5GB 이하의 파일 크기로 제한됩니다. 어떤 이유로 업로드가 중단되면 다시 시작해야 합니다.
  • 멀티파트 업로드는 파일을 청크로 푸시합니다. 업로드 시 여러 연결을 활용할 수 있으므로 더 효율적입니다. 또한 업로드가 중단되면 나머지 청크로 중단된 위치에서 다시 시작할 수 있습니다.
프로세스의 이 단계는 Brightcove 지원 범위를 벗어납니다. 파일을 추가하기 위한 PUT 요청은 AWS S3 규칙을 따라야 합니다. 아래 링크가 도움이 될 수 있습니다.
  • AWS S3 단일 작업 업로드
  • AWS S3 멀티파트 업로드
  • 미리 서명된 URL을 사용하여 AWS S3 업로드
