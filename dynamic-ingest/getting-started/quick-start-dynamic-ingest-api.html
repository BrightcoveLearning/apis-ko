소개

Dynamic Ingest API로 여러 가지 작업을 수행할 수 있지만 주요 기능은 비디오를 Video Cloud에 추가하는 것입니다. 이 빠른 시작에서는 API를 사용하여 첫 번째 비디오를 수집하는 과정을 안내하는 간단한 자습서를 제공합니다.

이 자습서에서는 HTTP 요청을 만들기 위해 CURL이라는 인기 있는 명령줄 도구를 사용합니다. CURL에 익숙하지 않은 경우 다음을 참조하십시오. CURL 설정 . CURL은 기본적으로 Mac 및 Linux 시스템에 설치됩니다. Windows에서는 설치해야 합니다. 지침은 CURL 설정 가이드에 있습니다.

단계

클라이언트 자격 증명 가져오기

이것은 API 요청에 대한 클라이언트 자격 증명을 만드는 일회성 단계입니다. 필요한 클라이언트 자격 증명이 이미 있는 경우 이 단계를 건너뛸 수 있습니다.

  1. 로 API 자격 증명 관리섹션.

  2. 딸깍 하는 소리신규 신청 등록

  3. 적절한 계정을 선택하고 적어도 이러한 권한:

    • 동적 수집 생성
    • 수집 프로필 구성 읽기
    • 수집 프로필 읽기

  4. 복사 및 저장클라이언트 ID그리고클라이언트 비밀생성됩니다(대화 상자를 닫으면 클라이언트 암호를 다시 얻을 수 있는 방법이 없습니다!).

    --user CLIENT_ID:CLIENT_SECRET \
    --header 'content-type: application/x-www-form-urlencoded' \
    --data grant_type=client_credentials

비디오 만들기

전화 CMS API Video Cloud에서 비디오 개체를 만듭니다. 요청 본문은 다음과 같은 메타데이터를 규정합니다. name그리고description . 유일한 필수 필드는name .

  1. 다음 코드를 텍스트 편집기에 복사하고 바꿉니다. ACCOUNT_ID자신의 Video Cloud 계정 ID로ACCESS_TOKEN이전 단계의 요청에 대한 응답의 토큰으로 
        curl --request POST \
    --url https://cms.api.brightcove.com/v1/accounts/ACCOUNT_ID/videos/ \
    --header 'authorization: Bearer ACCESS_TOKEN' \
    --header 'content-type: application/json' \
    --data '{"name": "My Video"}'
  2. JSON 응답에는 비디오에 대한 다양한 메타데이터가 포함되지만 우리의 목적을 위해 중요한 필드가 맨 위에 있습니다. 
        {
      "id": "123456789001",
      …
    }

    당신은 이것을 사용할 것입니다id Video Cloud에 비디오 소스 파일을 연결할 비디오를 알려주려면 아래의 Ingest 요청에 입력하십시오.

수집 요청

Dynamic Ingest에 대한 호출은 수집할 파일의 위치를 제공합니다.

  1. 다음 코드를 텍스트 편집기에 복사하고 바꿉니다. ACCOUNT_ID자신의 Video Cloud 계정 ID로VIDEO_ID이랑id비디오를 만들 때 반환된 값, ACCESS_TOKEN이전 단계의 요청에 대한 응답의 토큰으로 
        curl --request POST \
      --url https://ingest.api.brightcove.com/v1/accounts/ACCOUNT_ID/videos/VIDEO_ID/ingest-requests \
      --header 'authorization: Bearer ACCESS_TOKEN' \
      --header 'content-type: application/json' \
      --data '{
        "master": {
          "url": "https://learning-services-media.brightcove.com/videos/mp4/Great_Blue_Heron.mp4"
        },
        "callbacks": ["https://solutions.brightcove.com/bcls/di-api/di-callback-app.php"]
      }'

    여기의 비디오 및 콜백 앱은 테스트 목적으로 Brightcove Learning Services에서 제공합니다. 이를 자신의 비디오 및/또는 콜백 앱에 대한 경로로 바꿀 수 있습니다. 인제스트 작업에 대한 알림을 받을 필요가 없다면 생략 가능callbacks들.

    콜백 앱을 사용하는 경우알림 보기 . 이 앱은 다른 여러 앱에서 콜백에 사용되므로 작업 ID를 사용하여 작업과 관련된 항목을 검색하십시오.

  2. 다음과 같은 응답을 받아야 합니다. 
        {"id":"bcc3dd36-50de-4343-b42a-ad02bd8f0241"}
  3. 이 작업과 관련된 알림을 식별하거나 지원팀에 문의해야 하는 경우 작업 ID를 저장해야 합니다.

위의 단계를 성공적으로 완료했다면 Dynamic Ingest API를 사용하여 계정에 비디오를 수집한 것입니다. 어려운 점이나 질문이 있으시면 언제든지 연락하십시오브라이트코브 학습 서비스도와주기 위해.

API를 광범위하게 사용하려는 경우 CURL 이외의 다른 테스트 도구를 사용해 보는 데 관심이 있을 수 있습니다.
