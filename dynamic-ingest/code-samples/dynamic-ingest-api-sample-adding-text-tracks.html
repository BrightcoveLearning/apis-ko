개요

이 샘플 및 관련 코드는 프로덕션 개발을 위한 가이드로 제공됩니다. 코드는 프로덕션 용도로 인증되지 않았으며 공식적으로 지원되지 않습니다.

이 샘플에서는 한 번에 계정 20의 비디오를 표시하고(CMS API 사용) 사용자가 표시된 비디오 중 하나 이상에 WebVTT 캡션을 추가할 수 있습니다. 샘플을 쉽게 수정하여 썸네일 및/또는 포스터 이미지를 대신 추가하거나 비디오를 다시 코드 변환할 수 있습니다. 샘플의 요점은 기본 API 요청을 넘어 실제 앱으로 조합하는 방법을 보여주는 것입니다.

메모: Dynamic Delivery는 매니페스트에 캡션과 자막을 제공합니다. 이 경우 카탈로그에서 전송된 트랙보다 매니페스트 내 트랙을 선호합니다. 매니페스트의 트랙 유형은 "CLOSED-CAPTIONS"가 608/708 캡션용으로 예약되어 있기 때문에 "CLOSED-CAPTIONS"가 아닌 "SUBTITLES"만 될 수 있습니다. Brightcove Player는 자막을 자막으로 표시합니다.

아래는 앱 로직의 개략도입니다.

이 앱은 또한 하나 또는 여러 계정의 여러 사용자가 사용하도록 설계되었습니다. 실제로는 그렇게 하지 않지만 아래 설명에서 후크가 일부 백엔드 시스템에서 사용자 정보를 가져오는 위치를 볼 수 있습니다. (또는 대부분의 API 샘플과 마찬가지로 사용할 때마다 사용자에게 계정 ID와 클라이언트 자격 증명을 요청할 수 있습니다.)

동적 수집은 ~ 아니다여러 비디오를 처리하기 위한 메커니즘이 있습니다. 이 샘플은 단순히 비디오 세트를 반복하여 각 비디오에 대해 Dynamic Ingest 요청을 만듭니다.
