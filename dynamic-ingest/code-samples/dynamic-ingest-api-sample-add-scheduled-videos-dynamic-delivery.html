개요

이 샘플 및 관련 코드는 프로덕션 개발을 위한 가이드로 제공됩니다. 코드는 프로덕션 용도로 인증되지 않았으며 공식적으로 지원되지 않습니다.

계정에 새 비디오를 추가할 때 향후 어느 정도까지는 볼 수 있도록 비디오를 공개하고 싶지 않을 수 있습니다. 그러나 재생 가능한 변환이 생성되는 즉시 비디오를 재생할 수 있습니다. 누군가가 실수로 동영상을 게시하는 것 외에도 목록에 대한 동영상을 선택하는 검색 기준과 일치하기 때문에 스마트 재생 목록에 표시될 수 있습니다.

우발적인 미성숙 출시를 방지하는 가장 좋은 방법은 비디오를 수집하는 시점에 비디오를 사용할 수 있게 하려는 시간에 비디오를 예약하는 것입니다. 다음을 포함하여 간단히 수행할 수 있습니다. schedule요청 본문의 개체 CMS API비디오를 생성하는 호출:

              {
                "name": "greatblueheron.mp4",
                "schedule": {
                  "starts_at": "2017-03-10",
                  "ends_at": "2018-12-31"
                }
              }

NS starts_at그리고ends_at값은 전체 ISO 8601 날짜-시간 스탬프가 될 수도 있고 여기에 표시된 것처럼 날짜 부분만 포함할 수도 있습니다. 두 필드 모두 선택 사항입니다. 생략starts_at필드는 비디오를 즉시 사용할 수 있음을 의미합니다. 생략ends_at필드는 비디오를 무기한 사용할 수 있음을 의미합니다.
