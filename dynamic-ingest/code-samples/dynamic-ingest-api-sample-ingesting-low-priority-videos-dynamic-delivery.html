소개

이 샘플 및 관련 코드는 프로덕션 개발을 위한 가이드로 제공됩니다. 코드는 프로덕션 용도로 인증되지 않았으며 공식적으로 지원되지 않습니다.

이 샘플 앱은 우선순위가 낮은 수집 대기열을 사용하여 동영상 그룹을 수집하기 위해 데이터 세트를 반복합니다. 이 대기열을 사용하면 동영상 처리 시간이 더 오래 걸리지만 현재 처리 중인 작업 수의 제한에 대해 걱정할 필요가 없다는 장점이 있습니다(일반 우선 순위 대기열의 경우 해당 제한은 100개 작업). . 많은 수의 동영상을 계정에 추가하거나 이전하고 있고 동영상을 사용할 수 있을 때까지 하루 정도 기다릴 수 있다면 이 방법이 좋습니다. 보다동적 전달 수집 우선 순위 큐자세한 사항은.

제한 사항

  • 동적 수집은 ~ 아니다여러 비디오를 처리하기 위한 메커니즘이 있습니다. 이 샘플은 단순히 루프를 실행하여 각 비디오에 대해 Dynamic Ingest 요청을 만듭니다.
