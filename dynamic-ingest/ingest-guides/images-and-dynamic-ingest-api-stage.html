페이지 내용

    이미지 및 동적 인제스트 API

    이 항목에서는 비디오 용 이미지를 캡처하거나 수집하는 방법에 대해 설명합니다.

    서문

    6 가지 유형의 이미지를 수집하여 Video Cloud 비디오와 연결할 수 있습니다.

    포스터
    사용자가 비디오를 재생하기 전에 포스터 이미지가 Brightcove Player에 표시됩니다. 일반적으로 비디오에서 캡처 된 프레임이지만 반드시 그럴 필요는 없습니다.
    썸네일
    썸네일 이미지는 재생 목록, 최종 화면 등에 표시됩니다. 일반적으로 비디오에서 캡처 된 프레임이지만 반드시 그럴 필요는 없습니다.
    초상화
    일반적으로 휴대 전화 및 기타 기기에서 세로 모드의 포스터로 사용되는 이미지입니다. 필수 Brightcove 비콘 . 사양에 대한 자세한 내용은표이하.
    광장
    일반적으로 채널 로고 또는 그리드 디스플레이로 사용되는 이미지입니다. 필수 Brightcove 비콘 . 사양에 대한 자세한 내용은표이하.
    넓은
    일반적으로 휴대 전화 및 기타 기기에서 가로 모드의 포스터로 사용되는 이미지입니다. 필수 Brightcove 비콘 . 사양에 대한 자세한 내용은표이하.
    초광각
    일반적으로 TV와 같은 초광각 디스플레이의 포스터로 사용되는 이미지입니다. 필수 Brightcove 비콘 . 사양에 대한 자세한 내용은표이하.

    자동 크기 조정

    Studio 또는 Dynamic Ingest API를 통해 커스텀 포스터 및 / 또는 썸네일 이미지를 추가하면 기본적으로 Video Cloud는 계정의 기본 인제 스트 프로필에 정의 된 이미지 크기와 일치하도록 크기를 조정합니다.

    크기 조정 및 패딩은 Video Cloud 또는 Brightcove 비콘 . 이미지는 잘리지 않습니다.

    이미지 캡처

    트랜스 코딩 중에 Video Cloud는 중간 지점에서 비디오의 스냅 샷을 캡처하여 포스터 및 썸네일 이미지로 저장할 수 있습니다.

    Dynamic Ingest에 의해 생성되는 포스터 및 썸네일의 수는 각각 하나씩입니다. 프로필에 이미지 변환이 두 개 이상 있어도 포스터 및 축소판 변환을 두 개 이상 만들 수 없습니다.

    사용자 정의 수집 프로파일의 경우 이미지 캡처를 사용하려면 사용중인 수집 프로파일에 두 개의 특수 변환을 추가해야합니다. 보다수집 프로필의 이미지이하.

    또한 설정해야capture-images요청 데이터에 플래그 지정true트랜스 코딩 중에 포스터 및 썸네일을 비디오에서 캡처하려는 경우 (기본값) 만약선택한 프로필에는 이미지 변환이 포함됩니다. (프로필에 이미지 변환이없는 경우 기본값은capture-images이다false .)

    샘플 요청 데이터 :

          {
        "master": { "url": "http://learning-services-media.brightcove.com/videos/mp4/Bird_Titmouse.mp4" },
        "profile": "multi-platform-standard-static",
        "capture-images": true
      }
    다음을 포함해야합니다. profile필드에 이미지 변환이있는 프로필을 지정합니다 (모든 표준 프로필에 포함됨).
