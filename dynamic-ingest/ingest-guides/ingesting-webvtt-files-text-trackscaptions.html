설정

Dynamic Ingest 요청에 대한 설정은 비디오, 이미지, WebVTT 파일 또는 다음 모두를 수집하는지 여부에 관계없이 동일합니다.

요청 URL
        https://ingest.api.brightcove.com/v1/accounts/account_id/videos/video_id/ingest-requests
입증
인증에는 다음으로 전달된 액세스 토큰이 필요합니다. Bearer토큰Authorization헤더: 
        Authorization: Bearer {access_token}

S3에 대한 참고 사항

WebVTT를 보호된 S3 버킷에서 가져오면 Video Cloud가 파일에 액세스할 수 있도록 버킷 정책을 설정해야 합니다. 보다 S3에서 동적 수집 사용자세한 내용은.

WebVTT 파일 수집

비디오에 WebVTT 캡션을 추가하거나 Dynamic Ingest를 사용하여 기존 비디오에 업로드할 수도 있습니다.

비디오 수집 중 텍스트 트랙 추가

요청 URL

        https://ingest.api.brightcove.com/v1/accounts/account_id/videos/video_id/ingest-requests

방법

POST

샘플 요청 본문

        {
            "master": {
                "url":"http://learning-services-media.brightcove.com/videos/mp4/Water-Droplet.mp4"
            },
            "capture-images": false,
            "text_tracks": [
                {
        "url": "http://learning-services-media.brightcove.com/captions/for_video/Water-in-Motion.vtt",
        "srclang": "en",
        "kind": "captions",
        "label": "English",
        "default": true
                }
            ]
        }

SRT 또는 DFXP 추가/변환

을위한 Dynamic Delivery 비디오 전용 , 같은 방식으로 SRT 또는 DFXP 파일을 수집할 수 있습니다. Video Cloud 수집 시스템은 자동으로 WebVTT 형식으로 변환하여 비디오에 추가합니다.

참고:
  • DFXP 파일에 설정된 언어xml:lang에 대한 수집 호출의 값과 일치해야 합니다. srclang . 예를 들어, srclang수집 요청의 값은en , DFXP 파일은 다음을 포함해야 합니다. xml:lang="en" .
  • SRT 파일의 사용자 지정 형식(예: 페이딩)은 WebVTT로 변환될 때 트랙에서 제거됩니다.
        {
            "text_tracks": [
              {
                "url": "http://media.brightcove.com/captions/Water.dfxp",
                "srclang": "fr",
                "kind": "captions",
                "label": "français",
                "default": true
              }
            ]
          }

텍스트 추가 트랙 기존 비디오

기존 비디오에 텍스트 트랙을 추가하는 것은 요청 본문에 비디오 정보나 프로필이 필요하지 않다는 점을 제외하고 완전히 동일합니다.

요청 URL

        https://ingest.api.brightcove.com/v1/accounts/account_id/videos/video_id/ingest-requests

방법

POST

샘플 요청 본문

        {
          "text_tracks": [
            {
              "url": "http://learning-services-media.brightcove.com/captions/for_video/Water-in-Motion.vtt",
              "srclang": "es",
              "kind": "captions",
              "label": "Español",
              "default": true
            }
          ]
        }

노트:

  1. 캡션 파일 이름에는 문자(대문자 또는 소문자), 숫자, 대시(-) 및 밑줄(_)만 포함해야 합니다.
  2. NS label캡션의 경우 Brightcove 플레이어가 캡션 언어 메뉴에 사용합니다. 가장 좋은 방법은 레이블에 적절한 언어를 사용하는 것입니다. 예를 들어: français ~을위한French기타 등등.
  3. 캡션의 텍스트는 일본어 및 한국어와 같은 2바이트 언어에 대해 제대로 표시되도록 UTF-8로 인코딩되어야 합니다.
  4. 기본값은 트랙당이 아니라 동일한 언어( srclang ).
  5. 이미지 및 텍스트 트랙의 경우 공개 HTTP 또는 HTTPS URL 또는 서명된 S3 URL만 지원됩니다.
  6. 텍스트 트랙의 경우 Dynamic Ingest는 재인코딩 요청인 경우에도 이를 비디오의 vtt 목록에 추가/추가합니다. 당신은 사용해야합니다 CMS API관리(업데이트/삭제)합니다.
  7. Dynamic Ingest는 한 번에 최대 100개의 텍스트 트랙을 허용합니다.
  8. 하지만"mime_type": "text/vtt"일반적으로 WebVTT에 대해 지원되지만 Dynamic Ingest는 ~ 아니다현재 텍스트 트랙의 수집을 위해 해당 필드를 허용합니다.
