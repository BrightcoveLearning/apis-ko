오디오 트랙 메타데이터

오디오 트랙에 연결된 일부 메타데이터 필드가 있습니다. 이 중 일부는 트랙을 수집할 때 설정되고 다른 일부는 다음으로 생성됩니다. 비디오 클라우드 . 이러한 필드 중 일부는 읽기 전용이고 다른 필드는 사용자가 업데이트할 수 있습니다. 메타데이터 필드는 관련 API 요청의 맥락에서 아래에 자세히 설명되어 있지만, Brightcove Player가 여러 오디오 트랙을 처리하는 방법을 결정하는 데 중요한 두 가지 특정 필드에 대해서는 여기에서 약간의 설명이 필요합니다.

language

NS language수집하는 동안 각 트랙에 대해 설정된 필드는 트랙의 언어를 지정합니다. 트랙이 비디오에서 구어체의 더빙인 경우 중요합니다. 이 필드의 값은 다음과 같은 두 글자 코드입니다. en또는hi . 언어 코드의 전체 목록은 다음을 참조하십시오. Subtag값 http://www.iana.org/assignments/language-subtag-registry/language-subtag-registry .

variant

NS variant필드는 오디오 트랙의 종류를 설명합니다. 표준 의미와 함께 가능한 값은 다음과 같습니다.

  • main - 메인 트랙, 일반적으로 비디오 파일에 다중화되는 트랙
  • alternate - 대체 오디오 트랙
  • commentary - 비디오 트랙에 대한 해설을 제공하는 오디오 트랙
  • dub - 다른 언어로 된 음성 단어의 더빙 버전이 포함된 트랙
  • descriptive - 트랙은 어떤 식으로든 비디오 콘텐츠를 설명합니다.

계정 기본값

에 대한 계정 기본값을 설정할 수 있습니다. language그리고variant Brightcove Player에서 어떤 오디오 트랙을 기본값으로 처리할지 결정합니다(아래 섹션에서 볼 수 있는 것처럼 트랙 메타데이터를 업데이트하여 기본값을 재정의할 수도 있습니다). 계정의 기본값을 설정하려면 Brightcove 지원에 문의 .

          {

      }
  2. 위의 JSON(자리 표시자 텍스트를 동영상 이름으로 대체)을 요청 본문으로 제출하십시오. POST요청하다https://cms.api.brightcove.com/v1/accounts/YOUR_ACCOUNT_ID/videos
  3. 응답에서 많은 비디오 메타데이터를 얻을 수 있지만 여기서 중요한 부분은id (비디오 ID), 다음 단계에 필요합니다.

비디오 및 오디오 트랙 인제스트

다음으로 비디오 및 오디오 트랙을 수집합니다(이미지 및 텍스트 트랙과 같은 다른 자산을 추가할 수 있지만 여기서는 간단하게 유지하겠습니다). 조금 헷갈릴 수 있는 점은audio_tracks요청 본문에 대해 JSON에 두 번 나타납니다.

  • NS audio_tracks안에 있는 물건master객체는 비디오 파일에 포함된 오디오 트랙에 대한 메타데이터를 포함합니다(있는 경우 - 이것은 오디오의 muxed라고도 함) - 오디오 트랙이 이미 오디오 파일에 포함되어 있기 때문에 오디오 파일에 대한 URL 없이 메타데이터만 포함합니다. 비디오 파일. 인코딩된 오디오 트랙은 다음 규칙을 준수하는 한 스테레오, 5.1, 7.1로 압축할 수 있습니다. 오디오 형식그리고 에 명시된 바와 같이알려진 문제 , 기본 오디오를 설정하는 것 외에는 오디오 트랙을 주문할 수 있는 다른 방법이 없습니다.
  • 최상위audio_tracks수집 중인 추가 오디오 트랙을 설명하는 개체 - 여기에는 오디오 파일의 URL과 기타 메타데이터가 포함됩니다.

기존 비디오에 오디오 트랙 추가

기존 비디오에 추가 오디오 트랙을 추가하는 절차는 CMS API이미 존재하기 때문에 비디오를 생성합니다. 그리고 요청서에는동적 수집 API를 사용하는 경우 비디오 파일의 URL 또는audio_tracks아래에master오디오의 muxed에 대한 메타데이터를 제공합니다. 그러나 기존 비디오의 오디오 트랙에서 muxed에 대한 메타데이터를 포함하려고 합니다. 오디오 트랙은 다음 규칙에 따라 압축되어야 함을 기억하는 것도 중요합니다. 오디오 형식 . 따라서 수집 요청에 대한 JSON은 다음과 같습니다.

수집을 위한 오디오 트랙 필드

오디오 트랙 필드
들 유형 설명
master.audio_tracks 물체[] 오디오의 muxed에 대한 메타데이터
master.audio_tracks.language 끈 하위 태그의 muxed in 오디오에 대한 언어 코드 http://www.iana.org/assignments/language-subtag-registry/language-subtag-registry
master.audio_tracks.variant 끈 오디오 트랙의 종류: main | alternate | dub | commentary | descriptive ( main일반적으로 오디오의 muxed에 사용됨)
audio_tracks 물체 추가 오디오 트랙에 대한 정보
audio_tracks.merge_with_existing 부울 이 트랙을 기존 트랙으로 병합할지 아니면 대체할지 여부

audio_tracks.masters.url 끈 오디오 트랙 파일의 URL
audio_tracks.masters.language 끈 에 있는 하위 태그의 오디오 트랙에 대한 언어 코드 http://www.iana.org/assignments/language-subtag-registry/language-subtag-registry
audio_tracks.masters.variant 끈 오디오 트랙의 종류: main | alternate | dub | commentary | descriptive ( main일반적으로 오디오의 muxed에 사용됨)

알림

하나 이상을 지정하는 경우콜백 URL(위의 수집 요청에 대한 샘플 JSON에서와 같이), 비디오 클라우드에서 수집한 각 오디오 트랙에 대한 알림을 보냅니다. 알림은 다음과 같이 표시됩니다.

      {
        "entity": "default/audio128",
        "entityType": "DYNAMIC_RENDITION",
        "version": "1",
        "action": "CREATE",
        "jobId": "0f703adb-0f17-4a35-8395-21c7fcdd2649",
        "videoId": "5298468208001",
        "dynamicRenditionId": "default/audio128",
        "accountId": "1910141565001",
        "status": "SUCCESS",
        "language" : "en",
        "variant" : "alternate"
      }

오디오 트랙에 대한 알림을 식별하려면language그리고variant알림의 필드. NS "action": "CREATE"그리고"status": "SUCCESS"필드는 트랙이 성공적으로 수집되었음을 나타냅니다.

비디오에 대한 모든 오디오 트랙 메타데이터 가져오기

비디오와 연결된 모든 오디오 트랙의 메타데이터를 검색하려면GET요청:

하나의 오디오 트랙에 대한 메타데이터 가져오기

비디오와 연결된 하나의 오디오 트랙에 대한 메타데이터를 검색하려면GET요청:

      https://cms.api.brightcove.com/v1/accounts/account_id/videos/video id/audio_tracks/audio_track_id

응답은 각 오디오 트랙에 대한 메타데이터를 포함하는 개체입니다. 자세한 내용은 아래 응답 필드 표를 참조하십시오.

오디오 트랙 메타데이터 업데이트

다음을 작성하여 오디오 트랙에 대해 쓰기 가능한 메타데이터 필드를 업데이트할 수 있습니다. PATCH요청:

      https://cms.api.brightcove.com/v1/accounts/account_id/videos/video id/audio_tracks/audio_track_id

요청 본문에 수정하려는 필드를 포함합니다. 예를 들면 다음과 같습니다.

      {
          "language": "es",
          "is_default": true
      }

오디오 트랙 삭제

오디오 트랙을 삭제하려면 DELETE 요청을 다음 주소로 보내십시오.

      https://cms.api.brightcove.com/v1/accounts/account_id/videos/video id/audio_tracks/audio_track_id

삭제 프로세스가 비동기적이고 즉시 완료되지 않을 수 있기 때문에 성공 응답 코드는 204(내용 없음)가 아니라 202(수락됨)일 수 있습니다.

language 끈 아니요 에 있는 하위 태그의 오디오 트랙에 대한 언어 코드 http://www.iana.org/assignments/language-subtag-registry/language-subtag-registry
variant 끈 아니요 오디오 트랙의 종류: main | alternate | dub | commentary | descriptive ( main일반적으로 오디오의 muxed에 사용됨)
duration 숫자 예 트랙의 길이(밀리초)
encoding_rates 숫자[] 예 이 트랙의 사용 가능한 인코딩 목록(bps)
is_default 부울 아니요 true이면 재생을 위한 기본 트랙이 사용됩니다(모든 계정 수준 기본값 재정의).

재생

Brightcove 웹 및 SDK 플레이어가 여러 오디오 트랙을 처리하는 방법에 대한 자세한 내용은 다음을 참조하십시오.

  • 다중 오디오 트랙(웹 플레이어)
  • 네이티브 SDK를 사용한 다중 오디오 트랙

최종 사용자 친화적인 라벨
오디오 트랙에 대한 사용자 정의 레이블은 지원하지 않습니다. 필요한 경우 Player API를 통해 클라이언트 측에서 변경해야 합니다.
경우에 따라 트랙을 전환하면 Brightcove Player가 불안정해질 수 있습니다.
  • 모든 오디오 세그먼트가 다운로드되기 전에 트랙 전환
  • Silverlight 플러그인을 사용하여 비디오를 재생하는 경우(IE 버전 10 미만 또는 Window 버전 8 미만의 모든 IE 버전) - 여러 오디오 트랙이 ~ 아니다 Silverlight에서 지원됩니다.
  • 오디오와 비디오의 길이가 다른 경우 더 짧은 길이가 모두 소진될 때마다 플레이어가 중지될 수 있습니다.
