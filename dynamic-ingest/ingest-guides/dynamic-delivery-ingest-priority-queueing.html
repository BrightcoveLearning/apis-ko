페이지 내용

    동적 배달 수집 우선 순위 큐

    이 항목에서는 우선 순위 설정을 사용하여 우선 순위가 높은 비디오의 수집 시간을 최적화하는 방법에 대해 설명합니다.

    서문

    이제 Dynamic Delivery Ingest에는우선 순위 대기열게시자가 원하는 우선 순위 설정으로 처리 작업을 제출하여 작업이 처리되는 순서와 적시성에 영향을 미칠 수 있습니다.

    Priority Queuing은 다음에 대해 활성화 된 계정에만 적용됩니다. 동적 전달 . 레거시 수집 시스템을 사용하는 계정에서는 사용할 수 없습니다.

    Dynamic Delivery 수집 큐잉 작동 방식

    보통 우선 순위 대기열을 사용하는 경우 Dynamic Delivery Ingestion은 계정 당 활성 작업을 100 개로 제한합니다. 이 한도를 초과하면 수집 시스템이 나중에 처리 할 추가 요청을 대기열에 추가합니다. 대기열의 크기에는 별도의 제한이 있으며, 이에 도달하면 고객에 대한 수집 요청을 다시 거부합니다 (429 오류 코드 포함). 작업이 완료되면 용량이 비워지고 대기열에있는 작업이 수신 된 순서대로 한 번에 하나씩 선택되고 처리됩니다.

    비디오 상태CMS API작업이 대기열에 있음을 반영하지 않습니다. 상태는pending 작업이 처리 중인지 대기 중인지 여부.

    Priority Queuing이 수집에 미치는 영향

    우선 순위 대기열을 통해 사용자는priority 수집 요청에 플래그를 지정합니다. 허용되는 값priority 아르low 과normal . 다른 값을 사용하면 422 오류 코드와 함께 요청이 거부됩니다. 사용자가 우선 순위를 지정하지 않으면 기본값은normal 사용. 다음은 Priority Queuing이 대기열에서 작업이 처리되는 방식을 변경하는 방법에 대한 간략한 설명입니다.

    1. 대기중인 작업이없고 작업을 실행할 수있는 용량이있는 경우 작업이 즉시 실행됩니다. 이는 우선 순위가 낮은 작업과 보통 작업 모두에 적용됩니다.
    2. 다른 작업을 실행할 용량이 없으면 작업이 대기열에 추가됩니다.
    3. 대기열에 작업이 있으면 새 작업도 대기열에 추가됩니다. 이는 대기열에있는 작업 전에 새 작업을 시작할 수 없음을 의미합니다.
    4. 다른 작업을 실행할 수있는 용량이 있고 대기열에있는 작업이있는 경우 대기열에서 작업을 가져옵니다.
      • 대기열에 보통 우선 순위 작업이 있으면 가장 오래된 보통 우선 순위 작업이 선택됩니다.
      • 대기열에 보통 우선 순위 작업이 없으면 가장 오래된 우선 순위가 낮은 작업이 선택됩니다.
    5. 보통 및 낮은 우선 순위 작업은같은몇 개의 실행중인 작업이있을 수 있는지. 우선 순위에 관계없이 처리되는 총 작업 수는 계정 당 100 개로 제한됩니다.
    6. 있습니다갈라진대기 할 수있는 정상 및 낮은 우선 순위 작업 수에 대한 할당량.

    주의해야 할 사항

    작업 제출 후 변경된 변환 세부 정보

    Dynamic Ingest 작업을 생성하면 지정된 수집 프로필이 Dynamic Ingest 시스템과 해당 프로필에 의해 복사됩니다. 부작업이 제출 된 이후 프로필이 수정 된 경우에도 처리에 사용됩니다.

    경우에 따라 상당한 시간 동안 처리가 지연 될 수있는 우선 순위가 낮은 작업의 경우해당 프로필에 지정된 변환을 수정할 수 있습니다.작업이 실제로 처리되기 전에 이 경우 비디오를 처리하는 데 사용되는 변환이새로운그 렌 디션의 정의; 작업이 제출 된 당시의 변환 정의가 아닙니다.

    당신이 원한다면보증당신은 변환을 얻습니다작업이 제출 될 때 지정된대로 , 변환의 복사본을 만들고 복사 된 변환을 참조하는 새 프로필을 만든 다음 수집 프로세스를 시작해야합니다. (비디오가 이미 Video Cloud에 포함 된 경우 업데이트 된 프로필로 비디오를 다시 트랜스 코딩 할 수 있습니다. 원래 작업 처리가 완료 될 때까지 비디오를 다시 트랜스 코딩 할 수 없습니다.)

    소스 파일 업로드

    소스 파일 업로드는 24 시간 후에 해당 S3 버킷의 파일을 삭제합니다. 우선 순위가 낮은 작업은 경우에 따라 24 시간 이상 처리되지 않을 수 있으므로 파일이 이미 삭제되었을 수 있으며이 경우 처리가 실패합니다. 소스 파일 업로드에 낮은 우선 순위 수집을 권장하지 않습니다. .

    우선 순위 대기열 사용

    이전 섹션의 마지막 예에서아니사용priority 인제 스트 작업의 필드에 있으면 이전과 동일하게 계속 처리됩니다.

    추가priority 필드는 주로 다음과 같은 경우에 도움이됩니다.

    1. 상당히 많은 수의 동영상을 수집합니다.
    2. 얻는 것이 중요합니다약간가능한 한 빨리 온라인에 동영상을 게시하고 다른 사람을 게시 할 긴급한 상황은 없습니다 (콘텐츠가 덜 중요하거나 어쨌든 즉시 동영상을 게시 할 계획이 없기 때문).
    3. 재 트랜스 코딩하려는 동영상이 있지만 새 콘텐츠의 수집 속도가 느려지는 것을 원하지 않습니다.

    우선 순위가 낮은 대기열의 작동 방식

    이 섹션에서는 우선 순위가 낮은 대기열이 작동하는 방식을 설명합니다.

    보통 또는 낮은 우선 순위 작업이 시작될 때 보장되지 않습니다. 그러나 보통 우선 순위 작업은 항상 낮은 우선 순위 작업보다 먼저 시작됩니다.

    정상 우선 순위 작업을 지속적이고 빠르게 제출하는 경우 우선 순위가 낮은 작업이 시작되기까지 상당한 시간이 걸릴 수 있습니다.

    우선 순위가 낮은 작업은 시작하는 데 더 오래 걸리고 동일한 비디오에 대해 보통 우선 순위보다 처리하는 데 더 오래 걸릴 수 있습니다.
