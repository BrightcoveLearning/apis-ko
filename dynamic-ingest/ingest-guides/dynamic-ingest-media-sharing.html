소개

이 가이드의 Dynamic Ingest 부분은 다른 문서 및 샘플에서 설명한 프로세스와 다르지 않으며 Studio의 업로드 모듈을 사용하여 비디오를 수집하는 것도 사용할 수 있습니다. 그러나 완성도를 위해곱슬 곱슬하다여기에 세 단계 모두에 대한 예가 나와 있습니다.

  1. 비디오 만들기
  2. 수집 요청 만들기
  3. CMS API로 비디오 공유

입증

이 항목에 설명된 모든 요청에는 다음을 통한 인증이 필요합니다. 브라이트코브 OAuth 시스템 .

비디오 만들기

Video Cloud 시스템의 비디오 개체는 다음을 사용하여 생성됩니다. CMS API . 다음은 컬의 예입니다.

    curl --request POST \
      --url https://ingest.api.brightcove.com/v1/accounts/MASTER_ACCOUNT_ID/videos \
      --header 'authorization: Bearer OAUTH_TOKEN' \
      --header 'content-type: application/json' \
      --data '{"name": "Video to be shared"}'

반환된 JSON에는 새 동영상 ID가 포함됩니다.

    {
      "id": "VIDEO_ID",
      "account_id": "ACCOUNT_ID",
      ...
    }

수집 요청 만들기

그런 다음 다음을 사용하여 비디오를 수집할 수 있습니다. 동적 수집 API비디오 ID가 CMS API요구. 다음은 요청의 컬 예입니다.

    curl --request POST \
      --url https://ingest.api.brightcove.com/v1/accounts/MASTER_ACCOUNT_ID/videos/VIDEO_ID/ingest-requests \
      --header 'authorization: Bearer OAUTH_TOKEN' \
      --header 'content-type: application/json' \
      --data '{"master": {"url": "http://example.com/video.mp4"}}'

CMS API로 비디오 공유

비디오가 수집되면 다음을 사용하여 다른 계정과 공유할 수 있습니다. CMS API .

비디오를 다른 계정과 공유하려면 먼저 공유 관계를 설정해야 합니다. CMS API로 미디어 공유자세한 사항은.

다음은 요청의 컬 예입니다.

    curl --request POST \
    --url 'https://cms.api.brightcove.com/v1/accounts/MASTER_ACCOUNT_ID/videos/VIDEO_ID/shares?=' \
    --header 'authorization: Bearer ACCESS_TOKEN' \
    --header 'content-type: application/json' \
    --data '[
      {
        "id":"AFFILIATE_ACCOUNT_ID"
      }
    ]'

응답에는 공유 계정에 있는 새 비디오의 ID가 포함됩니다.

    [ {
      "video_id" : "5555086013001",
      "affiliate_id" : "1752604059001",
      "affiliate_video_id" : null,
      "status" : "PROCESSING",
      "shared_at" : "2017-09-01T12:04:39.911Z",
      "updated_at" : "2017-09-01T12:04:39.911Z"
    } ]
