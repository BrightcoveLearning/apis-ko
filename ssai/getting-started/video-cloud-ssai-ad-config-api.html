광고 구성은 광고 호출, 비콘 및 기타 구성 옵션을 포함하여 SSAI 재생의 다양한 측면을 정의합니다. 계정에는 여러 구성이 있을 수 있으며 현재 구성은 계정 간에 공유할 수 없습니다.

Video Cloud Studio에서 광고 구성을 쉽게 생성, 나열 및 삭제할 수 있습니다. 자세한 내용은서버 측 광고 설정 구성문서.
노트:
  • SSAI를 사용하려면 Video Cloud 계정을 다음으로 구성해야 합니다. 동적 전달 SSAI에 대해 활성화되었습니다. 이 기능을 사용하려면 계정 관리자에게 문의하세요.
  • SSAI는 주문형 비디오(VOD)와 함께 작동하며 라이브 재생을 지원합니다.
  • 제한 목록은 다음을 참조하십시오. SSAI 개요문서.

일반 정보

다음 정보는 모든 SSAI API 요청과 관련됩니다.

기본 URL

SSAI API의 기본 URL은 다음과 같습니다.

https://ssai.api.brightcove.com/v1

계정 경로

모든 경우에 특정 Video Cloud 계정에 대한 요청이 이루어집니다. 따라서 항상 용어를 추가합니다. accounts기본 URL에 대한 계정 ID:

https://ssai.api.brightcove.com/v1/accounts/your account id

입증

요청에 대한 인증에는 Authorization 헤더가 필요합니다.

Authorization: Bearer your access token

NS access_token Brightcove OAuth 서비스에서 가져와야 하는 임시 OAuth2 액세스 토큰입니다. 클라이언트 자격 증명을 얻고 이를 사용하여 액세스 토큰을 검색하는 방법에 대한 자세한 내용은 다음을 참조하십시오. Brightcove API 요청에 대한 인증 .

운영

클라이언트 자격 증명을 요청할 때 원하는 계정 액세스 또는 작업 유형을 지정해야 합니다. 다음은 SSAI API에 대해 현재 지원되는 작업 목록입니다.

  • SSAI 데이터:

    video-cloud/ssai/read
    video-cloud/ssai/all

구성 필드 세부정보

이 표는 요청의 본문 섹션 내에서 사용되는 광고 구성 필드를 정의합니다.

들 유형 설명
name 끈 사람이 읽을 수 있는 이름
vmap_response_namespace 끈 레거시 Unicorn Once 네임스페이스 형식을 사용하거나 새로운 Brightcove 네임스페이스 형식을 사용하도록 VMAP 출력을 조정합니다.

값: uo , bc기본: bc
description 끈 사람이 읽을 수 있는 설명
ad_config.expected_ad_response 끈 응답을 구문 분석하는 데 사용할 기술입니다. [1]

값:
  • dfp_ad_rules
  • dfp_vmap
  • smart_xml
  • vast_3_0
ad_config.disable_server_beacons 부울 광고 노출/비콘의 서버 측 실행을 비활성화할지 여부를 플래그합니다.

로 설정하면true , SSAI는 서버 측 비콘을 실행하지 않으며 VMAP 출력에 모든 비콘을 포함합니다.

로 설정하면false , SSAI는 서버 측에서 실행할 수 있는 비콘을 실행하고 VMAP 출력에 포함할 수 없는 비콘을 포함합니다.
ad_config.round_up_cue_points 부울 광고를 삽입하기 위해 가까운 위치를 선택할 때 다음 키프레임으로 반올림할지 여부에 플래그를 지정합니다.

기본: false , 원하는 광고 위치에 가장 가까운 키프레임을 선택합니다.
ad_config.template_url.template 끈 광고 태그 템플릿입니다. 사용 가능한 변수는 이후 섹션에서 설명합니다.
ad_config.template_url.defaults 물체 url 매개변수가 제공되지 않은 경우 사용할 기본값을 정의하는 String 대 String의 맵입니다.

리터럴이 될 수 있습니다{ "url.foo": "SomeString" }

또는 다른 변수 참조{% raw %}{ "url.foo": "{{ metadata.title_id }}" }{% endraw %}
discontinuities.dash [2] []끈 다중 기간 대시 매니페스트를 제공할 대시 버전을 제어합니다.

로 설정["*"]모든 버전에 대해 다중 마침표 대시 제공

절대 빈 목록

예시: ["live-timeline"] hbbtv용이 아닌 라이브 타임라인용으로 제공
discontinuities.hls [2] []끈 불연속으로 전달할 hl 버전을 제어합니다.

로 설정["*"] HLS의 모든 버전에서 단절된 상태로 제공

절대 빈 목록

예시: ["v4","v5"] v4 및 v5용으로 제공하지만 v3용은 제공하지 않음
beacon_templates 정렬 실행할 비콘 배열(예: 타사 비콘)
beacon_templates[].type 끈

발사할 신호의 유형입니다. 값:

  • content_start
  • content_first_quartile
  • content_midpoint
  • content_third_quartile
  • content_complete
  • content_quartiles
  • content_interval
  • ad_start
  • ad_first_quartile
  • ad_midpoint
  • ad_third_quartile
  • ad_complete
  • ad_quartiles
  • ad_break_start
  • ad_break_end
  • segment_start
  • segment_end
  • on_load
beacon_templates[].template_url.template 끈 비콘 URL 템플릿
extend_beacon_guard_ttl 부울 Beacon Guard TTL(Time to Live)의 길이를 콘텐츠의 세션 TTL 길이로 설정합니다. 그렇지 않으면 기본값은 1분입니다.

다음 참고 사항은 위 표의 필드를 참조합니다.

[1] 때expected_ad_response로 설정됩니다vast_3_0 , SSAI는 Video Cloud에 정의된 각 광고 큐 포인트에 대해 하나의 VAST 호출을 수행합니다. VMAP 및 광고 규칙의 경우 SSAI는 초기 광고 응답에서 정의된 광고 시간을 기반으로 요청합니다.

[2] 불연속성 태그는 콘텐츠 비디오와 광고 세그먼트 간의 프레임 속도 인코딩 변경을 나타냅니다. 그들은 시스템에 렌더링 체인을 재구성하도록 지시합니다. 확장 가능한 SSAI의 경우 비디오 플레이어는 불연속성 태그가 있는 HLS와 여러 마침표가 포함된 MPEG DASH를 재생할 수 있어야 합니다.

시스템 변수

시스템 변수는 SSAI 시스템에서 제공하며 임의 값을 생성하기 위한 최종 사용자 또는 도우미 변수에 대한 정보일 수 있습니다. 값은 URL 템플릿에 삽입되기 전에 URI로 인코딩되어야 합니다.

시스템 변수는 다음과 같이 식별됩니다. {% raw %}{{system.*}}{% endraw %}

들 설명
ad.position_time 광고 요청을 트리거한 큐 포인트의 시간(초)입니다. VAST 광고 응답 유형에만 사용 가능
ip_address 최종 사용자의 IP 주소
random_number_32 임의의 32비트 정수
random_guid 무작위 UUID
referer 최종 사용자의 참조자 헤더 값
timestamp_utc 현재 시간을 유닉스 타임스탬프로
unique_user_id MD5(ip_address + user_agent)
unix_timestamp Unix 타임스탬프로 표시된 현재 시간(초)
user_agent 최종 사용자의 User-Agent 헤더 값
uuid 임의의 uuid
x_forwarded_for 최종 사용자의 X-Forwarded-For 헤더 값
xfp.correlator 임의의 64비트 정수
xfp.ip_address 최종 사용자의 IP 주소
xfp.unique_user_id MD5(ip_address + user_agent)
xfp.scor 임의의 64비트 정수

진입점 VMAP/매니페스트에 제공된 쿼리 매개변수는url네임스페이스. 다른 네임스페이스의 변수와 달리 이러한 매개변수는 템플릿에 삽입할 때 url로 인코딩되지 않습니다. 변수 값이 광고 제공자로 가는 URL 인코딩이 필요한 경우 진입점 URL에 이중 URL 인코딩이 필요합니다.

URL 변수는 다음과 같이 식별됩니다. {% raw %}{{url.*}}{% endraw %}

메타데이터 변수

메타데이터 변수는 Video Cloud 및 Dynamic Delivery 데이터 소스에서 파생된 콘텐츠 비디오를 설명하는 변수입니다. 값은 URL 템플릿에 삽입되기 전에 인코딩된 URL입니다.

메타데이터 변수는 다음과 같이 식별됩니다. {% raw %}{{metadata.*}}{% endraw %}

들 설명
ad_keys Video Cloud Studio 미디어 모듈에서 추가 및 편집할 수 있는 자유 형식 텍스트 문자열
{% raw %}custom_fields.{field_name}{% endraw %} 비디오 클라우드 사용자 정의 필드
long_description Video Cloud 긴 설명
name 비디오 클라우드 비디오 이름
reference_id 비디오 클라우드 참조 ID
tags 비디오용 Video Cloud 태그의 쉼표로 구분된 목록
title.duration 콘텐츠 지속 시간(초)
title.id Dynamic Delivery 제목 ID
title.name Dynamic Delivery 제목 이름
video_id 비디오 클라우드 비디오 ID
다른 Video Cloud 키/값 쌍도 여기에 있습니다.

진입점 URL 매개변수

일부 동작을 조정하기 위해 SSAI 진입점 URL(VMAP 또는 매니페스트)에 추가할 수 있는 몇 가지 쿼리 매개변수가 있습니다.

매개변수 설명
?rule=sd-only 계정 구성에 설정된 임계값보다 높이가 낮은 모든 비디오 변환을 필터링합니다.
?rule=discos-enabled Dash의 HLS 및 MultiPeriods에서 불연속 재생을 활성화합니다. Playback Config의 불연속 설정보다 우선합니다.

구성 참고 사항

  1. SSAI로 광고를 미리 로드하면 안 됩니다. 그 이유는 플레이어를 미리 로드하면 동영상이 재생되기 전에 광고 노출과 첫 번째 사분위수 비콘이 보고되기 때문입니다. 이는 부정확한 광고 분석으로 이어질 수 있습니다. Studio에서 SSAI를 구성하면 이 작업이 자동으로 수행되지만 SSAI를 수동으로 설정하는 경우 이 문제를 알고 있어야 합니다.
  2. 웹 플레이어가 SSAI를 사용하고 있고 그렇게 하는 동기 중 하나가 광고 차단기를 해결하는 것이라면 서버 측 비콘을 사용해야 합니다. 클라이언트 측 비콘은 차단되므로 사용해서는 안 됩니다.

클라이언트 측 매크로

사용page클라이언트 측 광고 매크로를 사용하려는 경우 접두사. 이러한 매크로를 사용하면 VMAP 및 서버 URL에서 변수를 사용할 수 있습니다. 광고 매크로에 대한 자세한 내용은광고 매크로 및 serverURL IMA3 플러그인을 사용한 광고 문서 섹션.

클라이언트 측 매크로에는 다음과 같은 접두사가 붙습니다. {% raw %}{{page.*}}{% endraw %}

예를 들어 추가하려면document.referrer그리고pageVariable DOM 창 변수, 접두사page다음과 같이 광고 템플릿에서

{% raw %}https://adserver.com/{{page.document.referrer}}/{{page.pageVariable.whateverIwant}}{% endraw %}

재생 API 반환document.referrer그리고pageVariable.whateverIwant VMAP 및 SRC URL에 추가됩니다. 그런 다음 Brightcove 플레이어는 요청을 보내기 전에 클라이언트 측 매크로 교체 로직을 실행하여 적절한 값을 교체합니다.

{% raw %}https://bolt-prefix/blah.vmap?document.referrer={document.referrer}&pageVariable.whateverIwant={pageVariable.whateverIwant}{% endraw %}

광고 오류 비콘

SSAI를 사용할 때 VAST 광고 오류 비콘은 광고 워크플로의 문제를 사전에 찾고 수정하는 데 도움이 될 수 있습니다. 자세한 내용은 SSAI가 있는 광고 오류 비콘문서.
