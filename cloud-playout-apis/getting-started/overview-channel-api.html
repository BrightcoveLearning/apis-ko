요구 사항

  • 비디오 클라우드에서 활성화된 Cloud Playout API 인증 범위로 등록된 애플리케이션. ( 스튜디오 UI -> 관리 -> API 인증 )
Cloud Playout Channel Operations 외에도 CMS API를 사용하여 해당 콘텐츠를 관리해야 하므로 클라이언트 자격 증명에 필요한 모든 권한을 얻는 것이 좋습니다.

보다자세한 내용은 Brightcove API 인증을 참조하십시오.

CloudPlayoutChannelManager 워크플로 만들기

다음은 일반적인 Cloud Playout 채널 관리 워크플로입니다.

  • 채널 만들기
  • 재생 목록에 자산 및 라이브 이벤트 추가
  • 채널 활성화
  • 채널 시작
  • 채널 중지
  • 채널 삭제

이 외에도, 채널 가져오기채널 정보 및 채널 상태를 확인하는 데 유용합니다.

채널 만들기

채널을 생성하려면 채널 이름, 시작 시간, 입력 및 출력 그룹, 지역 및 live_profile과 같은 필수 속성이 필요합니다. DateTime 값은 ISO 8601 날짜 형식(UTC 또는 시간 오프셋 포함)에서 선호됩니다.

재생 목록에 콘텐츠 추가

다음 단계는 채널 콘텐츠에 콘텐츠를 추가하는 것입니다. 이를 위해 다음을 사용할 것입니다. CMS API (참고로 이것은 전체의 축약된 버전입니다. CMS API 참조 Cloud Playout과 관련된 작업 및 Cloud Playout에 더 적합한 설명 포함).

채널의 재생 목록 ID를 가져옵니다(채널 만들기 요청에 대한 응답 또는 채널 가져오기 요청에서). 이것은 Cloud Playout 채널의 자산을 관리하는 재생 목록입니다.

요구

채널 활성화

참고: 채널을 활성화하기 전에 채널 재생 목록에 하나 이상의 콘텐츠 자산이 추가되었는지 확인하세요!

아래 요청을 통해 채널을 활성화하세요. 요청 본문이 필요하지 않습니다.

샘플 요청

요구

을 만들다POST요청:

https://cm.cloudplayout.brightcove.com/accounts/{account_id}/cp_channels/{channel_id}/create

헤더

Content-Type: application/json
  Authorization: Bearer {token}
샘플 응답 
{
  "message":"Channel activation initiated"
}

채널 상태를 확인하려면 다음을 참조하십시오. 채널 섹션 가져오기위에.

채널 시작

아래 요청을 통해 채널을 시작하세요. 요청 본문이 필요하지 않습니다.

채널 중지

참고: 채널이달리기채널을 중지하기 전에 상태를 확인하십시오!

아래 요청을 통해 채널을 중지합니다. 요청 본문이 필요하지 않습니다.

샘플 요청

요구

을 만들다POST요청:

https://cm.cloudplayout.brightcove.com/accounts/{account_id}/cp_channels/{channel_id}/stop

헤더

채널 상태를 확인하려면 다음을 참조하십시오. 채널 섹션 가져오기위에. 일단 중지되면 채널은 초안 상태가 됩니다.

채널 삭제

참고: 채널이 둘 중 하나에 있지 않은지 확인하십시오. 생성/시작채널을 삭제하기 전의 상태!

아래 요청을 통해 채널을 삭제하세요. 요청 본문이 필요하지 않습니다.

헤더

  Authorization: Bearer {token}
