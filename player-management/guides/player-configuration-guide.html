소개

플레이어 구성에서 설정할 수 있는 속성은 아래에 자세히 설명되어 있습니다(왼쪽 탐색 목록에 나열됨). 계정 ID . 이 문서에는 속성 값을 보고 구성하는 방법에 대한 정보도 포함되어 있습니다.

당신은 사용해서는 안됩니다data-setup브라이트코브 플레이어로. 의 사용을 볼 수 있습니다. data-setup API 문서에 있지만 이는 해당 문서가 Video.js 플레이어 소스 코드에서 직접 생성되고 해당 속성을 다음과 함께 사용할 수 있기 때문입니다. video순수 Video.js로 태그를 지정합니다. 이 속성은 플레이어에 구성 정보를 보내지만 Brightcove Player는 다른 방법을 사용하여 이 작업을 수행합니다. data-setup신뢰할 수 없는.

구성 보기

이 섹션에 자세히 설명된 플레이어의 구성을 보는 방법에는 여러 가지가 있습니다.

스튜디오 사용

Studio를 사용하여 플레이어의 JSON 구성을 볼 수 있습니다. 다음 단계에서는 프로세스를 자세히 설명합니다.

  1. 열기선수모듈을 선택하고 구성을 보려는 플레이어를 찾으십시오.
  2. 플레이어에 대한 링크를 클릭하여 플레이어의 속성을 엽니다.
  3. 딸깍 하는 소리 JSON 편집기왼쪽 탐색 메뉴에서
  4. 다음과 유사한 플레이어의 구성을 볼 수 있습니다.
구성을 변경하고 저장하는 경우 매우 주의하십시오.

브라우징config.json

새 Brightcove 플레이어 구성의 인스턴스는 JSON 파일에 저장됩니다. 이 JSON 파일 또는 그 일부를 여러 가지 방법으로 볼 수 있습니다.

플레이어 구성을 보는 한 가지 방법은 다음으로 끝나는 플레이어의 URL을 변경하는 것입니다. index.html에게config.json . 예를 들면 다음과 같습니다.

    http://players.brightcove.net/1507807800001/HJyMlHiuZ_default/config.json

플레이어 구성의 일부 속성은 다음을 정의합니다.

  • 플레이어의 이름과 버전
  • 플레이어가 재생할 비디오
  • 플레이어가 속한 계정
  • 플레이어의 ID
  • 플레이어가 사용할 수 있는 모든 플러그인과 플러그인 코드의 위치에 대한 URL
  • 비디오 재생 전에 표시할 포스터

이름/설명 설정

사용하는 플레이어 구성을 변경하려면/{account_id}/players/{player_id}/configuration끝점이며 이것이 이 문서의 강조점입니다. 그러나 다음을 사용하여 변경하는 두 가지 속성이 있습니다. /{account_id}/players/{player_id}끝점. 그 두 속성은name그리고description .

참고: name플레이어에 대해 입력하면 다음과 같이 반환됩니다. player_name넣어 구성을 얻을 때config.json플레이어 미리보기 URL 끝(다음으로 반환됨name플레이어 관리 API를 통해 구성을 가져올 때. 그러나 당신은 사용해야합니다name플레이어를 생성하거나 업데이트할 때

첫 번째 코드 블록은 플레이어의name그리고description . JSON 응답은 PATCH 메서드에 의해 생성된 플레이어의 미리 보기 버전을 보여주므로 최적화된 버전을 얻으려면 플레이어를 게시해야 합니다.

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request PATCH \
    --data '{
      "name": "My New Player Name",
      "description": "My new player description"
    }' \

일부 구성 속성은 플레이어 관리 서비스에 의해 설정되며 수정할 수 없습니다. 사람들은:

  • 계정 ID
  • 플레이어 아이디

base_url(IP 제한 문제의 경우)

북미 이외의 지역에서 사용되는 Brightcove Player에서 IP 제한을 사용할 때 문제가 발생할 수 있습니다. Brightcove Player가 Video Cloud 계정에서 비디오를 로드하라는 요청을 받으면 이 주소의 카탈로그에 연결합니다. https://edge.api.brightcove.com/playback/v1/ .

해당 Edge 끝점은 IP 제한 문제를 일으킬 수 있는 방식으로 작동합니다.

  • WITHIN North America에서 Edge를 치면 Edge에 직접 부딪히며 Edge는 사용자의 IP 주소를 봅니다. IP 제한 화이트리스트에 따라 카탈로그에 대한 액세스를 허용하거나 거부할 수 있습니다. 이것은 원하는 기능입니다.
  • OUTSIDE North America에서 Edge를 칠 때 로드 밸런싱/로컬 캐싱 CDN 이유로 Cloudfront를 통해 이동합니다. 그 결과 Edge는 사용자의 IP 대신 Cloudfront IP를 보고 모든 사람을 거부하기 때문에 중요합니다. Cloudfront가 화이트리스트에 없기 때문입니다. 그러면 Cloudfront를 사용하는 모든 사람이 액세스할 수 있기 때문에 화이트리스트에 포함되어서는 안 됩니다.

이 문제를 해결하려면 대체 Edge 엔드포인트를 사용하도록 플레이어를 구성할 수 있습니다. edge-elb.api.brightcove.com . 이 변경은 북미 이외의 지역에서 IP 제한 계정에 액세스해야 하는 모든 플레이어에게 적용되어야 합니다.

이 접근 방식에는 단점이 있습니다. 카탈로그 요청이 로컬 서버가 있는 Cloudfront의 글로벌 네트워크를 통하지 않고 버지니아로 이동하기 때문에 미국 외부에서 캐시된 콘텐츠를 검색하는 데 약간의 지연이 도입됩니다. 이것은 거의 눈에 띄지 않을 것입니다.

실제 구성 변경에는 Video Cloud에 대한 값 설정이 포함됩니다. base_url . IP 제한이 북미 이외 지역의 Brightcove 플레이어에 사용되는 경우에만 변경해야 함을 기억하십시오. 기존 플레이어의 경우 다음을 사용하는 curl 명령을 사용합니다. PATCH HTTP 메서드(환경 변수를 데이터로 대체해야 합니다. $EMAIL , {account_id}그리고{player_id} ):

    curl
    --header "Content-Type: application/json"
    --user $EMAIL
    --request PATCH
    --data '{
        "video_cloud": {
      "base_url": "https://edge-elb.api.brightcove.com/playback/v1/"
        }
      }'
    https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration

플러그인

NS plugins속성은 video.js 플러그인을 초기화하는 순서와 플러그인에 적용할 사용자 정의 설정을 지정합니다. 플러그인은 해당하는scripts속성 URL 항목 및 필요한 경우 지원stylesheets URL. 의 각 요소는plugins배열은name속성이며 선택 사항을 포함할 수도 있습니다. options재산:

  • :name 지원하는 JavaScript 코드에서 가져온 video.js 플러그인의 이름
  • :options 초기화 시 플러그인에 제공할 설정 재정의 개체 해시

옵션 자식

이에options자식 속성에서 초기화 시 활용할 데이터를 전달합니다. 다음 JSON은options .

    "plugins": [{
      "name": "navigateOnVideoEnd",
      "options" : {"redirectURL": "http://docs.brightcove.com"}
    }]

이를 통해 플레이어 구성을 통해 플러그인에 데이터를 전달할 수 있습니다. 예를 들어 사용하려면redirectURL위에 표시된 개체에서 플러그인 자체에서 다음을 수행합니다.

    videojs.registerPlugin('navigateOnVideoEnd', function (options) {
      var myPlayer = this;
      myPlayer.on("ended", function () {
        window.location.href = options.redirectURL;
      });
    });

참고: 표준 플러그인 구현을 사용하지만options익명 함수의 매개변수로. 그런 다음 다음을 사용하여 값에 액세스할 수 있습니다. options.propertyName표기법.

NS 옵션 속성은 필요한 모든 데이터 구조가 될 수 있습니다(예: 객체 배열). 객체 배열을 전달하는 경우 데이터에 객체 배열로 액세스해야 함을 기억하십시오.
