자동 플레이어 업데이트

기본적으로 Brightcove는 정기적으로 모든 플레이어를 자동으로 업데이트하므로 고객의 노력 없이 새로운 기능과 수정 사항을 활용할 수 있습니다. 대부분의 경우 Brightcove는 업데이트를 선택 해제하는 것을 권장하지 않지만 Brightcove는 때때로 일부 고객이 플레이어가 완전히 손상되지 않은 상태로 유지되는 것을 선호한다는 것을 알고 있습니다(예: 큰 이벤트를 준비하기 위해 엄격한 테스트를 수행했지만 변화하는 모든 위험을 감수하고 싶습니다).

자동 업데이트 선택 해제

일정 기간 동안 플레이어 업데이트를 선택 해제해야 하는 경우 가장 간단한 방법은 플레이어 구성에 PATCH를 적용하는 것입니다. 다음 curl 문은 이 작업을 수행합니다.

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request PATCH \
    --data '{
        "player": {
            "template": {
              "locked": true
            }
        }
    }' \
    https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration

이렇게 하면 미리 보기 플레이어에 잠긴 설정이 추가됩니다. PATCH는 처음에 미리 보기 플레이어에만 영향을 미치므로 플레이어를 게시해야 합니다. Studio에서 또는 다음 curl 문을 사용하여 이 작업을 수행할 수 있습니다.

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request POST \
    https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/publish

자동 업데이트 다시 선택

업데이트를 다시 수신할 준비가 되면 다른 업데이트를 적용하기만 하면 됩니다. locked에게false그런 다음 플레이어를 다시 게시합니다. 먼저 locked를 false로 변경합니다.

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request PATCH \
    --data '{
      "player": {
        "template": {
          "locked": false
        }
      }
        }' \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration

현재 가지고 있는 것과 다른 경우 미리보기 플레이어를 현재 플레이어 템플릿 버전으로 업데이트합니다. 플레이어가 여전히 예상대로 작동하는지 확인하려면 이 호출에서 반환된 미리보기 플레이어 URL로 테스트해야 합니다. 만족스러우면 플레이어를 게시하여 새 플레이어를 라이브 상태로 만듭니다.

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request POST \
    https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/publish

특정 버전으로 전환

플레이어를 특정 버전으로 전환할 수 있습니다. 업데이트를 선택 해제할 수 있지만 업데이트를 선택 해제한 후에 제공된 특정 버전을 사용하고 싶을 수 있습니다. 이렇게 하려면: 물론 원하는 플레이어의 특정 버전으로 대체하여 다음 curl 문을 사용하십시오.

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request PATCH \
    --data '{
      "player": {
        "template": {
          "version": "1.14.11"
        }
      }
        }' \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration

버전을 설정하고 동시에 업데이트를 선택 해제할 수 있습니다.

    curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request PATCH \
    --data '{
      "player": {
        "template": {
          "version": "1.14.11",
          "locked": true
        }
      }
        }' \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration

모든 PATCH 업데이트와 마찬가지로 변경 사항에 만족하면 플레이어를 게시해야 합니다.

    curl \
    --header "Content-Type: application/json"cp
    --user $EMAIL \
    --request POST \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/publish
