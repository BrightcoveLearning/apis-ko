개요

플레이어를 CDN에 게시하기 위해 플레이어 관리 API는배달 시스템 API . Delivery System API를 사용하면 콘텐츠가 호스팅되는 git 저장소를 만들 수 있습니다. player.brightcove.net . 플레이어가 생성되면 Brightcove는 배후에서 리포지토리를 생성합니다. 게시자가 직접 요청하거나 Brightcove 플레이어 업데이트가 릴리스될 때 플레이어가 다시 게시될 때마다 해당 커밋 및 해당 저장소로의 푸시가 있습니다.

Git 리포지토리는 무엇보다도 시간 경과에 따른 변경 기록을 제공합니다. 플레이어를 디버깅하는 동안 변경 기록을 직접 검사할 수 있도록 이 리포지토리의 복사본을 얻는 것이 때때로 매우 유용할 수 있습니다. 먼저 명령줄 도구 curl 및/또는 Chrome 플러그인 Postman을 사용하여 플레이어에 대한 일반 정보를 검색하는 방법을 배웁니다. 그런 다음 생성 및 게시를 위해 플레이어 기록을 보는 방법을 배웁니다.

선수 정보

커맨드 라인에 익숙한 사람들은 플레이어 정보를 얻기 위해 curl을 사용하기를 원할 것이고 GUI에 더 익숙한 사람들은 Postman 옵션을 선택할 것입니다.

curl 및 환경 변수 설정에 익숙하지 않은 경우 처음 두 섹션을 읽으십시오. 단계별: 선수 관리 . 다음 환경 변수가 설정되어 있는지 확인하십시오.

  • 이메일
  • 계정 ID
  • PLAYER_ID

        --user $EMAIL \

우편 배달부 사용

Postman은 API와 상호 작용할 수 있는 보다 친숙한 UI를 제공합니다. Postman이 익숙하지 않다면 다음을 읽어보십시오. HTTP 요청에 Postman 사용기본적인 이해를 위한 문서. Postman을 사용할 때는 환경 변수를 생성하지 않습니다. 대신 URL에 적절한 값을 사용하여 URL 위치를 빌드합니다. 예를 들어 다음은 계정 및 플레이어 ID가 포함된 Postman에서 사용하기에 적절한 URL을 보여줍니다.

      https://players.api.brightcove.com/v2/accounts/1507807800001/players/6aac508a-741e-41a2-bc7a-5805644cf089/embeds/default

Postman을 사용하려면 다음을 수행하십시오.

  1. 양식 상단에 URL을 입력합니다.
  2. HTTP 메소드를 다음으로 설정하십시오. 가져 오기 .
  3. 다음을 사용하여 헤더 생성컨텐츠 타입그리고 해당 값애플리케이션/json .
  4. 클릭보내다단추.
  5. 메시지가 표시되면 Studio 자격 증명을 입력합니다.

그러면 응답이 반환됩니다. 보기 쉽도록 클릭하십시오. 예쁜그리고 JSON응답 바로 위에 있는 버튼.

선수 기록

이 섹션에서는 git 명령을 사용합니다. git에 대한 매우 간단한 자습서는 다음을 참조하십시오. 힘내 개요문서.

플레이어 저장소는 다음과 같이 처리되어야 합니다. 읽기 전용 . 이러한 리포지토리의 내용은 Player Management API에 의해 완전히 관리됩니다. 리포지토리에 직접 사용자 지정 추가 및 편집을 수행해서는 안 됩니다. 이러한 변경 사항은 플레이어가 다시 게시되거나 내부 업그레이드가 수행될 때마다 삭제되며 이는 언제든지 발생할 수 있습니다. 캡션, 글꼴 또는 플러그인에 대한 파일을 추가해야 하는 경우 해당 파일에 대한 새 저장소를 만들어야 합니다.

먼저 플레이어 저장소를 복제합니다. 발급한 디렉토리에 플레이어 ID의 이름으로 디렉토리가 생성됩니다. git clone명령.

  1. 터미널/명령 프롬프트를 열고 폴더를 만들 위치로 이동합니다.
  2. 다음 형식으로 플레이어 리포지토리의 URL을 만듭니다. 
          https://repos.api.brightcove.com/v1/accounts/ACCOUNT_ID/repos/PLAYER_ID_default

    예시:

          https://repos.api.brightcove.com/v1/accounts/1507807800001/repos/0d92f482-49d9-44e6-ba1d-e2360dab838a_default
  3. 명령 프롬프트에서 플레이어 리포지토리를 복제할 디렉터리를 만들거나 이동합니다.
  4. 해당 디렉토리에 있으면 다음 git 명령을 입력하여 자신의 계정과 플레이어 ID를 대체하여 플레이어 저장소를 복제합니다. 
          자식 복제 https://repos.api.brightcove.com/v1/accounts/1507807800001/repos/0d92f482-49d9-44e6-ba1d-e2360dab838a_default
    Enter 키를 누릅니다.
  5. 응답에 다음과 같은 내용이 표시되어야 합니다. 
          '0d92f482-49d9-44e6-ba1d-e2360dab838a_default'로 복제... 원격: 개체 계산: 39, 완료. 원격: 압축 개체: 100% (31/31), 완료. 원격: 총 39개(델타 13), 재사용 0(델타 0) 개체 풀기: 100%(39/39), 완료. 연결 확인 중... 완료되었습니다.
  6. 명령줄(또는 파일 관리자)에서 복제된 플레이어 디렉토리를 찾아 원하는 위치에 복제되었는지 확인합니다. 다음과 같이 이름이 지정됩니다. 
          {player_id}_default
  7. 명령줄에서 디렉터리를 이전 단계에서 찾은 복제된 플레이어 리포지토리 디렉터리로 변경합니다.
  8. 다음 명령을 입력합니다. 
          자식 로그
    Enter 키를 누릅니다.
  9. 다음과 유사한 응답이 표시되어야 합니다.
