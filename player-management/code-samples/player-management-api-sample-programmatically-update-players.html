신청 흐름

이 애플리케이션의 기본 논리는 다음과 같습니다.

  • 지정된 계정에서 모든 플레이어를 검색한 다음 업데이트하려는 플레이어 집합을 필터링합니다.
  • 필터링된 플레이어 집합의 ID로 배열을 만듭니다.
  • 필터링된 플레이어 집합에서 원하는 업데이트를 정의하고 그에 따라 각 플레이어가 업데이트되도록 요청합니다.
  • 업데이트된 각 플레이어를 게시합니다.
NS makeRequest함수는 Brightcove REST API를 사용하는 과정에서 사용하기 위해 거의 수정할 필요가 없는 표준 코드입니다. 이러한 이유로 이 문서에서는 다루지 않지만 다음 문서에 자세히 설명되어 있습니다. 학습 가이드: REST API 사용문서.
