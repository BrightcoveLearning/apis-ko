부인 성명

이 샘플 앱은 Player Management API의 기본 사항을 탐색하기 위한 방법입니다. 엔터프라이즈용 배포 가능한 앱이 아닙니다. 예를 들어 양식 유효성 검사는 완전하지 않으며 암호는 HTTPS를 사용하지 않고 전송됩니다.

이 샘플 앱의 아키텍처, 특히 인증 및 AJAX 구현과 관련된 일반적인 접근 방식에 대해 알아보려면 다음을 참조하세요. HTTPRequest를 사용하는 예 .

앱 아키텍처

이 앱의 상위 수준 응용 프로그램 논리는 다음과 같습니다.

  1. 플레이어 생성에 필요한 사용자 데이터 수집
  2. 플레이어 생성을 위해 API에 전달할 JSON 데이터 구조 빌드
  3. 버튼 클릭 시 AJAX 호출을 수행하여 플레이어 생성
  4. AJAX 호출 성공 시 다음을 수행합니다.
    • HTML 페이지에 JSON 응답 데이터 삽입
    • 플레이어를 게시하는 함수 호출
  5. AJAX 호출을 수행하여 플레이어 게시
  6. AJAX 호출 성공 시 다음을 수행합니다.
    • JSON 응답 데이터를 HTML에 삽입
    • 선택한 플레이어 정보를 표시하는 HTML을 동적으로 빌드하고 HTML 페이지에 삽입
