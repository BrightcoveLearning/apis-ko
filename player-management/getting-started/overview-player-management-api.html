소개

플레이어 관리 시스템은 세 가지 주요 개체로 구성됩니다. 기본 템플릿, 플레이어 및 하위 플레이어입니다.

브라이트코브 기본 템플릿 - 템플릿은 사용자에게 제공되는 시청 경험을 생성하기 위한 기반으로 사용되는 파일 및 설정의 집합이며 생성된 모든 플레이어의 기반입니다. 이러한 파일은 결국 사용자 정의된 플레이어 설정으로 컴파일되어 브라우저(플레이어)에 로드되는 실제 코드가 됩니다. 현재 시스템 전체에서 사용되는 단일 템플릿이 있습니다. 따라서 API를 통해 템플릿을 볼 수 없습니다.

선수 - 플레이어는 템플릿에 적용할 일반 정보 및 사용자 지정을 캡처하는 고객별 설정입니다. 이러한 설정은 API 및 플레이어 개체 결과로 전송됩니다. 사용자 정의에는 설정, 스타일 및 플러그인이 포함될 수 있습니다. 사용자 정의 가능성의 전체 목록은 다음을 참조하십시오. 플레이어 구성 가이드 .

어린이 선수 - Embed API를 사용하여 생성된 하위 플레이어는 플레이어의 여러 인스턴스를 생성한 결과입니다. 플레이어는 플레이어가 갖고자 하는 대부분의 속성을 가지고 있으며 하위 플레이어를 사용하여 다른 플레이어 인스턴스의 속성 하위 집합을 사용자 지정할 수 있습니다. 하위 플레이어는 지정된 플레이어에 추가할 미디어만 정의하거나 속해 있는 일반 플레이어를 변경하는 설정 또는 스타일 재정의를 정의할 수 있습니다. 상위 플레이어 설정에 대한 변경 사항은 상위 플레이어에서 생성된 모든 하위 플레이어에게 적용됩니다. 체크 아웃 Embed API 가이드자세한 내용은.

이 다이어그램은 기본 템플릿, 플레이어 및 하위 플레이어 간의 관계를 보여줍니다.

플레이어 관리 API를 처음 사용하면 계정에 Brightcove 기본 플레이어가 생성됩니다. 그 플레이어 개체는 당신이 선택한 대로 하는 것입니다. 그대로 사용하거나 설정을 변경하거나 스타일을 변경하거나 플러그인을 추가할 수 있습니다. 사용자 정의와 함께 Brightcove 기본 플레이어만 사용하도록 선택하거나 다른 플레이어를 생성하여 웹 사이트의 다른 위치에 대해 다른 모양과 설정을 캡처하도록 선택할 수 있습니다.

출판 개요

플레이어 관리 시스템의 기능 중 하나는 테스트하려는 변경 사항을 공개적으로 게시된 플레이어 버전과 분리하는 기능입니다. UI와 API 모두에서 플레이어의 구성을 변경하면 플레이어의 미리 보기 버전을 통해 변경 사항을 즉시 사용할 수 있습니다. 이러한 미리 보기 버전은 완전히 사용 가능하고 테스트할 수 있습니다. 승인을 위해 미리 보기 URL을 공유하거나 변경 사항에 만족할 때까지 플레이어 미리 보기에서 작업할 수 있습니다. 모든 것은 플레이어를 게시하여 공개적으로 사용할 수 있도록 하기 전에입니다.

폭포 업데이트

플레이어를 게시하면 하위 플레이어에게 영향을 줄 수 있는 결과가 있습니다. 게시에 대한 세부정보는 다음과 같습니다.

  • 플레이어 게시는 해당 플레이어에서 생성된 모든 하위 플레이어 인스턴스의 다시 게시를 트리거합니다.
  • 하위 플레이어 게시는 해당 하위 플레이어에만 영향을 줍니다.
  • Brightcove 기본 템플릿 업데이트(Brightcove 제어)는 모든 플레이어의 다시 게시를 트리거하고 플레이어 다시 게시는 차례로 각 플레이어에서 생성된 모든 하위 플레이어의 다시 게시를 트리거합니다.

설정은 어린이 승리 방식으로 적용됩니다. 즉, 하위 수준의 설정이 상위 수준의 변경 사항보다 우선합니다. 다음을 고려하세요:

  • 기본 템플릿은 CSS 값을 빨간색으로 정의합니다.
  • 플레이어는 동일한 CSS 값을 흰색으로 정의합니다.
  • 자식 플레이어는 동일한 CSS 값을 파란색으로 정의합니다.

아이가 이기면 결과는 다음과 같습니다.

  • 플레이어의 CSS 값은 흰색입니다.
  • 자식 플레이어의 CSS 값은 파란색입니다.

속성의 데이터 유형이 배열인 경우 이 상속 동작을 변경할 수 있습니다. NS 배열 필드 Embed API 가이드의 섹션에서 자세한 내용을 설명합니다.

속도 제한

Player Management API에는 활성화된 고유 IP별로 속도 제한이 있습니다. 다음은 속도 제한입니다.

요청 유형 한계
게시 및 생성(플레이어 및 포함) 60개 요청/분
읽기(GET) 600개 요청/분
쓰기(POST, PUT, PATCH 및 DELETE) 300개 요청/분
플레이어 미리보기 100개 요청/분
모든 속도 제한은 해당 범주의 각 요청에 대해 계산됩니다. 예를 들어, 1분 이내에 200개의 POST와 200개의 PUT을 시도하면 쓰기 요청 제한인 300개 요청/분을 초과합니다.

제한을 초과하는 모든 요청은 상태 코드를 반환합니다. 429및 응답 본문{"error": "Rate limit exceeded"} .

응답 상태 코드

응답 및 오류 코드/메시지는 다음을 기반으로 합니다. W3에 의해 정의 . 가장 일반적으로 볼 수 있는 코드는 다음과 같습니다.

  • 200요청 성공
  • 201생성됨(플레이어, 구성)
  • 400잘못된 요청 - API 호출의 구문이 올바르지 않을 수 있습니다.
  • 401유효하지 않은 인증 - 암호가 올바르게 입력되었는지 또는 다음을 따랐는지 확인하십시오. OAuth 지침바르게
  • 404찾을 수 없음 - 리소스가 존재하고 API 호출에 사용된 URL이 올바른지 확인하십시오.
  • 429속도 제한 초과
  • 500내부 서버 오류 - 요청을 이행하는 동안 오류가 발생했습니다.
