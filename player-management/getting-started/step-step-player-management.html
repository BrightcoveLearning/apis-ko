소개

이 단계별 문서에서는 다음 작업을 수행합니다.

  • 플레이어 만들기
  • 비디오를 자동 재생하도록 플레이어 업데이트
  • 플러그인을 추가하여 플레이어 사용자 지정
  • 플레이어에 대한 구성 표시
이 문서에 사용된 API에 대한 자세한 내용은플레이어 관리 API 개요문서와플레이어 관리 API참조.
문서의 단계에는 Mac 및 Linux 시스템에 고유하지만 Windows 운영 체제에는 포함되지 않는 curl 및 기타 명령줄 도구가 포함됩니다. Windows를 사용하는 경우 다음을 설치하여 이러한 도구를 얻을 수 있습니다. 시그윈 . 설치하는 동안 설치할 패키지를 묻는 메시지가 표시됩니다. 열기그물패키지를 확인한 다음곱슬 곱슬하다옵션을 선택하고 설치를 마칩니다. 이 작업을 완료하면 단계별로 수행할 수 있습니다. 플레이어 관리 문서가 cygwin 명령줄을 사용하여 작성된 대로 성공적으로 작성되었습니다. 도구를 사용하여 일반 Windows 명령줄에서 curl을 실행할 수 있지만 이 문서에서는 자세히 설명하지 않습니다.

시작하다

이 단계별 문서에서 취하는 접근 방식은 curl 문을 사용하여 Player Service API와 통신하는 것입니다. curl 도구는 명령줄에서 URL 구문으로 데이터를 전송하는 데 사용됩니다. curl에 대한 자세한 내용은 다음에서 얻을 수 있습니다. http://curl.haxx.se .

단순함을 위해 이 단계별 문서에서와 같이 curl을 사용할 필요는 없습니다.' 때문. 물론 선호하는 언어를 사용하여 API와 통신할 수 있습니다. 체크 아웃플레이어 관리 샘플 설정기본 인증, AJAX 및 JavaScript를 사용하여 플레이어 생성, 모든 플레이어 표시, 플레이어 삭제 등과 같은 기본 작업을 위한 일부 미니 앱을 작성하는 방법을 보여줍니다.

API 사용을 시작하기 전에 몇 가지 예비 단계를 수행해야 합니다. 이것들은:

  • 스튜디오에 로그인합니다. 계정이 여러 개인 경우 드롭다운을 사용하여 새 플레이어를 만들 계정을 선택합니다. 자격 증명 시스템이 제대로 작동하려면 이 계정에 대한 관리자 권한이 있어야 합니다. 사용자에게 관리자 권한이 있는지 확실하지 않은 경우 사용자 설정 페이지로 이동하여 역할과 함께 나열된 사용자를 확인하십시오.
  • Studio에서 다음을 선택했는지 확인하십시오. 집모듈 목록에서. 페이지의 왼쪽 상단 모서리에서 계정 이름 바로 아래에 있는 계정 이름을 복사할 수 있습니다. 계정 ID .
  • 명령줄에서 다음을 입력하여 계정 ID 값을 환경 변수에 할당합니다. 
            export ACCOUNT_ID=YourAccountID
  • 인증은 curl 문의 일부로 계정 이메일 주소를 제공하여 처리되며 API는 비밀번호를 묻는 메시지를 표시합니다. 이메일 주소는 자주 사용되므로 해당 값을 환경 변수에도 할당합니다. 
            export EMAIL=YourEmailAddress
환경 변수가 올바르게 할당되었는지 확인하려면 다음을 사용하여 표시할 수 있습니다. echo 명령, 예: echo {account_id} .

이제 API를 사용할 준비가 되었습니다.

플레이어 만들기

이제 플레이어 관리 API를 호출하여 플레이어를 생성할 수 있습니다. 이 API 호출은플레이어 관리 API 개요문서. 다음 curl 문을 복사하여 명령줄에 붙여넣는 것이 좋습니다.

다음 단계는 플레이어를 만드는 데 도움이 됩니다.

메모: curl 문과 명령줄을 사용하는 것이 익숙하지 않다면 많은 REST 클라이언트 앱이 있습니다. Postman이라는 REST 클라이언트 Chrome 브라우저 플러그인 사용에 대한 정보는 HTTP 요청에 Postman 사용문서.
  1. 첫 번째 curl 문은 플레이어를 만들고 이름과 설명을 할당합니다. 이 명령문을 명령줄에 붙여넣고 키를 누른 후 입력하다, 암호를 입력하라는 메시지가 표시됩니다. 
            curl \
          --header "Content-Type: application/json" \
          --user $EMAIL \
          --request POST \
          --data '{
            "name": "My New Player Name v2",
            "description": "My new player description"
            }' \
          https://players.api.brightcove.com/v1/accounts/{account_id}/players

    서버가 JSON 형식으로 응답 데이터를 반환하는 것을 볼 수 있습니다.

            {
          "id": "S1qN4xeG7",
          "url": "http://players.brightcove.net/1507807800001/S1qN4xeG7_default/index.html",
          "embed_code": "<iframe src='//players.brightcove.net/1507807800001/S1qN4xeG7_default/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>",
          "embed_in_page": "http://players.brightcove.net/1507807800001/S1qN4xeG7_default/in_page.embed",
          "preview_url": "http://preview-players.brightcove.net/v2/accounts/1507807800001/players/S1qN4xeG7/preview/embeds/default/master/index.html",
          "preview_embed_in_page": "http://preview-players.brightcove.net/v2/accounts/1507807800001/players/S1qN4xeG7/preview/embeds/default/master/in_page.embed",
          "preview_embed_code": "<iframe src='//preview-players.brightcove.net/v2/accounts/1507807800001/players/S1qN4xeG7/preview/embeds/default/master/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>"
        }
    이때preview_url 그리고preview_embed_code 논의되지 않을 것입니다. 이 문서의 뒷부분에서 미리 보기와 게시된 플레이어 간의 차이점에 대해 설명합니다.
  2. 다음 값 중 하나(또는 모두)를 사용하여 플레이어를 봅니다(이 시점에서 플레이어에 비디오가 없지만 Studio를 사용하여 새로 만든 플레이어를 사용하여 비디오를 게시할 수 있음).
    • 복사url 플레이어를 보려면 브라우저로 이동하십시오.
    • 복사embed_code iframe에서 플레이어를 보려면 HTML 페이지로 이동합니다. 이것이 제대로 작동하려면 iframe이 포함된 페이지가 실제 HTTP 서버에서 제공되어야 합니다.
    • NS embed_in_page 기능의 사용은고급(인페이지) 포함 코드에 사용 가능한 옵션문서.
    동일한 HTML 페이지에서 여러 플레이어를 사용할 수 있습니다.

비디오 클라우드 비디오

물론 Video Cloud 라이브러리의 비디오를 사용하여 플레이어를 만들 수 있습니다. 를 사용하는 대신media 섹션에서 JSON 데이터를 사용하면video_cloud 부분. 아래의 curl 문은 Video Cloud 비디오의 ID를 사용한 플레이어 생성을 보여줍니다.

        curl \
          --header "Content-Type: application/json" \
          --user $EMAIL \
          --request POST \
          --data '{
            "name": "Video Cloud CURL 10 March",
            "configuration": {
              "video_cloud": {
                "video": "4093372393001"
              }
            }
          }' \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players
ID를 사용하는 대신 다음 구문과 함께 동영상의 참조 ID를 사용할 수 있습니다. "video": "ref:1234refid" .

응답은 다음을 사용하여 플레이어를 생성할 때의 응답과 논리적으로 동일합니다. 미디어/소스/src위에 표시된 대로.

        {
        "id": "HJyMlHiuZ",
        "url": "http://players.brightcove.net/1507807800001/HJyMlHiuZ_default/index.html",
        "embed_code": "<iframe src="//players.brightcove.net/1507807800001/HJyMlHiuZ_default/index.html" allowfullscreen="" webkitallowfullscreen="" mozallowfullscreen=""></iframe>",
        "embed_in_page": "http://players.brightcove.net/1507807800001/HJyMlHiuZ_default/in_page.embed",
        "preview_url": "http://preview-players.brightcove.net/v1/accounts/1507807800001/players/HJyMlHiuZ/preview/embeds/default/master/index.html",
        "preview_embed_code": "<iframe src="//preview-players.brightcove.net/v1/accounts/1507807800001/players/HJyMlHiuZ/preview/embeds/default/master/index.html" allowfullscreen="" webkitallowfullscreen="" mozallowfullscreen=""></iframe>"
        }

반환된 URL 값을 검색하여 플레이어의 구성을 확인하고 변경할 수 있습니다. index.html에게 config.json . 위의 curl 문으로 생성된 플레이어의 경우 다음이 표시됩니다.

        {

NS policy_key 플레이어의 구성에 자동으로 추가됩니다. 이것은 Policy API를 사용하여 생성되며 다른 비디오에 액세스하기 위해 플레이어에 특별한 제한을 가할 수 있습니다. 즉, 정책 키는 언제 볼 수 있는 비디오를 제어합니다.

정책 키를 생성하는 대신 구성에 자체 정책 키를 추가할 수도 있습니다. 플레이어 및 플레이어 하위 구성 상호 작용에 대한 일반 규칙이 여기에 적용됩니다. 플레이어 하위 수준에서 설정된 정책 키는 플레이어 수준에서 설정된 모든 정책 키보다 우선합니다.

플레이어 업데이트

이제 플레이어 생성을 위한 기본을 수행했습니다. 다음으로 플레이어의 간단한 업데이트를 수행하는 방법을 배웁니다. 이 경우 브라우저에서 허용하는 경우 플레이어에서 비디오를 자동 재생하도록 플레이어를 설정합니다.

  1. 플레이어 생성에서 반환된 JSON에서id 값이 표시되었습니다. 이것을 복사PLAYER_ID 환경 변수. 
            export PLAYER_ID=YourPlayerID
  2. 플레이어를 업데이트하려면 HTTP를 사용합니다. PATCH 방법. 플레이어를 업데이트하기 위해 데이터를 보내드립니다. 당신은 설정합니다autoplay옵션true . 다음 curl 문을 복사하여 붙여넣고 실행합니다. 
            curl \
          --header "Content-Type: application/json" \
          --user $EMAIL \
          --request PATCH \
          --data '{
            "autoplay": true
          }' \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration
  3. 서버가 다음을 포함하는 데이터를 반환하는지 확인하십시오. id , preview_url 그리고preview_embed_code 다음 JSON 형식의 값: 
            {
          "id": "S1qN4xeG7",
          "preview_url": "http://preview-players.brightcove.net/v2/accounts/1507807800001/players/S1qN4xeG7/preview/embeds/default/master/index.html",
          "preview_embed_in_page": "http://preview-players.brightcove.net/v2/accounts/1507807800001/players/S1qN4xeG7/preview/embeds/default/master/in_page.embed",
          "preview_embed_code": "<iframe src='//preview-players.brightcove.net/v2/accounts/1507807800001/players/S1qN4xeG7/preview/embeds/default/master/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>"
        }
    업데이트 후 플레이어의 미리 보기 버전이 있습니다. 변경 사항은 업데이트 이전에 작업했던 플레이어의 게시된 버전에 반영되지 않습니다.
  4. 다음 중 하나를 사용하십시오. preview_url 또는preview_embed_code 값을 확인하여 플레이어 구성의 변경 사항을 확인합니다. 이제 자동 재생이 사실임을 알 수 있습니다.
  5. 이제 플레이어를 게시하여 사용자가 볼 수 있는 올바른 위치로 푸시합니다. 다음 명령을 입력합니다. 
            curl \
        --header "Content-Type: application/json" \
        --user $EMAIL \
        --request POST \
        https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/publish

    플레이어가 게시되면 미리 보기 버전과 다른 플레이어가 표시됩니다. 게시된 버전은 다음과 같은 점에서 미리 보기 버전과 다릅니다.

    • JavaScript와 CSS는 축소, 연결 및 플레이어에 직접 인라인됩니다.
    • 포스터 이미지의 저해상도 버전이 생성되어 페이지에 인라인되어 요청 설정 시간이 긴 네트워크(즉, 셀룰러 데이터 네트워크)에서 인지된 로드 시간을 개선합니다.
    • 업데이트가 적용된 후 문제가 발견되면 플레이어를 복구할 수 있도록 이전 버전의 플레이어가 저장됩니다.
    메모: 플레이어 생성 시(위의 1단계) 플레이어도 자동으로 게시됩니다. 이것이 플레이어 생성 시 JSON 응답에 게시된 정보와 미리 보기 정보가 모두 포함된 이유입니다.
  6. 플레이어 게시의 JSON 응답이 다음과 같이 나타나는지 확인합니다. 
            {
        "id": "S1qN4xeG7",
        "url": "http://players.brightcove.net/1507807800001/S1qN4xeG7_default/index.html",
        "embed_code": "<iframe src='//players.brightcove.net/1507807800001/S1qN4xeG7_default/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>",
        "embed_in_page": "http://players.brightcove.net/1507807800001/S1qN4xeG7_default/in_page.embed"
        }
  7. 플레이어 구현 중 하나를 사용하여 게시된 플레이어가 올바르게 작동하는지 확인하십시오.

플레이어 사용자 지정

이 섹션에서는 Studio 또는 Player Management API를 통해 비디오를 플레이어와 연결했다고 가정합니다. 접근 방식에 따라 결과 플레이어 구성이 표시된 것과 다를 수 있습니다.

플러그인을 사용하여 플레이어를 사용자 정의할 수 있습니다. 플레이어에 기존 플러그인을 포함하거나 플러그인을 빌드하는 방법을 배울 수 있습니다. 단계별: 플러그인 개발 .

이 경우 매우 간단한 기존 플러그인을 사용하여 비디오에 문자 메시지를 오버레이합니다. 다음은 HTML 단락 태그를 생성하는 플러그인의 코드입니다. 참고: 클래스 이름과 텍스트가 단락에 할당된 다음 기존 플레이어에 추가됩니다. 이 플러그인은 사용자의 편의를 위해 이미 존재하며 다음 위치에 있습니다. //solutions.brightcove.com/bcls/video-js/new-player/first-plugin.js .

        videojs.registerPlugin('firstPlugin', function() {
          var player = this,
          overlay = document.createElement('p');
          overlay.className = 'vjs-overlay';
          overlay.innerHTML = "First Plugin Working!";
          player.el().appendChild(overlay);
        });

이 특정 경우에 검정색 배경에 검정색 텍스트가 표시되지 않도록 단락의 스타일을 지정해야 합니다. 다음은 단락의 클래스 이름에 적용되는 스타일입니다. 이 스타일시트는 사용자의 편의를 위해 이미 존재하며 다음 위치에 있습니다. //solutions.brightcove.com/bcls/video-js/new-player/first-plugin.css .

        .vjs-overlay {
          background-color: #333333;
          color: white;
          font-size: 2em;
          padding: 5px;
          position: absolute;
          top: 100px;
          left: 20px;
          width: 150px;
        }

  • 플러그인 코드가 포함된 JavaScript 파일의 위치
  • 스타일시트의 위치
  • 플레이어와 함께 사용할 플러그인 이름
  1. 플러그인이 작동하도록 하려면 다음 curl 문을 사용하십시오.
    메모: 옵션을 제공하지 않더라도 여기에 표시된 대로 플러그인 이름을 입력해야 합니다.
            curl\
        --header "Content-Type: application/json"\
        --user $EMAIL\
        --request PATCH\
        --data '{
          "scripts": [
            "//solutions.brightcove.com/bcls/video-js/new-player/first-plugin.js"
           ],
           "stylesheets": [
            "//solutions.brightcove.com/bcls/video-js/new-player/first-plugin.css"
           ],
           "plugins": [{
             "name": "firstPlugin"
           }]
          }'\
        https: //players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration
  2. 다음 curl 문을 사용하여 업데이트된 플레이어를 게시합니다. 이것은 이전 섹션에서 플레이어를 게시하는 데 사용한 것과 동일한 게시 코드입니다. 
            curl \
          --header "Content-Type: application/json" \
          --user $EMAIL \
          --request POST \
          https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/publish
  3. 반환된 URL을 복사하고 찾아보십시오. 플러그인이 작동하고 플러그인 텍스트가 표시되는 것을 볼 수 있습니다.
    플레이어가 게시되는 데 약간의 시간이 걸릴 수 있으므로 플러그인이 올바르게 작동하는 것을 보기 전에 약간의 지연이 있을 수 있습니다. 플러그인이 표시되지 않으면 주기적으로 페이지를 새로고침하세요.

플레이어 관리 API를 사용하여 플러그인을 플레이어에 추가하면 Brightcove에서 제공한 플러그인이든 사용자가 빌드한 사용자 정의 플러그인이든 상관없이 플러그인은 플레이어 코드 자체의 일부가 됩니다.

자체 플러그인을 개발하고 소스 코드를 변경하는 경우, 그렇게 하지 않는 한 새 플러그인 코드는 JavaScript 파일의 일부가 아니므로 플레이어를 다시 게시해야 합니다.
