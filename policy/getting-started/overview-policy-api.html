소개

NS 정책 키다음을 사용하여 Video Cloud 라이브러리에서 비디오 및 재생 목록 콘텐츠를 검색하는 데 사용됩니다. 재생 API .

또한 참조 API 참조 .

하나 필요하세요?

고유한 정책 키를 생성해야 하는 특별한 경우만 있습니다.

  • 정책 키는 Brightcove Player를 생성할 때 자동으로 생성됩니다. 플레이어에 대한 지역 필터링이 포함된 키를 가져오거나 플레이어를 생성하지 않고 Playback API에 액세스하려는 경우 이 API를 사용하여 직접 정책 키를 생성하기만 하면 됩니다. 일반적으로 이는기기 SDK . 보다정책 키모든 Brightcove Player에 대한 정책 키를 얻는 방법에 대한 지침을 참조하십시오.
  • 비디오 검색 요청 수행:
    • 다음을 사용해야 합니다. 검색 가능 정책 키 .
    • 새 Brightcove Player에 대해 생성된 정책 키는 기본적으로 검색이 활성화되어 있지 않습니다. 다음을 사용하여 수동으로 설정해야 합니다. 정책 API .
    • 일반적으로 검색 가능한 정책 키는 재생 가능한 모든 비디오를 나열하는 데 사용할 수 있으므로 브라우저 플레이어나 모바일 앱이 아닌 서버에만 저장해야 합니다. 일부 계정의 경우 재생 가능한 모든 동영상이 검색될 수 있다는 사실이 마음에 들지 않으면 적용되지 않을 수 있습니다.
    • 검색 요청은 재생 가능한 비디오만 반환합니다. API 요청/응답에 대한 자세한 내용은비디오 가져오기섹션재생 API 참조 .

자주하는 질문

다음은 일반적으로 받는 Policy API에 대한 몇 가지 질문입니다.

여러 계정에 대해 정책 키를 만들 수 있습니까? 아니면 하나만 만들 수 있습니까?
정책 키를 사용하면 한 계정의 동영상과 재생목록에 액세스할 수 있습니다. 계정이 여러 개인 경우 각 계정에 대한 정책 키를 생성해야 합니다.
정책 키가 만료됩니까?
아니요, 정책 키는 영원히 유효합니다. 정책 키를 취소해야 하는 경우 다음 담당자에게 문의해야 합니다. 브라이트코브 지원 - 요청에 취소하려는 계정 ID와 정책 키를 포함합니다.
정책 키가 계정의 모든 사용자에게 작동합니까?
예, 정책 키가 있는 사람은 누구나 사용할 수 있습니다.
정책 키를 업데이트할 수 있습니까?
아니요, 정책 키를 업데이트할 방법이 없습니다. 다음 중 하나를 수행할 수 있습니다. 정책 키 가져오기 Video Cloud 계정의 기존 플레이어에서 가져오거나 Policy API를 사용하여 새 정책 키를 만들 수 있습니다.

기본 URL

정책 API의 기본 URL은 다음과 같습니다.

    https://policy.api.brightcove.com/v1

계정 경로

모든 경우에 특정 Video Cloud 계정에 대한 요청이 이루어집니다. 그래서, 당신은 항상 추가합니다/accounts/ 기본 URL에 대한 계정 ID:

    https://policy.api.brightcove.com/v1/accounts/{account_id}

입증

요청은 전달된 액세스 토큰에 의해 인증됩니다. Authorization 헤더:

    Authorization: Bearer {access_token}

액세스 토큰을 얻으려면 다음 단계를 따르세요.

  1. 클라이언트 자격 증명 얻기( client_id 그리고client_secret ) 을 통해비디오 클라우드 스튜디오 (권장).

    정책 키 생성 요청과 함께 사용할 수 있는 액세스 토큰의 경우 클라이언트 자격 증명에 Studio에서 다음 권한이 있어야 합니다. 플레이어 읽기 및 읽기/쓰기

    다음을 통해 직접 클라이언트 자격 증명을 얻을 수도 있습니다. 브라이트코브 OAuth API .

  2. 클라이언트 자격 증명이 있으면 다음을 수행할 수 있습니다. OAuth API에서 임시 액세스 토큰 얻기 .

필수 헤더

여기에 덧붙여Authorization이전 섹션에서 논의한 헤더와 마찬가지로 요청에는Content-Type헤더:

    Content-Type: application/json

정책 키 생성

새 정책 키를 생성하려면 다음을 위해 POST 요청을 수행합니다.

    https://policy.api.brightcove.com/v1/accounts/{account_id}/policy_keys

요청 본문

요청 본문에 정책에 대한 JSON을 포함합니다.

    {
      "key-data": {
        "account-id": "{account_id}"
      }
    }

이것은 계정에 대한 액세스를 제한하는 가장 기본적인 정책이며 생성하는 모든 정책에 이를 포함해야 합니다. 여기서 계정 ID는 요청 URL의 계정 ID와 일치해야 하며 그렇지 않으면 오류가 반환됩니다.

도메인 제한

계정에 대한 액세스를 제한하는 것 외에도 다음을 추가하여 특정 도메인에 대한 액세스를 제한할 수도 있습니다. allowed-domains 열쇠:

    {
      "key-data": {
        "account-id": "123456789001",
        "allowed-domains": [
          "http://www.abc.com",
          "https://www.abc.com",
          "http://www.xyz.com",
          "https://www.xyz.com"
        ]
      }
    }
  • 도메인은 프로토콜의 기본 포트가 아닌 경우(예: http://www.abc.com, https://www.def.com:8888) 프로토콜과 도메인 이름, 포트를 포함해야 합니다. 경로 또는 후행 슬래시("/")가 아님
  • 계정의 모든 키는 Playback API를 통해 계정 콘텐츠에 액세스하는 데 사용될 수 있으므로 도메인 제한을 사용하는 경우 모든 플레이어를 도메인으로 제한하는 것이 좋습니다.
  • 왜냐하면브라이트코브 네이티브 플레이어 SDK오리진 헤더를 보내지 마십시오. 도메인 제한 정책 키를 사용할 수 없습니다. 이것을 시도하면 얻을 것이다403 Forbidden Video Cloud 라이브러리에서 콘텐츠를 가져오려고 할 때 응답합니다.

지리적 제한

정책 API를 사용하여 플레이어 대신 또는 추가로 플레이어에 대한 지리 필터링을 구현할 수 있습니다. 개별 동영상.

  • 정책에 따라 시청자가 감지한 국가에 따라 재생이 제한될 수 있습니다. 이 정책으로 키를 만들고 사용하려면 계정에서 지역 필터링 기능을 사용할 수 있어야 합니다. 이 정책은 최종 사용자가 익명 프록시를 통해 또는 익명 프록시를 통해 요청하는 것처럼 보이는 경우에도 액세스를 거부합니다.
  • 정책 키에 적용된 지역 제한이 적용됩니다. 오직해당 정책 키를 사용하는 플레이어에 의해
  • 지리적 제한도 가능개별 동영상에 적용 . Video Cloud는 플레이어 및 비디오 수준 지리 필터링 정책을 모두 적용하므로 어느 한 국가에서 제외되는 경우 해당 국가에서 비디오를 사용할 수 없습니다.

견본

다음은 미국 및 미국 영토 및 군사 기지에서만 재생을 허용하는 화이트리스트 정책 세트의 예입니다.

    {"account-id": "8523232323",
      "geo": {
        "countries": ["us", "usmil", "pr", "gu", "vi", "as", "mp"],
        "exclude_countries": false
      }
    }

대신 해당 국가를 블랙리스트에 추가하려면 다음을 설정하십시오. exclude_countries 사실로.

SSAI 필요

SSAI(서버 측 광고 삽입)로 작업할 때 API 요청에 적절한 광고 구성을 적용하는 방법이 필요할 수 있습니다. 이를 요구하도록 정책 키를 구성하여 이를 수행할 수 있습니다. ad_config_id 모든 Playback API 요청에 URL 매개변수로 추가됩니다.

견본

다음은 예입니다.

    {
      "key-data": {
        "account-id": "1752604059001",
        "require-ad-config": true
      }
    }

사용에 대한 자세한 내용은ad_config_id Playback API 요청과 함께재생 API 개요: SSAI에 영상 요청문서.

동영상 검색

비디오 검색 요청을 수행할 때 검색이 가능한 정책 키를 사용해야 합니다. 검색 가능 정책 키를 생성하려면 호출의 요청 본문에 다음이 포함되어야 합니다.

    {
      "key-data": {
        "account-id": "57838016001",
        "apis": ["search"]
      }
    }
일반적으로 검색 가능한 정책 키는 재생 가능한 모든 비디오를 나열하는 데 사용할 수 있으므로 브라우저 플레이어나 모바일 앱이 아닌 서버에만 저장해야 합니다. 일부 계정의 경우 재생 가능한 모든 비디오가 검색될 수 있는지 여부에 상관하지 않는 경우 적용되지 않을 수 있습니다.

API 응답

정책 생성 요청에 대한 응답은 암호화된key-string Playback API에 대한 요청과 키와 관련된 정책을 인증하는 데 사용할 수 있습니다.

    {
      "key-data": {
        "account-id": "57838016001"
      },
      "key-string": "BCpkADawqM0NK0Rq8n6sEQyWykemrqeSmIQqqVt3XBrdpl8TYlvqN3hwKphBJRnkPgx6WAbozCW_VgTOBCNf1AQRh8KnmXSXfveQalRc5-pyNlSod5XzP99If2U"
    }

정책 요청

연결된 정책 개체를 검색하려면key-string , GET 요청:

    /accounts/v1/{account_id}/policy_keys/{key_string}

응답은 키 문자열 및 정책의 JSON 표현입니다.

    {
      "key-data": {
        "account-id": "57838016001"
      },
      "key-string": "BCpkADawqM0NK0Rq8n6sEQyWykemrqeSmIQqqVt3XBrdpl8TYlvqN3hwKphBJRnkPgx6WAbozCW_VgTOBCNf1AQRh8KnmXSXfveQalRc5-pyNlSod5XzP99If2U"
    }

정책 취소

기존 정책을 취소해야 하는 경우 요청을 제출하십시오. 브라이트코브 지원 . 요청에 취소하려는 정책의 키 문자열을 포함합니다.
