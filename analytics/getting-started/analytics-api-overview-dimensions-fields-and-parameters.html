매개변수

아래는 사용 가능한 매개변수를 요약한 표입니다. 애널리틱스 API . 매개변수의 사용은 다음 섹션에서 더 자세히 설명합니다.

매개변수 필수의 설명 가치 기본

계정

보고서를 원하는 Video Cloud 계정은 다음을 사용하여 지정됩니다. accounts매개변수. 예를 들면 다음과 같습니다.

  https://analytics.api.brightcove.com/v1/data?accounts={account1_id,account2_id}

필터 위치

필터의 일반 구문은 다음과 같습니다.

where=dimension1==value1;dimension2==value2

예를 들면 다음과 같습니다.

https://analytics.api.brightcove.com/v1?accounts=account_id(s)&dimensions=device_type&where=video==video_id;device_type==tablet

쉼표는 논리적 OR로 처리되고 세미콜론은 논리적 AND로 처리됩니다. 예를 들어, where=video==1234,5678;player==9876 "비디오 = 1234"로 해석됩니다. 또는 5678 그리고플레이어 = 9876"

공백 및 특수 문자

문자열 값은 URI로 인코딩되어야 합니다. ""를 사용하여 특수 문자를 이스케이프할 수도 있습니다.

where=search_terms==boston,%20ma

모든 차원을 필터로 사용할 수 있습니다. 하지만 오직 해당 차원이dimensions당신은 요청하고 있습니다.

플레이어를 필터링할 때 다음 사항에 유의하십시오.

  • 브라이트코브 플레이어 - 다음 형식으로 플레이어 URL을 입력합니다. players.brightcove.com/account_id/player_id _default
  • 사용한 경우애플리케이션 ID플레이어를 식별하려면 다음 형식으로 플레이어 URL을 입력하십시오. players.brightcove.com/account_id/player_id_default/ application-id - 예를 들어: players.brightcove.com/1752604059001/BJFYWrtUl_default/AlltimeViewsPlayerSample
  • 스마트 플레이어 - 플레이어 ID 입력

예

다음은 몇 가지 예입니다. where태그 및 사용자 정의 필드 검색을 위한 필터:

참고로video.q검색 기능에는 다음이 포함됩니다. AND , OR그리고NOT다음과 같은 논리:

  • NS +검색어 앞의 (더하기) 기호는 결과를 의미합니다. ~해야하다이 용어를 포함합니다.
  • NS -검색어 앞의 (빼기) 기호는 결과를 의미합니다. 해서는 안 된다이 용어를 포함합니다.
  • 더하기 또는 빼기 기호가 없으면 결과에 이 용어가 포함되거나 포함되지 않을 수 있습니다.

필터 및 허용 가능한 값 요약

다음 표는 필터로 사용되는 각 차원에 대해 허용되는 값을 보여줍니다.

차원 필터 허용되는 값

기간

에 지정된 날짜 범위from그리고to모든 유형의 보고서에 대한 매개변수는 다양한 형식으로 표시될 수 있습니다.

  • 텍스트 값:
    • to=now (사용 가능 및 모든 요청에 대한 기본값)
  • 밀리초 단위의 Epoch 시간 값(예: 1377047323000
  • ISO 8601 표준 국제 날짜 형식으로 표시된 날짜: YYYY-MM-DD와 같은 형식2013-09-12 . 이 형식으로 표시된 날짜의 경우:
    • 지정된 모든 날짜 범위가 해석됩니다. 계정에 설정된 시간대
    • 날짜 제공 시간은 자정으로 해석됩니다( 00:00:00 ) 지정된 날짜에 계정에 설정된 시간대
  • 상대 날짜: 다음 중 하나를 표현할 수 있습니다. to그리고from d(일) 또는 h(시간)의 다른 값과 관련된 값입니다. 예를 들면 다음과 같습니다.
    • from=2015-01-01&to=31d
    • from=-48h&to=now
    • from=-2d&to=now (이전 예제와 동일한 결과를 제공합니다)
    • from=-365d&to=2014-12-31

    음수(-2d)는 "before"(다른 값)로 해석되고 양수(48h)는 "from"(다른 값)으로 처리됩니다.

하루 동안 "동영상"과 같은 일부 측정기준에 대한 보고서를 생성하려면 시작 및 시작 값을 해당 날짜로 설정합니다.

...&dimensions=video&from=2013-11-01&to=2013-11-01

제한 및 오프셋

NS limit반환할 항목의 수입니다(기본값: 10). 모든 항목을 반환하려면 다음을 사용하십시오. limit=all . offset건너뛸 항목의 수입니다(기본값: 0). 당신이 사용할 수있는limit그리고offset결과를 살펴보는 앱을 함께 만들 수 있습니다.

조정된 데이터

NS reconciled매개변수는 부울입니다. 로 설정된 경우true , 결과는 조정된 데이터로 제한됩니다. 만약에false , 결과는 실시간(시간별 미조정) 데이터로 제한됩니다.

실시간 데이터는 지난 32일 동안만 사용할 수 있습니다. 설정하면reconciled=false그리고from 32일 전보다 이전 날짜인 경우 해당 시점에 조정되지 않은 데이터가 없기 때문에 API는 데이터를 반환하지 않습니다.

지리적 분석을 위한 차원

  • country - ISO-3611-1 국가 코드로. 예: '우리를'
  • region - ISO-3611-2 지역 코드로. 예: 'US-WA'

필드 및 정렬

사용fields반환할 필드를 지정하는 매개변수입니다. 기본적으로, video_view보고하는 차원에 해당하는 필드(예: destination_domain )가 반환됩니다. 보다차원 및 필드자세한 사항은.

사용sort반환된 항목을 정렬하는 데 사용되는 메트릭 필드를 지정하는 매개변수. 예를 들어: sort=video_view . 정렬 필드를 무효화하여 정렬 순서를 바꿀 수 있습니다. sort= -video_view

반환하거나 정렬할 수 있는 필드는측정항목보고 중인 차원에 대해 반환됩니다. 이러한 매개변수는 엄격하게 검증되며 다음과 같은 경우 요청에서 오류를 반환합니다. fields또는sort매개변수는 요청에 유효하지 않은 필드를 가리킵니다.

비디오 측정기준 보고서의 경우 유효한 정렬 값에는 비디오 메타데이터 필드가 포함됩니다. video.name , video.reference_id , 그리고video.tags .

video_name 대 video.name

  • video_name분석 데이터에 기록된 동영상 이름입니다.
  • video.name비디오 메타데이터의 비디오 이름입니다.

대부분의 경우 두 값이 동일하지만 그렇지 않은 경우도 있습니다.

  • 경우에 따라 다음 값이 표시될 수 있습니다. video.name , 하지만 위해video_name - 이 경우 어떤 이유로 동영상 이름이 분석 데이터 수집기로 전송되지 않았습니다.
  • 경우에 따라 값이 다를 수 있습니다. 이 경우 비디오의 이름이 어느 시점에서 변경되었습니다. video.name Video Cloud 미디어 라이브러리에 있는 비디오의 현재 이름입니다. video_name데이터가 분석 데이터 수집기로 전송되었을 때 비디오의 비디오 이름이 됩니다(이름이 변경된 기간을 쿼리하는 경우 API에서 반환된 값은 대부분의 조회수와 연결된 이름이 됩니다. 기간 동안).

계산된 필드

다음 구문을 사용하여 API 요청에 계산된 필드를 추가할 수 있습니다.

fields=calulated_field_name:expression

계산된 필드를 사용하여 기존 메트릭에서 고유한 사용자 지정 필드를 만들거나 기존 필드의 이름을 바꿀 수 있습니다.

할 수 있습니다. ~ 아니다다른 계산된 필드에 대한 표현식의 일부로 계산된 필드를 사용합니다.

계산된 필드의 이름은 모든 URI 호환 문자열이 될 수 있습니다. 표현식에는 일반 필드 이름과 다음과 같은 산술 연산자가 포함될 수 있습니다.

  • + (덧셈)
  • - (빼기)
  • * (곱셈)
  • / (분할)
  • ^ (멱지수)
  • () (괄호)

예

fields=avg_seconds_viewed:video_seconds_viewed/video_view,video.name
fields=avg_incomplete_ads:(ad_mode_begin-ad_mode_complete)/video_view,video.name
fields=Video%20Views:video_view,video.name
