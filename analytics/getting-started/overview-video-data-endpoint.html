소개

비디오 데이터 끝점은 특정 비디오에 대한 특정 데이터를 빠르게 가져올 수 있는 방법을 제공합니다. 현재 요청은 동영상의 전체 조회수를 반환합니다. 앞으로 추가 데이터가 반환될 수 있습니다. 이 끝점의 주요 사용 사례는 트래픽이 많은 페이지의 플레이어에게 전체 시간 조회수 정보를 전달하는 것입니다. Analytics API너무 느릴 것입니다.

끝점

비디오 데이터 요청의 끝점은 다음과 같습니다.

    https://analytics.api.brightcove.com/v1/alltime/accounts/{account_id}/videos/{video_id}

모든 요청은 GET입니다.

입증

모든 요청은Analytics API Brightcove의 OAuth 서비스를 통해 인증됩니다.

인증은 Authorization 헤더에 전달된 액세스 토큰을 기반으로 합니다.

    Authorization: Bearer {your token}

에 대한 클라이언트 자격 증명을 얻을 수 있습니다. Analytics API임시 액세스 토큰을 검색하는 데 사용합니다. OAuth 개요클라이언트 자격 증명 및 액세스 토큰을 얻는 데 사용할 수 있는 다양한 접근 방식에 대한 설명 API 자격 증명 관리 Studio를 통해 클라이언트 자격 증명을 얻는 방법에 대한 정보

응답

요청에 대한 응답은 다음 형식의 JSON입니다.

    {
        "alltime_video_views": 103123
    }

가능한 구현

다음은 비디오 데이터 엔드포인트를 사용하여 비디오가 플레이어에서 로드될 때 비디오의 전체 조회수를 표시하는 방법입니다.

참고: 이 끝점은 Professional 및 Enterprise 계정에서만 사용할 수 있습니다. 계정 업그레이드에 관심이 있는 경우 Brightcove 영업팀이나 계정 관리자에게 문의하십시오.
  1. 5-10분 동안 비디오 ID별로 조회수를 저장하도록 캐싱을 설정합니다.
  2. 서버 또는 페이지에 로직을 생성하여 먼저 캐시를 확인하여 현재 비디오에 대한 전체 시간 비디오 조회수 데이터가 이미 존재하는지 확인하고 존재하는 경우 이를 사용하십시오.
  3. 값이 아직 캐시되지 않은 경우 비디오 정보 끝점에 요청하여 검색하고 페이지에 표시하는 것과 함께 캐시에 값을 저장합니다.
  4. 비디오 정보 끝점에 대한 요청이 시간 초과되거나 실패한 경우 예외 처리를 설정해야 합니다. 전체 비디오 조회수를 표시하지 않거나 마지막 값을 사용할 수 있습니다.
