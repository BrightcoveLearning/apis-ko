오류 메시지

HTTP 코드 에러 코드 설명 해결
400 BAD_REQUEST 이 오류는 요청에 클라이언트 측 오류가 있음을 나타냅니다. 메시지에는 일반적으로 오류가 무엇인지에 대한 정보가 있습니다. NS message응답 필드에는 `invalid value for sort매개변수.
400 UNSUPPORTED_FIELD_COMBINATION_ERROR 이 오류는 다음을 나타냅니다. fields보고서 매개변수에 보고서 유형에 유효하지 않은 필드가 하나 이상 포함되어 있습니다. NS message응답의 필드에는 지정된 유효하지 않은 필드에 대한 정보가 포함됩니다.
400 TIME_RANGE_UNAVAILABLE 이 응답은 요청한 데이터가 기록/일괄 처리 시스템에 없을 때 다시 전송됩니다.
401 UNAUTHORIZED 이 오류는 요청에 사용된 권한 부여(OAuth2 또는 LA 토큰)가 유효하지 않음을 나타냅니다. 사용 중인 인증 토큰을 확인하세요. 사용된 토큰이 유효 기간을 넘어 새 토큰이 필요할 수 있습니다. 일반적으로 각 토큰은 5분 후에 만료됩니다. 인증 토큰은 url 매개변수로 지정할 수 있습니다. access_token또는Authorization헤더. 인용하다 API 자격 증명 관리자세한 사항은.
404 NOT_FOUND 이 오류는 일반적으로 요청된 엔드포인트가 없을 때 다시 전송됩니다. 요청 중인 엔드포인트를 확인하십시오.
405 METHOD_NOT_ALLOWED 이 오류는 api 요청이 다른 HTTP 메소드로 이루어질 때 발생합니다. GET . 다음을 사용하여 API에 대한 모든 요청을 수행합니다. HTTP GET .
500 SERVER_ERROR 이 오류는 서버에 내부 오류가 있을 때 발생합니다. 사용할 수 없거나 일시적인 문제가 발생한 경우 서버에서 이를 다시 보낼 수 있습니다. 어느 정도의 지수 백오프로 잠시 후에 다시 시도하십시오. 문제가 지속되면 지원 티켓을 만드십시오.
503 PROCESSING 이는 서버가 요청을 수신했으며 현재 처리 중임을 의미합니다. NS Analytics API장기 실행 쿼리가 발생하면 이 메시지를 다시 보낼 수 있습니다. 쿼리가 완료되면 최대 5분 동안 서버 캐시에 저장됩니다. 따라서 이 오류를 수신한 후 4분 후에 API를 쿼리하는 것이 좋습니다.
