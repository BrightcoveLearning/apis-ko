소개

Brightcove 신디케이션 구성 API는 공개적으로 액세스할 수 있는 API로 신디케이션을 생성, 관리하고 Video Cloud 계정의 비디오 카탈로그에서 동적 피드(예: MRSS 피드)를 생성하는 데 사용할 수 있습니다.

관련도 있습니다신디케이션 피드 API신디케이션과 관련된 피드를 검색하는 데 사용할 수 있습니다.

유효성

신디케이션 API는 플랫폼 API에 액세스할 수 있는 모든 Video Cloud 고객이 사용할 수 있습니다.

신디케이션 피드 생성은 이 문서에 설명된 API를 통해 수행해야 합니다. 이러한 작업에는 UI가 없습니다.

API 참조/기본 URL/헤더

NS 구성 API 참조사용 가능한 작업, 필드 및 매개 변수에 대한 모든 세부 정보가 포함됩니다.

기본 URL은 다음과 같습니다.

https://social.api.brightcove.com/v1

모든 요청은 Authorization 헤더를 통해 인증되어야 합니다.

Authorization: Bearer {your_access_token}

액세스 토큰에 대한 정보는 인증에 대한 다음 섹션을 참조하십시오.

요청 본문을 보내는 모든 요청의 경우 다음도 포함해야 합니다. Content-Type: application/json헤더.

입증

구성 API에 액세스하려면 다음을 지정해야 합니다. bearer토큰브라이트코브 OAuth 서비스요청의Authorization헤더. 또한 다양한 API 메서드를 사용하려면 문제의 자격 증명에 대해 다음 작업 중 하나(액세스된 메서드에 따라 다름)를 지정해야 합니다.

  • video-cloud/social/mrss/write

  {
    "name": "80s music videos syndication",
    "type": "advanced",
    "include_all_content": false,
    "include_filter": "tags:mytag",
    "title": "80s Music Videos",
    "description": "Amateur Tokyo drift!",
    "destination_url": "http://mywebsite.com",
    "keywords": "80s, rock",
    "author": "Rick Astley",
    "category": "Music",
    "album_art_url": "http://my_album_art.jpg",
    "explicit": "no",
    "owner_name": "http://my_album_art.jpg",
    "owner_email": "rick@astley.com",
    "language": "en-us",
    "fetch_sources": true,
    "fetch_digital_master": false,
    "fetch_dynamic_renditions": true,
    "sort": "-created_at",
    "content_type_header": "application/xml"

신디케이션 리소스
들 유형 읽기 전용 설명
id 끈 예 신디케이션이 생성될 때 생성됨
name 끈 아니요 이 신디케이션의 내부 이름 - POST 요청에 필요
content_type_header 끈 아니요 설정된 경우 이 신디케이션의 피드에 대해 피드 서버에서 반환된 Content-Type 헤더를 재정의합니다. 그렇지 않으면 피드는 기본적으로 신디케이션 유형별 헤더 값으로 설정됩니다.
include_all_content 부울 아니요 true인 경우 모든 카탈로그 비디오가 이 신디케이션에 포함됩니다.
include_filter 끈 아니요 include_all_content가 false인 경우 지정해야 합니다. 값은 CMS API다음을 사용하여 검색 문자열 CMS API 동영상 검색 구문 v2 . 모든 구문 규칙이 적용됩니다. 유일한 차이점은 검색 문자열이include_filter보다는 가치?query=매개변수
피드 응답은 페이지의 항목( 페이지당 최대 100개 ) 지정된 include_filter를 사용하여 카탈로그에서 가져온 다음 신디케이션 템플릿을 적용합니다. 결과적으로 다음을 통해 항목을 필터링합니다. include_filter템플릿 내에서 필터링이 가져온 페이지의 항목이 거의 또는 전혀 일치하지 않는 경우 반환될 수 있는 잠재적으로 훨씬 적은 수의 항목 대신 전체 항목 페이지를 반환할 수 있으므로 범용 템플릿 내에서 논리를 사용하여 항목을 필터링하는 것보다 선호됩니다. 카탈로그에서.
type 끈 아니요 생성될 피드 유형입니다. 범용 유형을 사용하면 업로드된 피드 템플릿에서 사용자 지정 피드를 생성할 수 있습니다. 유효한 값: advanced , google , iphone , ipad , mp4 , itunes , roku , source , universal . POST 요청에 필요
title 끈 아니요 이 피드의 제목입니다. 이는 해당 피드 유형에 대한 <channel> 태그 내부에 포함됩니다.
description 끈 아니요 이 피드에 대한 설명입니다. 이는 해당 피드 유형에 대한 <channel> 태그 내부에 포함됩니다.
syndication_url 끈 예 이 신디케이션의 MRSS 피드 URL
destination_url 끈 아니요 해당 피드 유형에 대한 <channel> 태그 내부에 포함될 URL
keywords 끈 아니요 쉼표로 구분된 목록, iTunes 및 (잠재적으로) 범용 피드에만 사용됨
author 끈 아니요 iTunes 및 (잠재적으로) 범용 피드에만 사용됨
owner_name 끈 아니요 iTunes 및 (잠재적으로) 범용 피드에만 사용됨
language 끈 아니요 itunes 및 (잠재적으로) 범용 피드에만 사용됨 - 다음과 같은 소문자의 두 글자 언어 코드"en"
owner_email 끈 아니요 iTunes 및 (잠재적으로) 범용 피드에만 사용됨
category 끈 아니요 iTunes 및 (잠재적으로) 범용 피드에만 사용됩니다. 하위 범주가 있는 범주를 지정하려면 콜론(:)으로 구분합니다. 예를 들면 다음과 같습니다. "Business:Business News". "category": "Music"
album_art_url 끈 아니요 이미지용 URL, itunes 및 (잠재적으로) 범용 피드에만 사용됨
fetch_sources 부울 아니요 범용 템플릿의 경우 비디오 소스 메타데이터를 가져올지 여부 - 템플릿에 이 메타데이터가 필요하지 않은 경우 다음을 지정하여 성능을 개선할 수 있습니다. false ; 사용 가능한 소스가 없으면 비어 있을 수 있습니다.
fetch_digital_master 부울 아니요 범용 템플릿의 경우 비디오 디지털 마스터 메타데이터를 가져올지 여부 - 템플릿에 이 메타데이터가 필요하지 않은 경우 다음을 지정하여 성능을 개선할 수 있습니다. false ; 디지털 마스터가 없으면 비어 있을 수 있습니다.
fetch_dynamic_renditions 부울 아니요 범용 템플릿의 경우 비디오 동적 변환 메타데이터를 가져올지 여부 - 템플릿에 이 메타데이터가 필요하지 않은 경우 다음을 지정하여 성능을 개선할 수 있습니다. false
sort 끈 아니요 원하는 피드 결과 반환 순서를 나타내는 CMS 비디오 정렬 지정자. 다음과 같은 CMS 지원 값name , reference_id , created_at , published_at , updated_at , schedule.starts_at , schedule.ends_at , state , plays_total , 그리고plays_trailing_week지정할 수 있습니다. 내림차순으로 정렬하려면 값 앞에 빼기(-) 기호를 사용합니다. -created_at , 지정하면 피드가 가장 최근의 순으로 정렬됩니다. updated_at기본적으로 날짜.

오류 메시지

API 요청이 실패하면 오류 메시지가 반환됩니다. 오류 응답은 다음과 같습니다.

  [
    {
      "error_code" : "Application error code 1",
      "message" : "Application error message 1"
    }, {
      "error_code" : "Application error code 2",
      "message" : "Application error message 2"
    }
  ]
GET 요청에 요청 본문을 포함하면 요청이 실패하고 오류가 반환됩니다.

신디케이션 생성

방법: POST

끝점: /accounts/{account_id}/mrss/syndications

샘플 요청 본문:

  {
    "name": "my mp4 feed",
    "type": "mp4"
  }

참고로name그리고type필드는 필수 항목입니다. 기타는 선택 사항입니다.

  }

범용 템플릿 언어

범용 신디케이션을 사용하면 카탈로그 데이터를 사용자 지정 템플릿과 병합하여 원하는 모든 종류의 피드를 생성할 수 있습니다. 지원되는 템플릿 언어는액체 . 기본 신디케이션 유형에 대한 피드는 동일한 유형의 템플릿을 사용하여 생성됩니다. 샘플로 제공되는 템플릿필요한 경우 사용자 지정 템플릿을 구축하는 데 도움이 됩니다.

액체로의 확장

toUTC 필터

우리는 대부분의 표준 ISO-8601 날짜/시간 문자열 형식을 구문 분석하고 표준 UTC 날짜/시간 문자열로 변환하는 toUTC 필터를 지원하도록 Liquid 파서를 확장했습니다. 이 형식은 Liquid의 날짜 필터에 사용할 수 있으며, 원하는 형식의 날짜/시간 문자열로 타임스탬프 형식을 다시 지정하는 데 사용할 수 있습니다. 예를 들면 다음과 같습니다.

  <pubDate>{{asset.published_at | toUTC | date: "%a, %d %b %Y %H:%M:%S %Z"}}</pubDate>

asset.published_at 값이 2019-08-09T13:32:52.031Z인 경우 다음과 같은 출력이 생성됩니다.

  <pubDate>Fri, 09 Aug 2019 09:32:52 +0000</pubDate>

에포크 필터

우리가 사용하는 Liquid 파서는 날짜를 Unix epoch 타임스탬프로 변환하기 위한 날짜 필터에서 "%s" 토큰을 지원하지 않습니다. 이 문제를 해결하기 위해 유효한 날짜 사양을 Epoch 형식으로 변환하는 데 사용할 수 있는 toEpoch 사용자 지정 필터가 제공됩니다. 필터는 수학 필터에 대한 입력에 적합한 Epoch 이후 밀리초를 나타내는 숫자 데이터 값을 반환합니다. 예를 들면 다음과 같습니다.

  <toEpochMillis>{{"now" | toEpoch }}</toEpochMillis>
  <toEpochSeconds>{{"now" | toEpoch | divided_by : 1000 }}</toEpochSeconds>
  <thirtyDaysAgo>{{'now' | toEpoch | minus:2592000000 | fromEpoch }}</thirtyDaysAgo>

다음과 같은 출력을 생성합니다.

  <toEpochMillis>1580917253024</toEpochMillis>
  <toEpochSeconds>1580917253</toEpochSeconds>
  <thirtyDaysAgo>2020-01-06T15:40:53.055Z</thirtyDaysAgo>

fromEpoch 필터

fromEpoch 필터는 epoch 이후 밀리초를 나타내는 숫자를 UTC 날짜 문자열로 변환합니다. 필터는 또한 epoch 값 숫자를 포함하는 문자열을 입력으로 받아들입니다. 필요한 경우 형식을 다시 지정하기 위해 출력을 날짜 필터로 전달할 수 있습니다.

예를 들면 다음과 같습니다.

  
  <fromEpochMillis>{{"now" | toEpoch | fromEpoch }}</fromEpochMillis>
  <thirtyDaysAgoAltFormat>{{1580917253024 | fromEpoch | date:"%Y-%m-%d" }}</thirtyDaysAgo>

다음과 같은 출력을 생성합니다.

  
  <fromEpochMillis>2020-02-05T16:09:37.809Z</fromEpochMillis>
  <thirtyDaysAgoAltFormat>2020-02-05</thirtyDaysAgo>
