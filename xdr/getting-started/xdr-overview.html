개요

Brightcove의 XDR(교차 장치 재개) 기능을 사용하면 시청자가 한 장치에서 비디오 시청을 시작하고 비디오를 일시 중지한 다음 나중에 다른 장치에서 계속 볼 수 있습니다. 시청자가 재생을 재개하면 중단한 부분부터 계속 시청하게 됩니다. 이는 콘텐츠에 대한 원활한 시청 경험을 제공합니다.

XDR 서비스는 가장 최근의 플레이 위치를 포함하여 시청 활동을 저장합니다. 이 정보는 모든 장치에서 재생을 재개하기 위해 검색됩니다.

Brightcove의 XDR 기능:

  • 고객 웹 사이트 및 모바일 장치 앱과 통합
  • 데스크톱 및 모바일 장치에서 REST 기반 API 및 모국어 호출 사용
  • Video Cloud 라이브러리에 저장된 콘텐츠와 함께 작동
  • 90일 동안 플레이 위치 정보를 유지합니다.
  • GDPR 및 COPPA 규정 준수
XDR 데이터의 제출과 API를 통한 해당 데이터의 가용성 사이에는 최대 30초의 지연이 있습니다.

요구 사항

교차 기기 이력서를 사용하려면 다음이 필요합니다.

  • 최신 버전의 Brightcove Player 및 교차 장치 재개를 지원하는 기본 SDK를 사용해야 합니다.
  • 재생 위치를 가져오기 위해 REST API 요청을 하려면 서버 측 애플리케이션이 필요합니다.

계정 설정

계정 관리자에게 문의하여 교차 기기 재개용 계정을 활성화하세요.

프로세스 흐름

이 기능을 사용하면 시청자가 한 기기에서 시청을 시작하고 일시 중지하고 다른 기기에서 계속 시청할 수 있습니다.

1 단계

시청자가 비디오를 시청할 때 플레이어는 시청 중인 비디오의 재생 헤드 위치를 지속적으로 기록합니다. 이 정보는 XDR API에 저장됩니다.

2 단계

시청자가 다른 장치에서 비디오 시청을 재개하면 XDR API에서 재생 헤드 위치가 검색됩니다. 시청자가 중단한 부분부터 재생이 다시 시작됩니다.

구현

교차 기기 이력서를 구현하려면 다음 단계를 따르세요.

  1. 뷰어 데이터가 자동으로 Brightcove 분석으로 전송되더라도 앱에서 뷰어 식별자를 설정해야 합니다. 자세한 내용은 3단계에서 링크된 문서를 참조하십시오.

    XDR API에 전송 및 저장되는 정보에는 다음이 포함됩니다.

    • account_id - 비디오 클라우드 계정 ID
    • viewer_id - 재생 헤드가 저장되는 뷰어의 ID
    • video_id - 시청자가 시청한 비디오의 Video Cloud ID

    • playhead_seconds - 현재 재생 헤드 위치(초)

  2. XDR API에 요청하여 뷰어의 재생 위치를 가져옵니다. 재생 위치에 대한 REST API 요청을 하려면 서버 측 애플리케이션이 필요합니다.

    XDR API에 대한 요청은 재생 위치를 초 단위로 반환합니다.

    자세한 내용은 다음을 참조하세요.

    • XDR API에서 재생 위치 가져오기
    • 교차 기기 이력서(XDR) API 참조

  3. 애플리케이션에서 재생을 재개합니다. 자세한 내용은 다음을 참조하세요.

    • Brightcove Player를 사용한 교차 장치 이력서
    • 네이티브 SDK를 사용한 교차 기기 이력서
