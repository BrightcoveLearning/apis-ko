소개

분석 데이터는 Native Player SDK에서 제공하는 데이터를 포함하여 Brightcove Player에서 자동으로 전송됩니다. 당신이있는 경우 ~ 아니다 Brightcove Player를 사용하여 Video Cloud 비디오를 제공하려면 데이터 수집기로 데이터를 보내는 데 사용하는 플레이어를 계측해야 합니다.

데이터 수집 API v2가 현재 표준입니다. v1 버전은 더 이상 사용되지 않습니다. v1 구현이 있는 경우 다음을 참조하십시오. v1의 변경 사항아래 섹션.

이 개요 및 API 참조 , 이것도 참조샘플 구현 .

Analytics Data Collection API는 실시간 분석 이벤트의 끝점입니다. 이벤트 데이터는 다음과 같은 HTTP 요청을 통해 제출된 일련의 매개변수를 통해 Brightcove로 전송됩니다.

  http://metrics.brightcove.com/v2/tracker?event=video_view&amp;domain=videocloud&account=123&video=789

이러한 매개변수는 다음을 설명합니다. 사실이벤트가 발생했을 때 시스템의 상태에 대해. 위의 예는video_view동영상 이벤트 발생789계정을 위해123 (또는: 사용자가 계정을 보기 시작함123님의 동영상789 . 보다아래에추적된 현재 분석 이벤트에 대한 설명).

치수

차원은 이벤트가 발생할 때 시스템 상태에 대한 정성적 사실입니다. 예를 들어 요청이 다음과 같은 경우:

  http://metrics.brightcove.com/tracker
  ?event=video_view&session=581136_2018-07-03T18:34:46.214Z
  &domain=videocloud&account=123
  &video=789

동영상 ID( 789 ) 및 계정 ID( 123 ), 요청 자체에서 수집한 모든 장치 및 위치 정보는 모두 관련 차원입니다. video_view이벤트. Analytics 시스템은 다음을 기록합니다. video_view이 차원과 함께 이 요청이 수행될 때 이벤트가 발생했습니다.

이벤트 및 도메인 매개변수

NS event매개변수는 발생한 이벤트를 설명합니다. NS domain매개변수는 이벤트에 대한 네임스페이스를 제공합니다. NS event , domain , 그리고session필수 매개변수(값domain항상videocloud ).

추가 매개변수

Analytics 시스템이 이벤트를 성공적으로 분석할 수 있도록 특정 매개변수는 이벤트에 포함되어야 합니다.

당신은해야 ~ 아니다보내다player (플레이어 ID) 또는player_name Brightcove가 아닌 플레이어를 위한 정보입니다. 대신 다음을 사용하십시오. destination동영상을 본 위치를 추적하는 매개변수입니다.

응답 유형

분석 데이터 수집 API 요청에 대한 응답에는 HTTP 응답 코드와 사람이 읽을 수 있는 메시지가 포함됩니다.

HTTP 상태 코드 설명 예시
200 요청이 수집기에서 성공적으로 수신되었으며 지속되었습니다. (1x1 픽셀 투명 GIF 이미지 반환)
400 클라이언트가 보낸 요청에 필수 매개변수가 누락되었습니다. domain , account또는event . (도메인별 매개변수가 누락된 경우 이 상태가 반환되지 않습니다.) "Invalid 'event' parameter"
50x 이것은 서버 측의 문제를 나타내는 오류 코드입니다. 귀하의 이벤트는 분석 시스템에 의해 성공적으로 기록되거나 기록되지 않았을 수 있습니다. "Server-side failure, please retry."

라이브 이벤트

Data Collection API가 이벤트를 다음과 같이 분류하려면 다음 조건이 충족되어야 합니다. 살다 :

  • 요청해야 ~ 아니다 video_duration 매개변수가 있습니다.
  • 요청에는 계정 매개변수가 있어야 합니다.
  • 요청에는 비디오 매개변수가 있어야 합니다.
  • 이벤트 유형은 다음 중 하나여야 합니다.
    • video_impression
    • video_view
    • video_engagement
    • 살아있는_ss_ad_start
  • 라이브 비디오 스트리밍을 위해 Brightcove 지원에서 계정을 활성화해야 합니다.

모범 사례

Collector에 올바른 데이터를 보내고 있는지 확인하려면 일반적으로 배포하기 전에 데이터 수집 스크립트를 테스트해야 합니다. 다음을 권장합니다.

  1. 플레이어에 대한 데이터 수집 스크립트를 빌드합니다.
  2. 최소한 하루 동안 통제된 환경에서 테스트하십시오.
  3. 분석 모듈 또는애널리틱스 API수집된 내용이 귀하의 기대와 일치하는지 확인합니다.

정크 데이터

일반적으로 Collector로 전송된 데이터는 Analytics 시스템에 의해 사실로 기록됩니다. 이벤트에 부적절하거나 잘못된 정보가 포함된 경우 Analytics 시스템은 데이터를 잘못 해석합니다.

예를 들어, 실수로 타임스탬프를 비디오 ID로 보내는 경우 분석 데이터가 전체 요약에 영향을 미치는 방식으로 왜곡됩니다.

데이터 전송을 시작하기 전에 생성 중인 URL을 주의 깊게 검사하십시오!

URI 인코딩

공백이나 특수 문자를 포함할 수 있는 데이터 수집 API에 보내는 모든 문자열 URI로 인코딩되어야 합니다.요청이 성공하려면. JavaScript를 통해 요청을 제출하는 경우 다음을 사용할 수 있습니다. encodeURI()메서드는 요청 문자열을 인코딩합니다. 예를 들면 다음과 같습니다.

  urlStr += "&video=" + currentVideo.id + "&video_name=" + encodeURI(currentVideo.video_name);

고유한 사용자 식별자 - 제공되지 않거나 비어 있는 경우 Video Cloud는 대체 방법을 사용하여Source IP address + the User-Agent고유 식별자로서의 문자열. Brightcove는 이 정보를 순 사용자 계산에만 사용합니다. API 또는 Analytics 모듈을 통해 사용자 데이터 자체를 검색할 수 없습니다.

사용자 매개변수

  • 플레이어/클라이언트 애플리케이션이 고유한 뷰어를 추적하려면 사용자 매개변수로 사용자의 고유 ID를 수집기에 보내야 합니다.
  • 만약user제공되지 않거나 비어 있으면Source IP address + the User-Agent String고유 식별자로.
  • 사용자 매개변수의 값은 로그/데이터베이스에 저장되지 않으며 해시(SHA-256 사용)만 저장됩니다.
  • 수집가가 설정한 쿠키가 없습니다.

고유 사용자

Brightcove Player의 플러그인 기능을 사용하여 보고된 분석에 고유한 비디오 뷰어 데이터를 추가할 수 있습니다. 이렇게 하려면 고유 식별자를settings분석 기능의 개체입니다.

물론 고유한 사용자 ID를 캡처하는 방법은 응용 프로그램마다 다르지만 예를 들어 이 코드에서는 다음과 같은 고유한 사용자 데이터가 포함된 로그인 URL이 캡처된다고 가정합니다. http://exampledomain.com/users/912389123 . 이 고유한 URL은 플러그인에 전달됩니다.

아래 플러그인 코드는 다음 작업을 수행합니다.

  • 표준 구문을 사용하여 플러그인 이름이 다음과 같이 정의된 Brightcove Player 플러그인을 만듭니다. uniqueUserForAnalyticsPlugin . 플러그인도 허용합니다. options플러그인에 전달된 데이터를 포함하는 객체.
  • NS myPlayer변수에는 플레이어에 대한 참조가 할당됩니다. 또한 두 개의 다른 변수가 생성됩니다.
  • NS userPath변수에는 플러그인을 통해 전달된 경로가 할당됩니다. options물체.
  • NS uniqueViewer변수에는 구문 분석된 버전이 할당됩니다. userPath , 따라서 사용자 ID 숫자만 변수에 할당됩니다.
  • Analytics 플러그인에 사용자 속성이 추가됩니다. settings물체.
  videojs.registerPlugin('uniqueUserForAnalyticsPlugin', function(options) {
  var myPlayer = this,
  userPath = '',
  uniqueViewer = '';
  //Assign uniqueViewer a value according to your app and business rules
  //In this example, parsing the path passed to the plugin in the options object
  userPath = options.path;
  uniqueViewer = userPath.substring( userPath.lastIndexOf('/') + 1 );
  //Assign a user variable to Analytic's settings object
  myPlayer.bcAnalytics.client.user(USER) = uniqueViewer;
  });

이 코드는 응용 프로그램 논리에 맞게 변경한 다음 인터넷에 액세스할 수 있는 URL에 저장해야 합니다.

Studio에서 사용플러그인섹션을 클릭하여 표시된 대로 플레이어에서 플러그인을 로드합니다.

이어지는 JSON 대신 사용자 데이터가 포함된 문자열을 플러그인에 전달합니다. 물론 고유한 사용자 ID를 추출하려면 플러그인 코드를 적절히 업데이트해야 합니다.

  {
  "path": "http://exampledomain.com/users/912389123"
  }

기본적으로 Analytics 시스템은 User-Agent 헤더에서 장치 유형 및 OS 정보를 감지하려고 시도합니다. 만약에둘 다device_type그리고device_os전송되면 User-Agent 헤더의 정보는 무시됩니다. device_type그리고device_os . 대부분의 경우 기기, OS, 브라우저 정보를 보낼 필요가 없습니다. -- 이 재정의는 User-Agent가 신뢰할 수 없거나 사용할 수 없는 경우에만 사용해야 합니다.

분석 시스템은 다음을 기록합니다. other요청에 장치 매개변수 재정의에 대해 인식할 수 없는 값이 포함된 경우.

대상 및 소스 매개변수

NS destination그리고source매개변수는 이벤트를 발생시킨 URI를 제공합니다( destination ) 및 사용자를 그곳으로 보낸 URI( source ).

NS source매개변수는 트래픽 소스 정보를 결정하는 데 사용됩니다. 만약에source지정되지 않은 경우 Analytics 시스템은 이벤트를 직접 트래픽에 의해 시작된 것으로 처리합니다.

NS destination매개변수는 트래픽 목적지 정보, 즉 동영상이 시청되는 위치를 결정하는 데 사용됩니다. URI에 권한이 없으면 API는destination_domain . NS destination_path URI에 경로로 기록됩니다.

웹 재생 중 동영상이 재생되는 페이지의 주소 표시줄에 있는 URL은destination , 그리고source참조자( top.document.referrer ).

예를 들어, Brightcove 지원 사이트에서 "라이브 스트리밍 와이어캐스트"를 검색하고 결과에 나타나는 비디오를 볼 때:

매개변수 값
source 
  https://support.brightcove.com/en/video-cloud/search/live%20streaming%20wirecast
destination 
  https://support.brightcove.com/en/video-cloud/training-videos/live-streaming-wirecast

URL이 없는 경우(예: 기본 재생의 경우와 같이) 둘 다destination그리고source비디오가 재생되는 위치와 사용자가 도달한 경로를 각각 식별하는 유효한 URI여야 합니다.

Analytics 시스템은 다음과 같이 처리합니다.

URI에권한 , API 응답은 해당 권한을destination_domain및destination_path . URI에 권한이 없으면 API는destination_domain . NS destination_path URI에 경로로 기록됩니다. NS destination체계가 없는 값과 마찬가지로 계층적 부분이 없는(예: 체계만) 유효하지 않은 것으로 간주됩니다.

NS video_engagement이벤트는 비디오가 재생되는 동안 비디오 참여를 추적하는 수단이며 재생 중에 여러 번 전송될 것입니다. (재생이 중단되지 않은 경우 Flash/HTML5 플레이어 계측은 10초마다 이 이벤트를 보냅니다.) 현재 20초가 넘는 범위를 설명하는 이벤트는 Analytics 시스템에 의해 폐기되므로 이러한 이벤트를 더 자주 보낼 필요가 있습니다.

두 가지 형태가 있습니다video_engagement이벤트는 다음을 취할 수 있습니다(간결함을 위해 다른 매개변수는 생략됨):

예시 의미 
  event=video_engagement&video=123&video_duration=75&range=0..9
동영상123 75초 동안 0초에서 9초까지 재생(총 10초 동안).
event=video_engagement&video=123&video_seconds_viewed=10 10초 영상123조회되었습니다.

두 버전 모두 조회 시간(초)을 추적하지만 다음을 포함하는 버전은video_duration그리고range또한 추가 참여 데이터를 계산하는 데 필요한 정보가 포함되어 있으며선호하는 방법보내다video_engagement이벤트 데이터를 Analytics 시스템으로 전송합니다. 라이브 스트림의 경우 또는 재생 중에 동영상의 타임라인이 계속 변경되거나 신뢰할 수 없는 경우, video_seconds_viewed유일한 데이터가 될 것입니다. VOD의 경우duration사용할 수 없으며, video_engagement이벤트에는 다음이 포함되어야 합니다. video_duration그리고range .

매개변수 파생 참여 측정항목(API)
video_duration , range video_seconds_viewed , video_percent_viewed , engagement_score ; 참여 곡선 데이터
video_seconds_viewed video_seconds_viewed

세 개의 매개변수가 모두 있는 경우( video_duration , range그리고video_seconds_viewed )와 함께 전송됩니다. video_engagement이벤트가 발생하면 Analytics 시스템은video_duration + range매개변수.

V2 변경 사항

이 섹션에서는 v1을 사용하는 사람들을 위해 Data Collector의 v1에서 v2로의 변경 사항에 대한 요약을 제공합니다.

트래커의 기본 URL

  http(s)://metrics.brightcove.com/v2

모든 이벤트에서 지원되는 추가 필드:

device_os_version : 장치에서 사용 중인 os 버전입니다. 지정하지 않으면 추적 요청에 대한 사용자 에이전트 문자열을 구문 분석하여 계산됩니다.

플랫폼 버전 : 지정된 플랫폼의 새 릴리스가 이벤트를 보내는 데 사용되고 있음을 나타내는 데 사용됩니다.

V2의 새로운 이벤트

카탈로그 요청 : videocloud 카탈로그 API에 대한 요청이 있을 때 전송됩니다. 이 이벤트는 내부 사용을 위한 것이며 Analytics 모듈이나 Analytics API를 통해 노출되지 않습니다.

  • 카탈로그 URL : 연결된 도착 URL catalog_request이벤트 - 이 이벤트는 내부용이며 Analytics 모듈이나 Analytics API를 통해 노출되지 않습니다.

카탈로그 응답 : 이전에 응답할 때 전송catalog_request수신됨 - 이 이벤트는 내부용이며 Analytics 모듈이나 Analytics API를 통해 노출되지 않습니다.

  • 카탈로그 URL : 연결된 도착 URL catalog_request이 응답을 시작한 이벤트 - 이 이벤트는 내부용이며 Analytics 모듈이나 Analytics API를 통해 노출되지 않습니다.
  • 응답 시간_ms : 사이의 시간(밀리초) catalog_request이벤트와catalog_response이벤트 - 이 이벤트는 내부용이며 Analytics 모듈이나 Analytics API를 통해 노출되지 않습니다.

play_request : 사용자가 재생 버튼을 명시적으로 클릭하여 재생이 시작되거나 자동 재생 시나리오에서 플랫폼이 재생을 트리거할 때 자동으로 전송됩니다.

ad_mode_begin : [대체ad_start ] 재생 플랫폼에서 광고 에이전트에게 제어권이 넘어갈 때 전송됩니다.

ad_mode_complete : [대체ad_end ]광고 에이전트에서 재생 플랫폼으로 제어권이 반환될 때 전송됩니다.

오류 : 재생 경험을 방해하는 치명적인 오류가 발생하면 전송됩니다.

  • 에러 코드 : 이벤트와 관련된 플랫폼별 오류 코드입니다.

V2에 대한 업데이트된 이벤트

video_view : 새로운 대기 시간 측정 포함

  • load_time_ms : 비디오에 대한 데이터 로드를 시작하고 비디오가 재생 가능하게 되는 사이의 시간(밀리초)입니다.
  • 시작 시간_ms : 재생 시작과 렌더링되는 비디오의 첫 번째 프레임 사이의 시간(밀리초)입니다. 이는 경험에 따라 다를 수 있습니다. 예를 들어 프리롤 광고가 구성되지 않은 경우 이 측정은 'play_request' 그리고video_view이벤트. 프리롤 광고가 있는 경우ad_mode_begin그리고ad_mode_complete포함하지 않아야 합니다.

video_engagement : 추가 변환 선택, 비트 전송률 측정 및 버퍼링 정보가 포함됩니다. 또한 참여 기간 동안 시청이 발생하지 않은 경우에도 주기적으로 전송되어야 한다는 점에서 비디오 참여에 미묘한 변화가 있었습니다. 이 변경 사항은 사용자가 재생을 기다리게 하는 재버퍼링 지연 및 횟수를 추적할 수 있도록 하기 위한 것입니다.

  • 범위 : 이제 범위 매개변수는 선택 사항이며 범위는 참여 이벤트에서 제외되어 이벤트가 적용되는 기간 동안 시청 활동이 없음을 표시할 수 있습니다. (예: re-buffering 활동만 있는 경우)
  • rendition_url : 가장 최근에 선택한 변환의 URL입니다. 예를 들어 HLS 스트림의 경우 가장 최근에 선택한 변형의 URL이 됩니다.
  • rendition_indicated_bps : 가장 최근에 선택한 변환의 표시된 비트 전송률(초당 비트 수)입니다.
  • rendition_mime_type : 가장 최근에 선택한 변환의 MIME 유형입니다.
  • rendition_height : 비디오 변환의 인코딩된 높이(픽셀)
  • rendition_width : 비디오 변환의 인코딩된 너비(픽셀)
  • rebuffering_seconds : 사용자가 참여 기간 동안 요청하지 않은 지연으로 인해 동영상이 재생될 때까지 대기한 시간(초)입니다.
  • rebuffering_count : 표시된 참여 기간 동안 재버퍼링으로 인해 재생이 중지된 횟수입니다.
  • forward_buffer_seconds : 현재 포워드 버퍼에 있는 비디오의 시간(초)입니다.
  • 측정된_bps : 가장 최근에 다운로드한 세그먼트에 포함된 비트 수와 해당 세그먼트를 다운로드하는 데 소요된 시간의 비율(초당 비트 수)입니다.
  • player_width참여 범위 끝에서 플레이어의 현재 픽셀 너비입니다.
  • 플레이어_높이참여 범위 끝에서 플레이어의 현재 픽셀 높이입니다.
  • drop_frames : 이 참여 기간 동안 비디오 재생에서 삭제된 프레임 수
