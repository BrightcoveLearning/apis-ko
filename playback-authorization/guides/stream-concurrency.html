이 기능은 재생 권한 부여 서비스의 제한된 가용성 단계에 액세스할 수 있는 특정 고객 집합에서 사용할 수 있습니다.

소개

동시 스트림 제한을 사용하여 특정 사용자가 주어진 시간에 시청할 수 있는 비디오 스트림 수를 정의합니다. 사용자당 동시 스트림을 제한하면 시청자가 계정이 없는 친구와 자격 증명을 공유하지 못하게 됩니다.

동시 스트림의 최대 수에 도달하고 뷰어 또는 계정 자격 증명을 가진 사람이 추가 스트림을 열려고 하면 가장 오래된 세션이 중지되거나 최신 요청이 차단됩니다. 이 동작을 제어합니다.

이 기능은 DRM 워크플로에 연결되어 안전하고 최소한의 통합으로 사용할 수 있습니다.

이 기능을 처음 사용하는 경우 다음을 참조하십시오. 개요: 재생 권한 관리 서비스문서.

계정 설정

스트림 동시성에 대해 계정을 활성화하려면 계정 관리자에게 문의하세요.

구현

동시 스트림 제한을 구현하려면 다음을 수행하십시오.

  1. 동시 스트림 제한과 연결된 클레임으로 JSON 웹 토큰(JWT)을 만듭니다. 자세한 내용은만들기 JSON 웹 토큰 (JWT)문서.
  2. 재생 제한을 사용하도록 플레이어를 구성하고 JSON 웹 토큰 (JWT) 비디오 콘텐츠를 검색할 때. 자세한 내용은플레이어 구성재생 권한 관리 서비스 구현 문서의 섹션.

갱신 간격

동시 스트림 제한이 적용되도록 하기 위해 정의된 갱신 간격에 따라 재생 중에 DRM 라이선스 갱신 요청이 전송됩니다. 설정 시 이 갱신 간격을 정의해야 합니다.

중간 스트림 재생 승인 요청은 공개 키 ID와 함께 JWT 토큰에 동시성 제한을 지정해야 합니다.

현재 각 갱신 요청은 구매한 라이선스 수에 포함되는 추가 라이선스로 계산됩니다. 간격을 결정할 때 이 점을 고려해야 합니다.

행동 이해하기

이 기능을 사용하면 스트림 차단 동작을 선택할 수 있습니다.

가장 오래된 스트림 차단

Brightcove는 DRM으로 보호된 콘텐츠의 재생 스트림 제한을 지원합니다. 새로운 갱신 요청이 처리되면 가장 오래된 세션이 중지됩니다. 이를 통해 새로운 사용자 요청에 액세스 권한을 부여하는 동시에 더 이상 유효하지 않을 수 있는 가장 오래된 요청을 차단하여 네트워크 또는 장치 오류에 대한 메시지를 표시할 수 있습니다.

이것이 기본 동작입니다.

사용자 활동:

  1. 벤, 제이슨, 사라는 영화를 보기 시작합니다.
  2. Jamie도 시청하기로 결정하고 Ben의 다가오는 갱신 요청이 거부됩니다.
  3. Ben은 영화를 다시 시작하여 이제 Jason의 갱신 요청이 실패하게 됩니다.
  4. 제이슨은 얼마 후 다른 영화를 보기 시작하여 사라의 영화가 중단됩니다.

새 요청 차단

또는 최대 동시 스트림 수에 도달하면 새 라이선스 요청을 차단할 수 있습니다. 이를 통해 모든 현재 사용자가 계속 시청하면서 새로운 사용자 요청에 대한 메시지를 표시할 수 있습니다.

이 동작을 활성화하려면 다음을 설정해야 합니다. cbeh당신의 주장 JSON 웹 토큰(JWT)다음과 같이: 

"cbeh": "BLOCK_NEW"

다음은 예입니다.

사용자 활동:

  1. 벤, 제이슨, 사라는 영화를 보기 시작합니다.
  2. Jamie는 시청을 시작하려고 하지만 최대 한도 때문에 요청이 차단됩니다.
  3. Jason은 시청을 중단하고 다른 시청자가 시청을 시작할 수 있도록 합니다. Jamie는 영화를 보기 시작합니다.

플레이레디

PlayReady를 사용할 때 이전 섹션에서 정의한 것과 동일한 일반적인 동작을 예상해야 합니다. 그러나 에 설명된 대로 플레이어에서 수동 라이선스 갱신을 구현해야 합니다. Microsoft의 동시성 제한문서.

원정 여행

Safari 브라우저에서는 동시성 제한이 지원되지만 갱신 요청은 전송되지 않습니다. 이는 새 세션/스트림이 다른 브라우저보다 더 오랜 기간 동안 차단된다는 것을 의미합니다.

노트

다음 참고 사항이 이 기능에 적용됩니다.

  • 수동 라이선스 갱신 요청은 다음과 같은 경우에 구현해야 합니다.
    • Widevine으로 보호되는 경우를 제외하고 비 Brightcove 플레이어의 동시성 제한
    • 모든 장치에서 PlayReady에 대한 동시성 제한
  • 수동 라이센스 갱신은 정의된 갱신 간격보다 짧은 기간에 요청해야 합니다.
