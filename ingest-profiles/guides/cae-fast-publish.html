소개

빠른 게시를 통해 다음을 사용할 수 있습니다. CAE즉시 재생할 수 있는 하나 이상의 임시 변환을 만들어 비디오를 가능한 한 빨리 온라인으로 가져옵니다. 그러면 다음 조건이 적용됩니다.

  • 동적 변환을 사용할 수 있게 되면 재생을 위해 임시 변환 대신 사용됩니다.
  • 모든 동적 변환이 처리된 후 어느 시점에서 임시 변환이 자동으로 삭제됩니다.

요구 사항

빠른 게시를 사용하려면 다음을 수행해야 합니다.

  1. 에 대해 계정을 활성화하십시오. 동적 전달
  2. 에 대해 계정을 활성화하십시오. CAE
  3. 다음을 사용하여 사용자 지정 CAE 수집 프로필을 만듭니다. Ingest Profiles API (지침아래에 )

빠른 게시를 위한 맞춤 프로필

빠른 게시가 가능한 사용자 지정 프로필을 만드는 것은 간단합니다. 일반 CAE 프로필처럼 보이지만 한 가지 차이점이 있습니다. renditions배열, dynamic_origin개체에는 다음이 포함됩니다. temporary_renditions하나 이상의 빠른 게시 변환을 포함하는 배열입니다.

빠른 게시 변환은 이름과 ~해야하다아래 목록에서 가져옵니다(다음 번호비디오 또는 오디오는 변환의 대상 비트 전송률입니다). 원하는 경우 비디오 변환 외에 오디오 전용 변환을 포함할 수 있습니다.

빠른 게시의 요점은 가능한 한 빨리 재생할 수 있도록 변환을 만드는 것이므로 임시 변환 중 하나 이상에 대해 중저 비트 전송률을 사용하는 것이 합리적일 수 있습니다.

사용자 지정 프로필을 만드는 단계

다음은 하나의 임시 변환으로 사용자 정의 빠른 게시 프로필을 만드는 단계입니다. 아래 단계에서는multi-platform-standard-dynamic사용자 지정 프로필의 기반으로 표준 프로필을 사용할 수 있지만 CAE프로필.

아래 절차를 사용하여 원하는 만큼 사용자 지정 프로필을 만들 수 있지만 혼란과 사용자 오류를 방지하기 위해 실제로 필요한 만큼만 만드는 것이 좋습니다.

계정에 맞춤 프로필 추가

이제 Ingest Profiles API를 사용하여 계정에 사용자 지정 프로필을 추가합니다.

REST API 요청을 수행하기 위해 일반적으로 수행하는 도구를 사용할 수 있습니다. 이 설정에 도움이 필요하면 사용 가이드를 참조하세요. 불명 증 , 우편 집배원 , 또는곱슬 곱슬하다 .
액세스 토큰 가져오기

각 API 요청은 액세스 토큰을 사용하여 인증되어야 합니다(정확한 방법은 이후 단계에서 볼 수 있음).

  1. 당신은해야 할 것입니다액세스 토큰을 얻다에서: 
        https://oauth.brightcove.com/v4/access_token
  2. 사용하는 경우불명 증또는우편 집배원 , 그런 다음 연결된 문서의 지침을 사용하여 OAuth2 인증을 설정하여 액세스 토큰을 가져오도록 할 수 있습니다.
  3. cURL을 사용하는 경우 액세스 토큰을 얻는 데 사용할 수 있는 cURL 명령이 있습니다. ENCODED_AUTHENTICATION_STRING다음으로 구성된 Base64 인코딩 문자열로client_id:client_secret: 
        curl --request POST \
      --url 'https://oauth.brightcove.com/v4/access_token?grant_type=client_credentials' \
      --header 'Authorization: Basic ENCODED_AUTHENTICATION_STRING' \
      --header 'Content-Type: application/x-www-form-urlencoded'
    당신이 사용할 수있는이 유틸리티 Base64로 인코딩된 인증 문자열을 가져옵니다.
  4. 액세스 토큰은 5분 후에 만료된다는 점에 유의하십시오. 그런 일이 발생하면 새 토큰을 받으십시오(Insomnia는 필요할 때 자동으로 하나를 얻습니다).
사용자 지정 수집 프로필 추가
  1. 사용하는 경우불명 증또는우편 집배원 , 새 POST 요청을 다음과 같이 설정합니다. 
        https://ingestion.api.brightcove.com/v1/accounts/account_id/profiles

    요청 본문은 이전에 생성한 사용자 지정 프로필에 대한 JSON입니다.

    포함Content-Type: application/json헤더

    Insomnia 또는 Postman 사용 가이드의 지침에 따라 OAuth2 인증을 설정합니다.

  2. cURL을 사용하는 경우 다음 요청을 사용하여 account_id 및 액세스 토큰 값을 자신의 값으로 바꿀 수 있습니다.

    {
      "entity": "contextAwareEncoding5",
      "entityType": "DYNAMIC_RENDITION",
      "version": "1",
      "action": "CREATE",
      "jobId": "413a9938-6d73-478c-b4d9-fdeb45927a4b",

      "dynamicRenditionId": "contextAwareEncoding5",
      "bitrate": 364,
      "width": 512,
      "height": 384,
      "accountId": "1910141566001",
      "status": "SUCCESS"
