소개

이 항목의 정보는 대부분 사용자 지정 수집 프로필에 적용됩니다. 표준은 모든 아카이브 마스터를 프로파일링합니다(참조아래에 )

비디오를 수집할 때 마스터를 보관하면 나중에 다시 수집하지 않고도 비디오를 다시 트랜스코딩할 수 있습니다. 이 항목에서는 마스터 보관 옵션에 대해 설명합니다. 마스터를 보관한다는 것은 나중에 처리할 수 있도록 마스터 비디오 파일을 계정에 저장하는 것을 의미합니다. 보다  Dynamic Delivery를 위한 수집 프로필 생성 자세한 내용은.

비디오를 수집할 때 디지털 마스터를 저장한 다음 마스터를 아카이브하지 않는 프로필을 사용하여 비디오를 다시 트랜스코딩하면 비디오의 디지털 마스터가 ~ 아니다제거되어도 비디오를 다시 코드 변환할 수 있습니다.

마스터 보관

비디오의 마스터를 보관하려면 다음을 포함해야 합니다. digital_master수집 프로필의 개체:

    {
        "account_id": 3423487123048,
        "digital_master": {
            "rendition": "passthrough",
            "distribute": false
        },
        "name": "nondrm-default",
        ...
    }

소스 또는 변환

Dynamic Delivery(Context Aware Encoding 포함) 프로필의 경우 비디오 소스 파일을 마스터로 보관할 수 있습니다. "rendition": "passthrough"에서digital_master프로필의 일부). digital_master는 ~ 아니다설정한 경우에도 이러한 프로필에 대해 배포(사용 가능한 변환으로 CDN에 푸시)됩니다. "distribute": true .

레거시 인제스트 프로필의 경우 비디오 소스 파일을 마스터로 사용하도록 선택하거나 마스터로 사용할 고품질 변환을 선택할 수 있습니다.

수집한 비디오 소스 파일을 마스터로 사용하려면 수집 프로필에 다음 코드를 포함하세요.

    "digital_master": {
        "rendition": "passthrough",
        "distribute": false
    }

원본 파일 대신 마스터에 대한 변환을 사용하려면(레거시 수집 프로필만 해당) rendition분야digital_master반대하다참조 ID변환:

    "digital_master": {
            "rendition": "av0",
            "distribute": true
        },
        "name": "nondrm-av0",
        "packages": [],
        "renditions": [
        {
            "media_type": "video",
            "reference_id": "av0", ...

설정하면rendition에게none , 마스터가 보관되지 않습니다. , 그리고 당신은 ~ 아니다비디오를 다시 트랜스코딩할 수 있습니다. 그러나 다음을 수행할 수 있습니다. 바꾸다동영상을 다시 수집합니다.

분포

배포는 다음에서만 작동합니다. 레거시 수집 프로필 .

마스터 배포 여부를 선택할 수 있습니다. Distributed는 마스터가 CDN으로 푸시되고 재생 가능한 렌디션으로 사용 가능함을 의미합니다. 마스터가 배포되지 않으면 Video Cloud에 보관되지만 CDN으로 푸시되지는 않습니다.

노트:
  • 마스터로 변환 중 하나를 선택하고 다음을 설정한 경우distribute에게false , 해당 변환은 CDN으로 푸시되지 않으며 재생할 수 없습니다.
  • 변환이 다음으로 설정된 경우passthrough그리고distribute에게true , 마스터는 h264 형식인 경우에만 배포됩니다.
  • 원본 비디오는 인터넷을 통해 재생할 수 있도록 수정될 수 있습니다. 아카이브된 마스터는 원본 파일과 동일하지만 매우 드문 경우를 제외하고는 코드 변환을 위해 수정해야 합니다.
  • 마스터를 배포하는 것은 ~ 아니다를 사용하여 수집된 비디오 작업동적 전달프로필. Dynamic Delivery 수집 프로필을 보면distribute로 설정됩니다false . 프로필을 복제하고 Studio에서 변경하려고 하면 값이 다음으로 되돌아갑니다. false자동으로.

마스터를 CDN에 배포하려면 다음을 설정하십시오. "distribute": true에서digital_master물체:

    "digital_master": {
        "distribute": true,
        "rendition": "passthrough"
    }

마스터가 배포되는 것을 방지하려면 다음을 설정하십시오. distribute거짓으로:

    "digital_master": {
        "distribute": false,
        "rendition": "passthrough"
    }

기본 동작

기본적으로(프로필에digital_master필드), Dynamic Ingest API그리고업로드 모듈프로필에 다음이 포함된 것처럼 정확하게 작동합니다.

    "digital_master": {
        "distribute": false,
        "rendition": "none"
    }

즉, 디지털 마스터가 저장되지 않습니다.

프로필에 다음이 있는 경우:

    "digital_master": {
        "distribute": false
    }

...없이rendition아카이브해야 할 항목을 지정하는 필드, 소스 비디오보관됩니다디지털 마스터로.
