재산 유형 설명 기본
account_id 정수 NS 비디오 클라우드계정 ID 없음
name 끈 프로필 이름 - 계정 내에서 고유해야 합니다. 없음
display_name 끈 Studio 관리 페이지에 표시할 프로필의 표시 이름은 다음과 같을 수 있습니다. name , 그러나 사용자가 프로필의 의도된 용도를 이해하는 데 도움이 되도록 설명해야 합니다. 없음
description 끈 프로필 설명 없음
model_version 정수 다음으로 설정1프로필을 라이브 모듈에 표시하려면 없음
renditions 정렬 rendition 객체의 배열 []
packages 정렬 DRM 패키지 배열 - 현재 DRM은 지원되지 않으므로 항상 빈 배열이어야 합니다. []

변환 필드

아래 표는 라이브 수집 프로필 업데이트를 위해 요청 본문에서 사용할 수 있는 필드를 보여줍니다.

재산 유형 설명 기본
audio_bitrate 정수 kbps의 오디오 비트 전송률 소스 일치
audio_codec 끈 오디오 코덱 - aac | mp3 형식 또는 프로필에 의해 결정< - 대부분의 경우, aac최고가 될 것입니다/td>
decoder_bitrate_cap 정수 kbps 단위, 버퍼를 통해 디코더에 공급되는 최대 비트 전송률 없음
decoder_buffer_size 정수 kbps에서, 디코더를 사용할 때 디코더에 공급되는 버퍼의 크기bitrate_cap 없음
forced_keyframe_rate 숫자 X 프레임마다 키프레임을 강제 적용하지만 장면 자르기 등을 위한 추가 키프레임은 계속 허용합니다. 없음
format 끈 비디오 형식 - ts , mp4 , flv 없음
h264_profile 끈 H.264에는 일반적으로 사용되는 세 가지 프로필이 있습니다. Baseline (최저), Main , 그리고High .[1-5] baseline
hls_optimized_ts 부울 HLS에 최적화된 시간 세그먼트 - HLS 출력에만 적용됨 없음
keyframe_interval 정수 키프레임 사이의 최대 프레임 수 250
keyframe_rate 뜨다 초당 최대 키프레임 keyframe_interval 250개 중
label 끈 이미지 변환을 위한 이미지 유형 - poster또는thumbnail 없음
live_stream 부울 이것이 라이브 비디오 스트림인지 여부 없음
live_dvr_sliding_window_duration 정수 초 단위로 LiveDVR을 제공할 수 있는 스트림 지속 시간 300
max_video_bitrate 정수 kbps 단위, 123 = 123kbps - h264 전용 없음
max_frame_rate 뜨다 제한frame_rate설정하기 보다는. 60FPS 지원에서 계정을 활성화하지 않은 경우 30으로 설정해야 합니다. 없음
media_type 끈 video또는audio 없음
video_bitrate 정수 kbps 단위, 123 = 123kbps 없음
video_codec 끈 예: h264, ts 형식 또는 프로필에 따라 결정됨
width 정수 프레임 너비(픽셀) 없음
height 정수 프레임 높이(픽셀) 없음
