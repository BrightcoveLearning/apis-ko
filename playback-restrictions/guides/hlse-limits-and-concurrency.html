이 기능은 Brightcove 재생 제한의 제한된 가용성 단계에 액세스할 수 있는 특정 고객 집합에서 사용할 수 있습니다.

소개

사용자당 콘텐츠를 제한하고 싶지만 DRM이 필요하지 않거나 통합하려는 경우 HLSe를 사용하는 것이 솔루션이 될 수 있습니다.

AES-128(HLSe)을 사용한 표준 암호화를 사용하면 인터넷을 통해 콘텐츠를 암호화할 수 있습니다. 공개적으로 액세스할 수 있는 암호 해독 키를 사용하기 때문에 DRM을 사용하는 것만큼 안전하지 않습니다.

HLSe 사용 Brightcove의 라이센스 키 보호 , 사용자 또는 고급 정보를 기반으로 재생 요청을 승인하여 암호화 키에 대한 액세스를 허용하거나 거부하고 표준 암호화의 보안을 향상할 수 있습니다.

라이선스 키 보호는 DRM 라이선스 요청 및 AES-128 암호 해독에 대한 보호에 중점을 둡니다. JWT 클레임은 라이선스 키 보호가 사용자 수준에서 제한을 지원할 수 있도록 합니다. 즉, 사용자 에이전트, 최대 수의 다른 IP 주소 또는 각 토큰에 대한 최대 사용자 수로 재생을 제한할 수 있습니다.

이 기능을 처음 사용하는 경우 다음을 참조하십시오. 개요: Brightcove 재생 제한문서.

시작하기

계정을 설정하고 콘텐츠를 준비하세요.

  1. HLSe 및 재생 제한에 대한 계정을 활성화하려면 계정 관리자에게 문의하십시오.
  2. 계정에 HLSe가 활성화되면 보호하려는 비디오를 수집하거나 다시 코드 변환하십시오.

워크플로

HLSe를 사용하여 장치 제한을 설정하거나 동시성을 스트리밍하려면 다음 단계를 따르세요.

  1. 만들기 JSON 웹 토큰(JWT)사용자당.

    각 사용자 세션에 대해 JWT를 생성해야 합니다. 이는 다른 청구와 함께 원하는 제한 사항을 기반으로 생성되어야 합니다.

    • 장치 제한은 다음을 참조하십시오. 기기 제한 설정부분.

    • 스트림 동시성에 대해서는 다음을 참조하십시오. 스트림 동시성 설정부분.

  2. 라이선스 키 보호를 사용하도록 플레이어를 구성합니다.

    자세한 내용은라이센스 키 보호 사용개요 섹션: Brightcove 재생 제한 문서.

기기 제한 설정

라이선스 키 보호를 사용하면 사용자당 설정된 장치 수로 재생을 제한할 수 있습니다.

클레임

  • :maxip

    HLSe 콘텐츠의 경우maxip클레임은 동시에 스트리밍할 수 있는 서로 다른 IP 주소의 수를 정의합니다.

예시

이 예에서는 사용자당 하나의 IP 주소(사용자당 하나의 장치)로 재생을 제한하는 JWT를 만듭니다.

  1. 공개-개인 키 쌍을 생성합니다.

    자세한 내용은공개-개인 키 쌍 생성 JWT(JSON 웹 토큰 만들기) 문서의 섹션을 참조하세요.

  2. 공개 키를 등록합니다.

    자세한 내용은인증 API 사용문서.

  3. JSON 웹 토큰을 만듭니다.

    다음은 bash를 사용한 예입니다. 반드시 추가하세요"maxip":1사용자당 하나의 IP 주소(장치)로 재생을 제한한다고 주장합니다.

    #! /usr/bin/env bash
  # Static header fields.
  HEADER='{
    "type": "JWT",
    "alg": "RS256"
  }'
  
  payload='{
    "pkid": "your policy key",
    "accid": "your account id",
    "maxip": 1
  }'
  
  # Use jq to set the dynamic `iat` and `exp`
  # fields on the payload using the current time.
  # `iat` is set to now, and `exp` is now + 1 second.
  PAYLOAD=$(
    echo "${payload}" | jq --arg time_str "$(date +%s)" \
    '
    ($time_str | tonumber) as $time_num
    | .iat=$time_num
    | .exp=($time_num + 60 * 60)
    '
  )
  
  function b64enc() { openssl enc -base64 -A | tr '+/' '-_' | tr -d '='; }
  
  function rs_sign() { openssl dgst -binary -sha256 -sign {your_private_key_path} ; }
  
  JWT_HDR_B64="$(echo -n "$HEADER" | b64enc)"
  JWT_PAY_B64="$(echo -n "$PAYLOAD" | b64enc)"
  UNSIGNED_JWT="$JWT_HDR_B64.$JWT_PAY_B64"
  SIGNATURE=$(echo -n "$UNSIGNED_JWT" | rs_sign | b64enc)
  
  echo "$UNSIGNED_JWT.$SIGNATURE"
