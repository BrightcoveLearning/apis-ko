소개

콘텐츠가 내부 또는 외부 청중을 대상으로 하든 무료든 수익을 창출하든 상관없이 Brightcove Playback Restrictions은 사용 사례에 맞는 적절한 수준의 콘텐츠 보안을 제공하는 동시에 시청자가 장소에 관계없이 모든 장치에서 콘텐츠에 액세스할 수 있도록 합니다.

이 기능을 사용하면 다음을 포함하여 콘텐츠 스트리밍을 위한 다양한 시나리오를 정의할 수 있습니다.

  • 콘텐츠를 사용할 수 있는 곳
  • 콘텐츠 스트리밍 방식
  • 콘텐츠를 볼 수 있는 경우
  • 내 콘텐츠를 볼 수 있는 사람

블로그 게시물을 확인하세요: 보안 그 이상 - 브랜드 무결성 및 고객 약속에 관한 것입니다.

Playback Restrictions은 어떻게 적용됩니까?

Playback Restrictions은 다음과 같이 적용할 수 있습니다.

  • 자산 수준 제한
  • 런타임 제한
  • 라이센스 키 보호

자산 수준 제한

자산 수준 제한은 재생 권한에 정의되어 있습니다.

다음에 대한 스트리밍을 제한하려는 경우 자산 수준에서 Playback Restrictions을 사용할 수 있습니다.

  • 지역(지역 제한)
  • 시간(스케줄링)
  • 도메인
  • 프록시 유형
기본적으로 새로 수집된 모든 자산에는 언제 어디서나 모든 사람에게 스트리밍할 수 있는 재생 권한이 있습니다.

지역

지리적 제한은 스트리밍 콘텐츠를 전 세계에서 볼 수 있는 곳을 제한합니다. 옵션에는 다음이 포함됩니다.

  • IP 주소
  • CIDR
  • 우편번호(미국 우편번호에만 사용 가능)
  • DMA(미국 DMA에만 사용 가능)
  • 국가

미정 - 지역 제한에는 다음과 같이 전체 동작을 정의하는 전역 규칙이 있습니다.

  • 전역 규칙이 다음으로 설정된 경우모두 허용 , 모든 재생을 허용하고 지역을 블랙리스트에 올릴 수 있습니다.
  • 전역 규칙이 다음으로 설정된 경우모두 차단 , 모든 재생을 차단하고 지역을 화이트리스트에 추가할 수 있습니다.

시간

예약을 사용하면 다음 두 가지 옵션으로 콘텐츠를 스트리밍할 수 있는 기간을 설정할 수 있습니다.

  • 특정 날짜
  • 특정 요일 및 시간에 반복

도메인

도메인 제한을 사용하면 스트리밍을 허용하거나 차단할 도메인을 설정할 수 있습니다.

대리

프록시 제한은 프록시가 있는 경우 콘텐츠 스트리밍을 방지합니다. 사용 가능한 프록시 유형은 5가지입니다.

  • 익명 - 클라이언트의 IP 주소를 사용할 수 없습니다.
  • 공용 - 공용 인터넷 액세스를 허용하는 위치에서 프록시된 여러 사용자
  • 기업 - 여러 사용자가 중앙 위치를 통해 프록시되는지 확인하고 단일 네트워크 겉보기 IP 주소를 공유할 수 있습니다.
  • 투명 - 클라이언트의 IP 주소는 HTTP 헤더를 통해 사용할 수 있지만 값이 반드시 신뢰할 수 있는 것은 아닙니다.
  • 호스팅
VPN 공급자가 서비스를 계속 개선함에 따라 IP 주소 확인은 차단된 IP 주소에 대해 정의한 것과 다를 수 있습니다. 이 때문에 프록시 차단이 100% 보장되지는 않습니다.

자산 수준 사용 사례

사용 사례

다음은 몇 가지 가능한 사용 사례입니다.

우편번호로 콘텐츠 제한

  • 목표

    Shari(스포츠 리그의 웹 프로듀서)는 스포츠 이벤트의 우편 번호 내의 시청자가 이벤트를 스트리밍할 수 없도록 우편 번호로 콘텐츠 액세스를 제한하려고 합니다.

  • 해결책

    Shari는 재생 권한을 정의하여 미국의 지역별로 우편번호를 블랙리스트에 올릴 수 있습니다. 콘텐츠가 공개되어 있으므로(미들웨어 애플리케이션 처리 권한이 없음) 재생 권한을 비보안으로 설정해야 합니다. 재생 권한이 정의되면 Shari는 이를 각 미디어 자산에 할당합니다.

국가/IP 주소로 콘텐츠 제한

  • 목표

    Dylan(CMO)은 시청자가 우크라이나와 한국을 제외한 모든 국가에서 스트리밍할 수 있도록 콘텐츠 액세스를 국가별로 제한하려고 합니다. 개발 팀이 한국에 있기 때문에 Dylan은 한국 내의 특정 IP 주소에서 스트리밍을 허용해야 합니다.

  • 해결책

    Dylan은 재생 권한을 정의하여 우크라이나와 한국을 차단하고 한국의 IP 주소를 허용하도록 추가할 수 있습니다. Dylan의 콘텐츠는 공개되어 있으므로 모든 재생 권한은 안전하지 않은 액세스를 허용해야 합니다. 재생 권한이 정의되면 Dylan이 이를 각 미디어 자산에 할당합니다.

자산 수준 제한 설정

재생 권한을 사용하면 비디오 재생에 대한 제한을 지정할 수 있습니다. 이러한 권리는재생 권한 API각 세트에는 고유한 ID가 할당됩니다( playback_right_id ). 권한 집합은 0개 이상의 비디오와 연관될 수 있습니다.

동영상에playback_right_id , 재생 권한 개체에 지정된 제한 사항은 해당 비디오에 대한 모든 요청에 적용됩니다.

이 다이어그램은 사용자의 비디오 및 라이선스 요청에서 재생까지의 흐름을 보여줍니다.

자산 수준 제한은 다음 중 하나를 사용하여 정의할 수 있습니다.

  • 브라이트코브 API 사용
  • 비디오 클라우드 스튜디오 사용하기

브라이트코브 API 사용

  2. 그런 다음 사용 CMS API제한 사항을 비디오 자산과 연결합니다.

노트

  • 재생 권한을 사용하면 콘텐츠를 암호화할 필요가 없습니다.
  • 기본적으로 재생 권한이 있는 자산을 스트리밍하려면 유효한 토큰이 필요합니다. 그러나 토큰이 필요하지 않도록 재생 권한을 설정할 수 있습니다.
  • 기본적으로 모든 자산에는 언제 어디서나 모든 사람에게 스트리밍할 수 있는 재생 권한이 있습니다.
  • 각 Video Cloud 계정에는 최대 100개의 재생 권한이 있습니다.
  • 재생 권한은 Video Cloud/CMS API 제한 사항(국가 및 일정)과 함께 작동합니다. 저작물에 대한 CMS 제한은 재생 권한 이전에 확인됩니다.
  • 재생 권한은 정책 키 제한(도메인 및 지역)과 함께 작동하지 않습니다. 웹 플레이어 또는 네이티브 SDK에서 재생 권한을 사용할 때 정책 키가 제거됩니다. 이는 정책 키로 설정된 제한이 무시됨을 의미합니다.
  • 재생 권한은 삭제할 수 없습니다. 활성화 또는 비활성화만

런타임 제한

런타임 시 스트리밍을 제한하려면 특정 권한(클레임)이 있는 JSON 웹 토큰(JWT)이 필요합니다. 이 토큰은 모든 스트림 요청을 통해 전송됩니다.

이 기능을 사용하려면 UMS(사용자 관리 시스템)와 Brightcove의 재생 서비스 간에 미들웨어 솔루션을 구현해야 합니다.

런타임 시 다음 제한 사항을 설정할 수 있습니다.

  • 재생 권한
  • 태그
  • 콘텐츠 목록
  • 특정 사용자 에이전트
  • 동시 스트림 제한
  • 기기 제한

재생 권한

자산에 재생 권한을 할당할 수 있습니다( 자산 수준 제한 ) 또는 다음을 사용하여 런타임에 설정할 수 있습니다. prid주장하다.

런타임 제한은 자산 수준의 재생 권한보다 우선합니다.

태그

런타임 시 스트리밍에 허용할 특정 태그 값이 있는 자산을 식별할 수 있습니다. 이러한 태그는 다음과 같이 정의됩니다. tags주장하다. 다음을 사용하여 태그로 비디오를 그룹화할 수 있습니다. 미디어 모듈 Brightcove의 비디오 클라우드 스튜디오에서.

Playback Restrictions 서비스는 각 비디오와 연결된 태그를 확인합니다. 태그 중 하나 이상이 JWT 토큰의 목록과 일치하면 비디오를 볼 수 있습니다.

JWT 토큰의 태그는 태그 배열로 나열됩니다.

JWT 토큰의 태그 배열은 40개로 제한됩니다.

콘텐츠 목록

사용vids클레임에서 스트리밍을 허용할 자산 목록을 런타임에 지정할 수 있습니다.

자격 확인

자격 확인은 비디오 ID 및/또는 태그를 기반으로 런타임 시 최종 사용자 액세스 권한을 설정합니다. 여기에는 JWT에서 다음 클레임 설정이 포함됩니다.

  • prid
  • vids
  • tags

만약prid (재생 권한 ID)가 JWT에 설정되어 있으면 자산에 대한 재생 권한을 덮어씁니다.

특정 사용자 에이전트

특정 사용자 에이전트에서 스트리밍을 허용하려면ua주장하다. 이를 사용하려면 콘텐츠를 암호화해야 하며 라이선스에 JWT가 필요한 계정 플래그를 설정해야 합니다. 이 계정 플래그를 활성화하려면 계정 관리자에게 문의하세요.

특정 사용자 에이전트를 사용할 때 라이선스 키 보호를 활성화해야 합니다.

동시 스트림 제한

사용자당 동시 스트림을 제한하면 시청자가 계정이 없는 친구와 자격 증명을 공유하지 못하게 됩니다. 동시 스트림 제한을 사용하여 특정 사용자가 주어진 시간에 시청할 수 있는 비디오 스트림 수를 정의합니다.

이 기능을 사용하려면 콘텐츠를 DRM 또는 HLSe로 암호화해야 합니다.

자세한 내용은뷰어당 동시 스트림 제한문서.

HLSe와 함께 동시 스트림 제한을 사용하는 경우 라이선스 키 보호를 활성화해야 합니다.

기기 제한

런타임 시 사용자의 스트리밍 장치 수를 제한할 수 있습니다. 이를 위해서는 콘텐츠가 DRM 또는 HLSe로 암호화되어야 합니다.

장치 등록은 DRM 라이선스 요청이 있을 때 발생하며, 이때 각 장치에 고유한 ID가 할당됩니다. 각 라이선스를 요청할 때마다 장치 제한이 확인되고 적용됩니다.

예를 들어 게시자가 단일 뷰어와 연결된 장치 수를 2개의 장치로 제한하기 위해 콘텐츠 소유자의 요구 사항을 준수해야 하는 경우를 들 수 있습니다. 이 제한에 도달한 후 시청자가 연결된 TV와 같은 제3의 장치에서 콘텐츠를 스트리밍하려고 하면 재생이 거부됩니다. 이 기능을 통해 콘텐츠 게시자는 수익 창출 목표를 달성하고 무단 액세스를 최소화할 수 있습니다.

자세한 내용은장치 제한 구현문서.

HLSe와 함께 장치 제한을 사용할 때 라이선스 키 보호를 활성화해야 합니다.

라이센스 키 보호

License Keys Protection은 Dynamic Delivery를 사용하여 DRM으로 보호된 콘텐츠를 전달할 때 추가 보안 수준을 제공합니다. 이는 콘텐츠에 대한 액세스를 제어하고 콘텐츠의 무단 공유를 방지하려는 고객에게 특히 유용합니다.

DRM 또는 HLSe 콘텐츠 보호는 JWT(JSON Web Token)를 사용하여 모든 스트림 요청을 보호할 수 있는 라이선스/키 요청을 사용합니다.

사용 사례

콘텐츠가 공개되어야 하지만 신뢰할 수 있는 요청이 필요한 경우 이 기능을 사용할 수 있습니다. 예를 들어, HLSe로 암호화된 독점 콘텐츠가 있는 지역 방송사는 해당 플랫폼의 요청만 콘텐츠에 액세스하도록 하려고 합니다. 즉, 시청자는 플랫폼의 콘텐츠만 스트리밍할 수 있습니다. 스트림 URL이 있더라도 플랫폼 외부에서 스트리밍할 수 없습니다.

DRM 및/또는 HLSe 보호 기능이 있는 비디오에 대한 재생 권한을 구현하면 Studio의 미디어 모듈에 있는 미리보기 플레이어에서 비디오가 재생되지 않습니다. 미리보기를 활성화하기 위해 Brightcove는명확한 출처 CMS API용 엔드포인트.

이러한 엔드포인트는 보호되지 않은 URL을 검색할 수 있는 유효한 API 자격 증명이 있는 경우 내부 또는 타사 사용자가 비디오를 미리 볼 수 있도록 하는 데 사용할 수도 있습니다. 그러나 다음 사항에 유의하십시오.

  • 보호되지 않은 URL을 사용할 수 있게 되면 Brightcove와 클라이언트는 콘텐츠에 액세스할 수 있는 사람이나 콘텐츠가 사용되는 방식을 제어할 수 없습니다.
  • 소스에 액세스할 때 계정에 송신 대역폭 요금이 부과됩니다.

라이센스 키 보호를 사용하려면 다음을 수행하십시오.

  1. 미들웨어 구현

    이 기능을 사용하려면 UMS(사용자 관리 시스템)와 Brightcove의 재생 서비스 간에 미들웨어 솔루션을 구현해야 합니다.

  3. 플레이어 구성

    토큰은 재생 요청과 함께 전달됩니다. 토큰이 유효하지 않거나 만료된 경우 콘텐츠에 대한 액세스가 제한됩니다. 플레이어는 토큰을 사용하도록 구성되어야 합니다.

    기본 Android 또는 iOS 플레이어

    Android 또는 iOS용 기본 플레이어 구성에 대한 자세한 내용은네이티브 SDK를 통한 키/라이선스 보호문서.

    나만의 플레이어

    플레이어에서 JWT를 사용하기 위해 추가 구성이 필요하지 않습니다.

노트

  • 이 기능은 재생 권한을 사용하지 않고 사용할 수 있습니다.
  • JWT 토큰에는 특정 권한(클레임)이 필요하지 않습니다. 고객의 개인 키로 서명하기만 하면 됩니다.
  • 콘텐츠는 HLSe 또는 DRM을 암호화해야 합니다.

제한 사항

이 기능에는 다음 제한 사항이 적용됩니다.

  • HLSe(AES-128)의 경우 토큰이 마스터 매니페스트 요청에 추가되고 암호화 키 URL로 전파됩니다. 이러한 URL은 마스터 매니페스트에 추가되므로 고객은 콘텐츠 노출을 최소화하기 위해 사용 횟수를 제한하는 것을 고려해야 합니다.
  • HLSe(AES-128)의 경우 플레이어는 비디오를 재생할 때 일반적으로 변환당 하나씩 여러 요청을 합니다. NS 최대 주장이러한 요청을 처리할 수 있을 만큼 충분히 높게 설정해야 합니다. 콘텐츠 노출을 최소화하려면 추가 소유권 주장을 고려해야 합니다. 변환을 전환하면 새 라이센스가 요청됩니다.
  • 추적 maxu 및 maxip 주장결국 일관성이 있습니다. 즉, 이러한 클레임을 사용하는 토큰을 빠르게 연속해서 여러 번 재사용하는 경우 해당 클레임이 선언한 것보다 더 많이 승인된 토큰을 볼 수 있습니다. 추적은 몇 초 이내에 일관성을 유지하고 해당 기간이 지나면 토큰이 예상대로 차단됩니다.
