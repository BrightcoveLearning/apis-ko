얻다client_credentials

이제 클라이언트 자격 증명을 검색하기 위해 OAuth 서비스를 호출할 준비가 되었습니다. 자격 증명을 요청하는 클라이언트 응용 프로그램 이름을 지정합니다. 이름은 임의적이며 자격 증명이 사용될 응용 프로그램을 추적하는 데 도움을 주기 위한 것입니다. 또한 액세스하려는 작업의 범위를 지정해야 하며 여기에서 다음을 사용합니다. 비디오 클라우드/플레이어/모두수행/플레이어/모두 (Player Management API용). 보다클라이언트 자격 증명 요청에 대한 API 작업현재 지원되는 모든 작업 목록을 보려면

  1. Postman을 시작하고 요청 유형을 다음으로 설정합니다. 우편 .
  2. 다음을 입력하십시오. 요청 URL : 
        https://oauth.brightcove.com/v4/client_credentials
  3. 다음 지침을 수행하여 헤더를 생성합니다.
    1. 요청을 클릭하십시오헤더옵션.
    2. 를 위해열쇠 , 입력 시작권한 부여그리고 언제든지 자동으로 제공되는 옵션에서 선택하십시오.
    3. 를 위해값입력, 문자열BC_TOKEN , 공백, 실제 토큰 값이 차례로 나옵니다.
  4. 클릭하여 사용할 요청 본문 유형을 선택하십시오. 신체옵션을 선택한 다음 라디오 버튼을 x-www-form-urlencoded .
  5. 실제신체내용을 보려면 스크린샷과 같이 4개의 키/값 쌍 항목을 입력합니다. 그들은:
    1. 열쇠: 유형 / 값: 신임장
    2. 열쇠: 최대 범위 / 값: [ { "identity": { "type": "video-cloud-account", "account-id": YOUR_ACCOUNT_VALUE } , "작업": [ "비디오 클라우드/플레이어/전체" ] } ]
      귀하의 계정 ID가 다음으로 대체되었는지 확인하십시오. 귀하의_ACCOUNT_VALUE값에.
    3. 열쇠: 이름 / 값: 당신이 선택한 이름
    4. 열쇠: 설명 / 값: 당신의 선택에 대한 설명
