개요

노트: 클라이언트 자격 증명을 얻는 기본 방법은 다음을 사용하는 것입니다. 스튜디오 관리 UI . 이 문서는 대신 OAuth API를 사용하기로 선택한 사람들을 위한 것입니다.

이 문서의 정보를 이해하는 가장 좋은 첫 번째 단계는 OAuth 개요아직 하지 않은 경우.

당신의BC_TOKEN Brightcove 사용자의 고유한 익명 식별자입니다. 백엔드 Brightcove 시스템에 존재하는 것 외에도 Studio에 로그인할 때 쿠키로 설정되어 Studio에서 사용자의 역할과 액세스 권한이 있는 계정을 알 수 있습니다. 이는 모두 투명하며 사용자의 조치가 필요하지 않습니다.

사용BC_TOKEN클라이언트 자격 증명을 생성하기 위해 OAuth API 요청을 인증하는 중입니다.

노트:
  • NS BC_TOKEN값은 Studio에서 처리하는 API Management 토큰과 관련이 없습니다.
  • 만약 너라면 Studio 관리자를 사용하여 클라이언트 자격 증명 생성 , 검색할 필요가 없습니다. BC_TOKEN - 스튜디오에서 알아서 처리해 드립니다.
  • NS BC_TOKEN쿠키는 세션 쿠키이며 Studio에서 로그아웃할 때마다 만료됩니다(24시간 후에도 자동으로 만료됨). 매번 새로운BC_TOKEN쿠키는 Studio에 로그인할 때 삭제되며 새로운 값을 갖게 됩니다.

요구 사항

  • 클라이언트 자격 증명을 생성하는 데 필요한 관리자 권한이 있는 유효한 Brightcove 계정
