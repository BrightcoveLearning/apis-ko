요구 사항

  • 이 자습서의 API 호출은 MacOS 및 모든 Linux/unix 시스템에 내장된 cURL을 사용합니다. Windows를 실행하는 경우 다음을 참조하십시오. cURL 설정
  • 시스템에 터미널 또는 기타 명령줄 콘솔이 필요합니다.

개요

Brightcove OAuth API를 사용하여 다른 Brightcove API에 액세스하는 데는 두 부분이 있습니다.

  • 클라이언트 자격 증명 가져오기(a client_id그리고client_secret ) 필요한 계정 및 API 작업에 유효합니다. 이는 나중에 사용할 수 있도록 client_id 및 client_secret을 안전한 장소에 저장하는 경우 일회성 작업입니다.
  • 받기access_token API 호출 권한 부여 - access_token은 5분 동안 유효하므로 대부분의 경우 API 호출마다 하나씩 받습니다.

Studio에는 다음을 수행하는 데 사용할 수 있는 UI도 있습니다. 클라이언트 자격 증명 획득 및 관리 . 가능하면 UI를 사용하는 것이 좋습니다. 새 API의 경우 한동안 UI를 통해 자격 증명을 사용하지 못할 수 있습니다. 이 경우 다음 단계에서 수행하는 방법을 배우므로 OAuth API에서 직접 자격 증명을 얻을 수 있습니다.

클라이언트 자격 증명 가져오기

이 단계에서는 cURL 명령을 사용하여 Player Management API에 대한 클라이언트 자격 증명을 가져옵니다.

단계

  1. 에 설명된 대로 사용 가능한 cURL이 있는지 확인하십시오. 요구 사항 ; 명령줄로 이동하여 다음을 입력하여 테스트할 수 있습니다. curl리턴을 누르면 - cURL이 설치된 경우 다음과 같은 메시지가 표시되어야 합니다. 
            curl: try 'curl --help' or 'curl --manual' for more information
  2. 이제 Studio에서 얻을 수 있는 두 가지 정보가 필요합니다. 다음으로 이동하여 Studio에 로그인합니다. 사진관 .
    • 계정 ID
    • BC_TOKEN

    참고: NS BC_TOKEN값은 Studio를 열 때 설정된 쿠키입니다. Studio에서 처리하는 API Management 토큰과 관련이 없습니다.

계정 ID 및 BC_TOKEN 가져오기

  1. 먼저 계정 정보에서 계정 ID가 필요합니다.
  2. 설정 아이콘을 클릭 Studio 헤더에 있습니다.
  3. 드롭다운에서 계정 정보를 선택합니다.
  4. 계정 정보 페이지에서 계정 ID를 복사하고 저장하세요. 나중에 필요합니다.
  5. 이제 브라우저 개발자 콘솔을 엽니다. 대부분의 브라우저에서 OPTION-CMD-i(Mac) 또는 ALT-CTRL-i(Windows)를 눌러 이 작업을 수행하거나 메뉴 항목도 찾을 수 있습니다.
  6. 콘솔에서 다음 JavaScript를 붙여넣고 return 키를 눌러 모달 대화 상자에 BC_TOKEN을 표시합니다.
  7. BC_TOKEN을 클립보드에 복사합니다.
  8. 여기에 계정 ID와 BC_TOKEN을 입력하세요.

    계정 ID:

    BC_TOKEN:

액세스 토큰 가져오기

NS client_id그리고client_secret이전 단계에서 얻은 값은 사용자 이름:비밀번호 요청 시 인증을 위한 조합access_token . 여기에서 사용할 cURL에서 이것을 다음과 같이 전달할 수 있습니다. --user매개변수. 다른 언어에서는 HTTP 요청과 함께 Basic Authorization 헤더에 다음을 전달합니다.

        Authorization: Basic {client_id:client_secret}

참고: 전체 client_id:client_secret ~해야하다 OAuth API에서 허용하도록 Base64로 인코딩해야 합니다. cURL은 --user 자격 증명을 자동으로 Base64로 인코딩하지만 다른 언어에서는 직접 Base64로 인코딩해야 합니다.
