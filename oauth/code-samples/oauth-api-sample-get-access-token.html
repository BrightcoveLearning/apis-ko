소개

대부분의 Brightcove API가 요청을 인증하기 위해 액세스 토큰을 사용하기 때문에 액세스 토큰을 가져오는 것은 대부분의 Brightcove API 작업에서 중요한 작업입니다.

액세스 토큰에 대한 전체 설명은 다음을 참조하세요. Brightcove OAuth 서비스 작동 방식그리고액세스 토큰 받기 .

이 샘플은 웹 앱에서 액세스 토큰을 가져오는 방법을 보여줍니다. OAuth API는 CORS를 지원하지 않고 서버 측에서 요청해야 하기 때문에 앱은 프록시에 정보를 보내고 프록시는 API를 요청하고 응답을 클라이언트에 다시 보냅니다. 여기의 프록시는 PHP로 작성되었지만 모든 서버 측 언어가 수행할 수 있습니다. 인터넷을 통해 앱에 POST 요청을 보낼 수만 있으면 됩니다.

이 샘플 및 관련 코드는 프로덕션 개발을 위한 가이드로 제공됩니다. 코드는 프로덕션 용도로 인증되지 않았으며 공식적으로 지원되지 않습니다.

프록시 코드

이 페이지에서 고유한 버전의 샘플 앱을 빌드하려면 고유한 프록시를 만들고 호스팅해야 합니다. (Brightcove Learning Services에서 사용하는 프록시는 Brightcove 도메인의 요청만 수락합니다.) 출처 제한이 없는 프록시 코드 버전은 아래와 같습니다.

    <?php /** * access-token-proxy.php - Brightcove RESTful API용 프록시 * 액세스 토큰을 가져와 반환합니다. * 액세스: * (*항상* HTTPS를 통해 프록시에 액세스해야 함) * 방법: 포스트 * * @포스트{string} client_id - 요청에 대한 충분한 권한이 있는 OAuth2 클라이언트 ID * @post {string} client_secret - 요청에 대한 충분한 권한이 있는 OAuth2 클라이언트 비밀 * * @returns {string} $response - OAuth API에서 받은 JSON 응답 */ // CORS 활성화 및 기타 헤더 header("Access-Control-Allow-Origin: *"); header("콘텐츠 유형: 애플리케이션/json"); header("X 콘텐츠 유형 옵션: nosniff"); header("X-XSS-보호"); // 단일 자격 증명에 대해 이 프록시를 사용하는 경우 // 클라이언트 ID와 비밀을 전달하는 대신 아래에 하드코딩할 수 있습니다. $client_id = $_POST["client_id"]; $client_secret = $_POST["클라이언트_비밀"]; $auth_string = " {$client_id} : {$client_secret} "; $request = "https://oauth.brightcove.com/v4/access_token?grant_type=client_credentials"; $ch = curl_init($request); curl_setopt_array($ch, array( CURLOPT_POST => TRUE, CURLOPT_RETURNTRANSFER => TRUE , CURLOPT_SSL_VERIFYPEER => FALSE, CURLOPT_USERPWD => $auth_string, CURLOPT_HTTPHEADER => array( '콘텐츠 유형: application/x-www-form-urlencoded', ) )), $response = curl_exec($ch), curl_close($ch ); // 오류 확인($response === FALSE) { die(curl_error($ch)); echo 'An error occurred'; }또 다른{ echo $response; } ?>
