소개

클라이언트 자격 증명을 가져오는 것은 대부분의 Brightcove API가 요청을 인증하는 데 사용하는 액세스 토큰을 가져오기 위한 일회성 전제 조건입니다.

클라이언트 자격 증명에 대한 전체 설명은 다음을 참조하십시오. Brightcove OAuth 서비스 작동 방식 .

이 샘플은 웹 앱에서 클라이언트 자격 증명을 가져오는 방법을 보여줍니다. OAuth API는 CORS를 지원하지 않고 서버 측에서 요청해야 하기 때문에 앱은 필요한 정보를 프록시로 보내고 프록시는 API 요청을 하고 응답을 다시 클라이언트로 보냅니다. 여기의 프록시는 PHP로 작성되었지만 모든 서버 측 언어가 수행할 수 있습니다. 인터넷을 통해 앱에 POST 요청을 보낼 수만 있으면 됩니다.

이 샘플 및 관련 코드는 프로덕션 개발을 위한 가이드로 제공됩니다. 코드는 프로덕션 용도로 인증되지 않았으며 공식적으로 지원되지 않습니다.

BC_TOKEN 받기

얻으려면client_id그리고client_secret OAuth API를 통해BC_TOKEN귀하의 요청을 인증합니다. Studio에 로그인하면 BC_TOKEN이 쿠키로 설정됩니다. 원하는 방식으로 쿠키를 얻을 수 있지만 더 쉽게 만들 수 있도록 다음 JavaScript 스니펫을 만들었습니다. Studio에 로그인할 때 개발자 콘솔에 붙여넣을 수 있습니다. 반품 , BC_TOKEN이 포함된 프롬프트가 나타납니다.

클라이언트 자격 증명을 생성하려면 계정의 관리자여야 합니다(또는 관리자의 BC_TOKEN을 얻음). 관리자 BC_TOKEN이 없는 경우 아래 요청은 오류를 반환합니다.
