소개

인페이지 경험과 관련된 두 가지 API가 있지만 매우 다른 용도로 사용됩니다.

In-Page Experience API(REST)

In-Page Experiences API는 프로그래밍 방식으로 In-Page Experiences를 관리할 수 있는 REST API입니다. 주로 Video Cloud 플랫폼을 CMS 응용 프로그램과 통합하는 데 사용하기 위한 것입니다.

또한 전체보기 API 참조사용 가능한 요청에 대한 자세한 내용은

인페이지 경험 클라이언트 API

In-Page Experiences Client API는 Brightcove Player API가 플레이어의 동작을 제어하는 데 사용되는 방식과 유사하게 런타임 시 In-Page Experience의 동작을 관리하는 데 도움이 되는 JavaScript 라이브러리입니다. API는 하나 이상의 Brightcove 내장 환경을 실행하는 모든 페이지에서 사용할 수 있습니다. 참조개요: 인페이지 경험 클라이언트 API자세한 내용은.

이 개요의 나머지 부분에서는 백엔드에서 인페이지 경험을 관리하기 위한 REST API를 다룹니다.

인페이지 경험에 대한 일반적인 소개는 다음을 참조하세요. 개요: 인페이지 경험 생성, 편집 및 게시 .

다음 섹션에서는 REST API 요청의 4개 그룹에 대한 개요를 제공합니다.

  • 경험담
  • 템플릿
  • 테마
  • 글꼴

경험담

체험 요청은 세 가지 하위 범주로 나뉩니다.

경험 관리
이러한 요청을 통해 계정에 대한 모든 경험 목록을 얻을 수 있습니다. 새 경험을 만들고 특정 경험을 가져오거나 업데이트하거나 삭제할 수도 있습니다. 마지막으로 경험을 복제하여 새로운 경험의 시작점을 만들 수 있습니다.
경험 게시/게시 취소
이는 계정 또는 특정 템플릿에 대한 모든 템플릿 목록을 가져오기 위해 all을 요청합니다.
라이브 경험의 상태 관리
이 두 가지 요청을 통해 라이브 경험의 현재 상태를 얻거나 해당 상태를 변경할 수 있습니다( preEvent 에게live , 예를 들어).

템플릿

템플릿은 경험의 전반적인 모양과 느낌을 제어합니다. 사용자 지정 템플릿을 만들 수는 없지만 몇 가지 표준 템플릿이 있으며 시간이 지남에 따라 더 추가될 수 있습니다. API를 사용하면 계정에서 사용할 수 있는 템플릿 목록을 가져오거나 특정 템플릿을 검색할 수 있습니다.

테마

테마를 사용하면 템플릿을 사용자 정의하여 경험에 필요한 모양과 느낌을 만들 수 있습니다. API는 테마 생성, 가져오기, 업데이트 또는 삭제를 완전히 관리하는 방법을 제공합니다.

글꼴

자신의 글꼴을 사용할 수 있습니다. 이러한 방법을 사용하면 글꼴을 추가, 업데이트, 가져오거나 삭제할 수 있습니다. CSS를 통해 로드된 웹 글꼴입니다.
