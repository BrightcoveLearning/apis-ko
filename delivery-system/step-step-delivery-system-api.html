개요

수행할 단계는 다음과 같습니다.

  • Delivery System API를 사용하여 리포지토리 만들기
  • 파일 저장을 위한 로컬 폴더 생성
  • 플러그인을 만들고 저장소에 복사
  • 새 플러그인을 사용하도록 플레이어 변경
  • 플러그인을 변경하고 플레이어가 변경된 플러그인을 사용하는지 관찰하십시오.

참고: 숙련된 Git 사용자라면 일부 API 호출이 Git에서 수행할 수 있는 동일한 기능을 수행하는 것을 볼 수 있습니다. 실제로 일부 파일 조작 API 호출은 단순히 Git 명령을 수행합니다.

전제 조건

이 단계별 지침을 시작하려면 다음을 완료했다고 가정합니다. 단계별: 선수 관리 . 상기 활동을 완료하여 명령줄 시스템에서 다음 환경 변수를 정의했습니다.

  • {account_id}
  • $EMAIL
  • {player_id}

저장소 만들기

이제 Delivery System API를 사용하여 리포지토리를 만들고 해당 리포지토리에 파일을 복사합니다. 리포지토리가 생성되면 API를 사용하여 리포지토리를 나열하고, 리포지토리의 세부 정보를 표시하고, 리포지토리를 삭제하는 등의 작업을 수행할 수도 있지만 많은 워크플로에서는 이것이 필요하지 않습니다.

새 리포지토리를 만들기 위한 curl 문에서 HTTP를 사용합니다. PUT메소드를 만들고 URL에 새 리포지토리의 이름을 포함합니다. 이 curl 문을 실행하여 이름이 리포지토리를 만듭니다. firstRepo .

    curl \
      --user $EMAIL \
      --request PUT \
      https://repos.api.brightcove.com/v1/accounts/{account_id}/repos/firstRepo

리포지토리 생성을 확인하는 응답을 받게 됩니다. 응답에 리포지토리의 이름과 공개 URL이 표시됩니다.

  • 이름: 리포지토리 이름
  • public_url: 리포지토리에 대한 공개 URL
  • repo_url: 저장소 URL
    {
      "name": "firstRepo",
      "public_url": "http://players.brightcove.net/1507807800001/firstRepo",
      "repo_url": "https://repos.api.brightcove.com/v1/accounts/1507807800001/repos/firstRepo"
    }

폴더 생성

다음으로 Brightcove Player와 연결된 파일을 저장하기 위해 컴퓨터에 로컬로 폴더를 만들어야 합니다. 원하는 위치에 폴더를 생성한 후 이동( cd ) 해당 폴더로 이동합니다.

    mkdir firstRepo
    cd firstRepo

플러그인 만들기

이제 폴더에 플러그인을 만들어 저장소에 복사합니다. 단계별에서 동일한 플러그인을 사용합니다. 표시되는 텍스트 변경을 제외하고 플레이어 관리. 파일을 만들고 아래의 텍스트를 파일에 붙여넣은 다음 저장합니다. 단계별로 사용된 것과 동일한 파일 이름을 사용하십시오. 선수 관리, first-plugin.js .

    videojs.registerPlugin('firstPlugin', function() {
      var player = this,
      overlay = document.createElement('p');
      overlay.className = 'vjs-overlay';
      overlay.innerHTML = "NEW TEXT!";
      player.el().appendChild(overlay);
    });

이제 새로 생성된 플러그인 파일을 리포지토리에 복사합니다. 다음 cURL 문을 사용하여 이 작업을 수행합니다.

    curl \
      --user $EMAIL \
      --form contents=@first-plugin.js \
      --request PUT \
      https://repos.api.brightcove.com/v1/accounts/{account_id}/repos/firstRepo/files/first-plugin.js

복사 후 다음과 유사한 확인이 표시되어야 합니다.

    {
      "name": "first-plugin.js",
      "public_url": "http://players.brightcove.net/1507807800001/firstRepo/first-plugin.js"
    }

다음 cURL 문을 사용하여 내용을 보면 파일이 올바르게 복사되었는지 확인할 수 있습니다.

    curl http://players.brightcove.net/{account_id}/firstRepo/first-plugin.js

검색하여 파일을 볼 수도 있습니다. public_url파일 복사본에서 반환됩니다.

플러그인을 변경하는 경우 Delivery System API를 통해 호스팅되는 모든 파일에 5분 캐싱이 적용되므로 결과를 보는 데 지연이 있을 수 있습니다. 즉, 플러그인을 읽고 사용하는 데 최대 5분이 소요될 수 있습니다. 이것이 초기 테스트라면 지연이 없을 것입니다.

새 플러그인 사용

단계별: 플레이어 관리에서는 CSS 위치 및 플러그인 이름과 함께 플러그인에 대한 JavaScript 코드의 위치를 플레이어에게 알려주기 위해 curl 문을 사용했습니다. 이번에는 플러그인 경로가 파일 사본의 응답에서 찾은 새 경로를 사용한다는 점을 제외하고는 동일한 작업을 다시 수행합니다(복사 응답의 위 2행에 표시됨). 다음 curl 문을 실행하여 새 플러그인 위치를 가리킵니다.

메모: 옵션을 제공하지 않더라도 여기에 표시된 대로 플러그인 이름을 입력해야 합니다.
    curl \
      --header "Content-Type: application/json" \
      --user $EMAIL \
      --request PATCH \
      --data '{
        "scripts": [
          "http://players.brightcove.net/{account_id}/firstRepo/first-plugin.js"
        ],
        "stylesheets": [
          "http://solutions.brightcove.com/bcls/video-js/new-player/first-plugin.css"
        ],
        "plugins": [{
          "name": "firstPlugin"
        }]
      }' \
      https://players.api.brightcove.com/v2/accounts/{account_id}/players/{player_id}/configuration
CSS 파일을 저장소에 보관하는 것도 모범 사례입니다. 이것은 단순함을 위해 이 경우에 수행되지 않았습니다. 또한scripts섹션data변경되는 유일한 섹션이기 때문에 curl 문의 일부입니다. 검토를 위해scripts , stylesheets그리고plugins섹션이 업데이트되었습니다.

반환된 사용preview_url플레이어를 탐색하면 플러그인의 새 텍스트가 표시되는 것을 볼 수 있습니다. 선택하는 경우 플레이어를 게시하고 게시된 플레이어 URL을 가져올 수도 있습니다.

플러그인 변경

새로운 Brightcove 플레이어를 사용할 때의 장점 중 하나는 플러그인을 한 번만 변경하면 쉽게 플레이어를 업데이트할 수 있고 해당 플러그인을 사용하는 모든 플레이어가 업데이트된다는 것입니다. 이 단계에서 플러그인을 약간 변경하고 플레이어 자체를 변경하지 않고 사용 중인 것을 볼 수 있습니다.

열기first-plugin.js파일을 읽고 읽을 텍스트를 업데이트하십시오. 새로운 텍스트!

    overlay.innerHTML = "NEWer TEXT!";

다시 파일을 리포지토리에 복사합니다.

    curl \
      --user $EMAIL \
      --form contents=@first-plugin.js \
      --request PUT \
      https://repos.api.brightcove.com/v1/accounts/{account_id}/repos/firstRepo/files/first-plugin.js

같은 찾아보기preview_url위에서 다시. 참고: 브라우저 캐싱으로 인해 브라우저 탭을 닫고 다시 탐색하거나, 업데이트된 텍스트를 보려면 브라우저의 캐시를 지워야 할 수도 있습니다.

최신 버전의 플러그인이 작동하는 것을 확인하면 플레이어를 게시할 수 있습니다.
