개요

하는 동안브라이트코브 갤러리비디오 포털 구축을 위한 강력한 시스템을 제공하므로 간단한 DIY 포털도 구축할 수 있습니다. 사용자가 비디오를 선택하기 전에 웹 페이지에서 플레이어를 로드하지 않으려면 이 샘플과 같이 비디오 축소판 격자를 표시할 수 있습니다. 축소판을 선택하면 선택한 비디오가 있는 페이지에 Brightcove Player가 로드됩니다.

당신이 필요로 할 것

이 포털을 구축하는 데 필요한 것은 다음과 같습니다.

  • 비디오 클라우드 계정
  • Playback API에 대한 요청을 인증하기 위한 검색 가능 정책 키
  • 일부 웹 개발 기술(HTML, CSS 및 JavaScript)

설계 고려 사항

비디오 데이터 가져오기

비록 CMS API또는 Playback API를 사용하여 비디오 데이터를 가져올 수 있는 경우 Playback API는 다음과 같은 몇 가지 이유로 명백한 선택입니다.

  • 응답 데이터가 캐시되기 때문에(최대 20분 동안) 훨씬 빠릅니다.
  • 비디오를 재생하는 데 필요한 모든 데이터를 반환하는 반면 CMS API비디오 메타데이터와 비디오 소스를 가져오려면 두 개의 개별 요청이 필요합니다.
  • CORS를 지원하므로 서버 측 프록시 앱을 통해 API 요청을 라우팅할 필요가 없습니다.

그러나 Playback API는 비디오 데이터를 가져오는 두 가지 방법을 제공합니다. id(또는 참조 id)로 비디오를 가져오는 기본적인 방법이 있으며, 각각을 요청하기 위해 id 배열을 쉽게 반복할 수 있습니다. 그러나 이는 그다지 효율적이지 않습니다.

더 나은 옵션은 비디오 검색 방법을 사용하는 것입니다. 이 방법을 사용하면 ID로 모든 비디오를 찾아 한 번의 요청으로 반환할 수 있습니다. 요청에 대한 검색 매개변수는 다음과 같습니다.

q=id:6543217890%20id:1234567890%20id:987654321...

NS %20이 검색 문자열에는 URI로 인코딩된 공간이 있습니다. 검색 구문에 대한 자세한 내용은 다음을 참조하십시오. CMS/Playback API: 동영상 검색 .

플레이어

플레이어에 대한 특별한 요구 사항은 없습니다. 기본 Brightcove 플레이어가 제대로 작동합니다.

일부 사용자는 비디오를 전혀 시청하지 않을 수 있으므로 필요할 때까지 플레이어를 로드하기 위해 대역폭을 사용하는 것은 의미가 없습니다. 우리는 사용할 것입니다브라이트코브 플레이어 로더재생을 위해 비디오를 처음 선택할 때 플레이어를 동적으로 로드합니다. 성능을 최대화하기 위해 Brightcove Player JavaScript도 미리 로드하지만 플레이어 로더가 아직 없는 경우 이를 자동으로 수행합니다.

정책 키 가져오기

정책 키는 모든 Brightcove 플레이어에 대해 자동으로 생성됩니다. 그러나 보안상의 이유로 표준 정책 키는 Playback API를 사용하여 비디오를 검색할 수 없습니다. 이 샘플의 경우 검색 가능 키가 필요합니다. 다음을 사용하여 만들 수 있습니다. Policy API , 하지만 가장 쉬운 방법은 Brightcove 학습 샘플 앱검색 가능 키를 가져오는 옵션을 선택했는지 확인하십시오.
