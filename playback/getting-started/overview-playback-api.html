소개

Playback API는 웹 페이지 또는 모바일 앱에서 비디오 또는 재생 목록 데이터를 가져올 때 클라이언트 측에서 사용하기 위한 저지연 API입니다. 그것은이다~ 아니다 Video Cloud를 CMS 또는 다른 시스템과 통합하는 데 사용할 일반 미디어 관리 API - 이를 위해서는 다음을 사용해야 합니다. CMS API .

Playback API 결과는 재생 가능한 동영상만 반환하도록 필터링됩니다(state=ACTIVE, 수집 완료 플래그 = true, 예약된 경우 올바른 데이터/시간). 즉, 다른 변환이 아직 처리 중이더라도 플레이어는 하나의 변환이 존재하는 즉시 비디오를 재생할 수 있습니다. 현재 재생할 수 없는 비디오를 가져와야 하는 경우 다시 사용해야 합니다. CMS API대신에.

또한 성능을 최대화하기 위해 Playback API에서 액세스하는 비디오 데이터는 짧은 기간 동안 캐시됩니다. 특정 데이터 세트 ID가 캐시되는 기간은 다양하지만 최대 20분이 될 수 있습니다.

또한 참조 API 참조 .

현재 플레이어에 있는 비디오에 대한 정보만 원하는 경우 다음을 사용할 수 있습니다. mediainfo Playback API를 호출하는 대신 개체를 사용합니다.

NS mediainfo객체는 현재 플레이어에 로드된 비디오에 대한 정보를 포함하는 반면 Playback API는 모든 비디오에 대한 정보를 검색할 수 있습니다. 자세한 내용은 mediainfo의 비디오 메타데이터문서.

사용 사례

NS 브라이트코브 플레이어그리고브라이트코브 SDK 플레이어 Playback API의 메소드와 동일한 카탈로그 메소드를 포함하고 대부분의 경우 재생을 위해 런타임에 비디오 또는 재생 목록을 검색하는 경우 간단히 사용할 수 있습니다.

그러나 플레이어가 없는 모바일 앱 화면에서 비디오 또는 재생 목록 보기를 제공할 수 있습니다.

또한 비디오 포털의 랜딩 페이지와 같이 화면에 플레이어를 표시하지 않고 비디오 또는 재생 목록을 검색하여 정보를 표시하려는 경우가 있을 수 있습니다.

Playback API는 이러한 경우에 유용합니다.

일반 정보

Playback API는 현재 IPv6 프로토콜을 사용하는 IP 주소에서 오는 요청을 지원하지 않습니다.

기본 URL

Playback API의 기본 URL은 다음과 같습니다.

https://edge.api.brightcove.com/playback/v1
  • 당신은 사용해야합니다 https프로토콜로.
  • IP 제한을 사용하는 고객 ~해야하다사용하다https://edge-elb.api.brightcove.com/playback/v1표준 URL 대신 IP 제한이 있는 북미 이외의 지역에서 사용되는 플레이어에 대한 자세한 내용은플레이어 구성 가이드발생할 수 있는 문제를 방지합니다.

계정 경로

모든 경우에 특정 Video Cloud 계정에 대한 요청이 이루어집니다. 따라서 항상 용어를 추가합니다. accounts기본 URL에 대한 계정 ID:

https://edge.api.brightcove.com/playback/v1/accounts/{account_id}

입증

요청은 다음 세 가지 방법 중 하나로 전달할 수 있는 정책 키로 인증됩니다.

  1. 에 대한 논쟁에서Accept헤더: 
    Accept: application/json;pk={policy_key}

    이것은 브라우저 CORS "사전 비행" 검사의 일부로 추가 요청 없이 먼저 요청을 진행할 수 있기 때문에 브라우저 기반 클라이언트에 권장되는 방법입니다. 이렇게 하면 브라우저가 처음 요청될 때 대기 시간이 절약됩니다.

  2. 에서Authorization realm 키워드를 사용하는 헤더BCOV-Policy : 
    Authorization: BCOV-Policy {policy_key}
  3. 안에BCOV-Policy헤더: 
    BCOV-Policy: {policy_key}

정책 키 가져오기

정책 키를 얻을 수 있는 세 가지 방법이 있습니다.

  1. 모든 Brightcove 플레이어는 자동으로 할당됩니다. 플레이어의 정책 키를 가져오는 방법에 대한 지침을 찾을 수 있습니다. 이 문서 . 정책 키는 계정 전체에 적합하므로 해당 플레이어가 페이지에 포함되어 있는지 여부에 관계없이 사용할 수 있습니다.
  2. 다음을 사용하여 기본 정책 키를 생성할 수 있습니다. 정책 키 빠른 시작
  3. 에 요청할 수 있습니다. Policy API

토큰 인증

HLS 및 DASH 콘텐츠에 대한 토큰 인증이 지원됩니다. 토큰 인증이 필요한 경우 계정 관리자에게 계정에 대해 활성화하는 방법에 대해 문의하십시오.

하드 코딩된 URL 피하기

썸네일, 포스터, 비디오 파일 및 기타 미디어의 URL은 페이지나 애플리케이션에 하드 코딩되어서는 안 됩니다. NS Playback API미디어 파일에 대해 캐시된 최신 버전의 URL을 반환하지만 URL 자체는 변경될 수 있습니다. 당신은 사용해야합니다 CMS API ( 또는Playback API ) 페이지가 로드될 때마다 이러한 URL을 검색하거나 6시간 이하 동안 캐시하도록 요청합니다.

비디오 및 이미지 URL 캐싱

페이지 성능을 향상시키기 위해 비디오 및 이미지의 URL을 캐시할 수 있지만 캐시는 정기적으로 새로 고쳐야 합니다. 페이지 성능을 향상시키기 위해 검색한 URL을 캐시하는 경우 6시간에 한 번 이상 API 호출을 반복하여 캐시를 새로 고쳐야 합니다.

동영상 검색

다음을 사용해야 합니다. 검색 가능 정책 키 , Playback API를 사용하여 동영상을 검색할 때. 자세한 내용은Playback API 참조문서.

Playback API 검색은 요청당 100개의 동영상으로 제한되며 최대 1000개의 동영상으로 제한됩니다. 1000개 이상의 동영상을 반환해야 하는 경우 CMS API .

매개변수

모든 매개변수는선택 과목 . 그들은 적용GET Videos요청만.

GET 비디오 요청 매개변수
매개변수 설명
q 검색을 위한 쿼리 문자열 - 자세한 내용은 참조검색 가이드
limit 반환할 동영상 수 - 1에서 100 사이의 정수여야 합니다. 기본: 20
offset 건너뛸 비디오 수(페이징 결과용). 양의 정수여야 합니다. 기본: 0
sort 정렬 기준이 되는 필드를 지정하는 문자열입니다.
시작-내림차순으로 정렬합니다.
값이q제공된 경우 기본 정렬은 "점수"(원래 쿼리에 대한 검색 결과의 관련성) 기준입니다.
값이 없는 경우q가 제공되면 기본 정렬은 다음과 같습니다. updated_at내림차순.
다음 필드는 정렬에 유효합니다. name , reference_id , created_at , published_at , updated_at , schedule_starts_at , schedule_ends_at , state , plays_total , 그리고plays_trailing_week
자세한 내용은비디오 가져오기 Playback API 참조 섹션

검색 매개변수 사용

브라이트코브의Playback API에서 동영상을 검색하는 프로그래밍 방식을 제공합니다. 비디오 클라우드도서관.

비디오 데이터에 대한 기본 및 복잡한 검색을 수행하려면 다음을 사용합니다. q매개변수:

https://edge.api.brightcove.com/playback/v1/accounts/1752604051111/videos?q={search terms}

동영상 검색 방법에 대한 자세한 내용은동영상 검색문서.

페이징 결과

사용limit요청 시 반환할 항목 수를 지정하는 매개변수(최대 100개). 그런 다음 사용할 수 있습니다. offset보다 큰 결과 세트를 통해 페이지에 대한 매개변수limit . NS offset건너뛸 항목의 수입니다.

예를 들어, 다음 검색은 총 결과 세트에 최소 75개의 비디오가 있다고 가정할 때 총 결과 세트의 비디오 51-75를 반환합니다.

/videos?q=updated_at:2014-01-01..2014-06-30&limit=25&offset=50

NS limit그리고offset매개변수는 비디오에만 사용할 수 있습니다.

페이징 모범 사례

Playback API에서 결과 세트를 페이징할 때 다음 단계를 따르는 것이 좋습니다.

  1. 사용limit그리고offset매개변수를 사용하여 결과 집합에서 데이터 그룹을 반환합니다. 
    /accounts/578380111111/videos?q=tags:nature&limit=20&offset=50
  2. 각 응답과 함께 다음을 확인할 수 있습니다. count필드는 항상 결과 세트의 최대 비디오 수를 표시합니다. 
    {"count": 171,
  "videos": [ ... ]
}
    Playback API 검색은 1000개의 동영상으로 제한됩니다.
  3. 전체count첫 번째 요청의 동영상 수를 사용하여 요청해야 하는 페이지 수를 결정할 수 있습니다.

    또는 다음과 같이 질문할 수 있습니다. count각 응답에 값을 입력하고count페이지 크기 곱하기보다 작거나 같습니다. limit . 
    count <= page-number * limit

  • 업데이트됨_at
  • schedule_starts_at(참고: 이것은종류필드 - 검색 필드는schedule.starts_at )
  • schedule_ends_at(참고: 이것은종류필드 - 검색 필드는schedule.ends_at )
  • 상태
  • play_total[1-2]
  • play_trailing_week[1-2]

노트

  • [1-1] 화상 검색 호출에 대한 정렬 값을 제공하지 않으면 결과가 관련성을 기준으로 정렬됩니다. 정렬 값을 제공하지 않으면GET화상 통화, 결과는 다음 기준으로 정렬됩니다. updated_at내림차순.
  • [1-2] 정렬할 수 있습니다. plays_total또는plays_trailing_week , 하지만 이러한 필드는 결과에 포함되지 않습니다.

모든 비디오 및 대용량 데이터 세트

계정에 있는 모든 동영상을 검색하거나 많은 수의 동영상을 검색하는 경우 다음 사항에 유의해야 합니다.

  1. 허용된 가장 큰 값을 사용하고 싶을 수 있습니다. limit (100), 그러나 API 요청 시간 초과 가능성을 최소화하려면 25개 이하의 배치로 비디오를 검색하는 것이 좋습니다.
  2. 큰 데이터 세트를 페이징할 때 작업 중에 비디오 데이터가 업데이트되어 응답에서 항목이 이동할 수 있습니다.
    • 연속 페이지에서 항목이 반복되는 것을 볼 수 있습니다.
    • 항목이 이전 응답 세트로 이동되었기 때문에 누락될 수 있습니다.

    첫 번째 가능성을 고려하려면 동영상 검색을 완료한 후 앱에서 전체 항목 목록의 중복을 제거해야 합니다. 두 번째 가능성을 처리하려면 검색된 총 항목 수(중복 제거 후)를 예상한 수와 비교한 다음 요청을 다시 실행하여 결과를 last_modified_date(내림차순)로 정렬해야 합니다. 놓친 항목을 선택하기 위해 둘 이상의 배치를 검색합니다.

  3. 반환된 결과를 적절하게 정렬하여 이전 항목의 시나리오 가능성을 줄일 수 있습니다. 기본 정렬 기준관련성검색은 키워드, 태그 및 사용자 정의 필드 값의 조합을 찾는 복잡한 알고리즘을 기반으로 합니다. 여러 키워드, 태그 및/또는 사용자 정의 필드를 기반으로 비디오를 검색하는 경우 관련성을 기준으로 정렬하는 것이 정확히 원하는 것입니다. 그러나 전체 또는 많은 비디오 세트를 검색하려는 경우에는sort매개변수를 명시적으로 지정하면 반환된 항목의 순서를 더 잘 제어할 수 있습니다.

  • name
  • short description
  • long description
  • tags

애플리케이션에서 Brightcove 웹 플레이어 또는 Brightcove 네이티브 SDK를 사용하지 않는 경우에도 자체 플레이어에서 SSAI(서버 측 광고 삽입)를 사용할 수 있습니다.

노트:
  • SSAI를 사용하려면 Video Cloud 계정을 다음으로 구성해야 합니다. 동적 전달 SSAI에 대해 활성화되었습니다. 이 기능을 사용하려면 계정 관리자에게 문의하세요.
  • 현재 SSAI는 지원하지 않습니다. 다중 오디오 트랙 .

SSAI로 비디오 받기

SSAI가 포함된 동영상을 반환하려면 광고 구성 ID를 Playback API 요청에 URL 매개변수로 추가하세요. 이 요청으로 비디오 ID 또는 비디오 참조 ID를 사용할 수 있습니다.

SSAI(서버 측 광고 삽입)를 사용하려면 동영상이 Dynamic Delivery를 위해 수집됨 .

캡션

자막은 액세스하려는 개인을 위해 비디오에 대한 추가 정보를 제공합니다. 청각 장애인이 동영상의 오디오 부분에 액세스하려면 자막이 필요합니다. 캡션을 처음 사용하는 경우 다음을 참조하십시오. 개요: 비디오에 캡션 추가문서.

외부 WebVTT

외부 WebVTT 캡션은 동영상과 별도의 텍스트 파일입니다. 다음과 같이 캡션 파일을 비디오와 연결할 수 있습니다.

  • 미디어 모듈을 사용하여 캡션 추가
  • CMS API를 사용하여 캡션 추가

샘플 응답

다음은 Media 모듈을 사용하여 WebVTT 캡션 파일이 비디오에 추가된 Playback API의 샘플 응답입니다. 텍스트 트랙src속성은 외부 캡션 파일을 가리킵니다.

매니페스트 내 WebVTT

매니페스트 내 WebVTT 캡션은 비디오와 함께 수집됩니다. 이것은 비디오를 처음 수집할 때 또는 비디오가 수집된 후에 수행할 수 있습니다. 매니페스트 내 캡션은 다음과 같이 생성할 수 있습니다.

  • WebVTT 파일 수집
  • Dynamic Ingest API를 사용하여 캡션 수집
