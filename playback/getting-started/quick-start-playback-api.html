소개

Playback API에는 네 가지 작업이 있습니다.

  • 아이디로 동영상 가져오기
  • 참조 ID로 비디오 가져오기
  • ID로 재생 목록 가져오기
  • 참조 ID로 재생 목록 가져오기
Playback API는 웹 페이지 또는 모바일 앱에서 클라이언트 측 사용을 위한 것입니다. 일반적인 미디어 관리 또는 Video Cloud와 시스템의 통합을 위해 CMS API대신에.

현재 플레이어에 있는 비디오에 대한 정보만 원하는 경우 다음을 사용할 수 있습니다. mediainfo Playback API를 호출하는 대신 개체를 사용합니다.

NS mediainfo객체는 현재 플레이어에 로드된 비디오에 대한 정보를 포함하는 반면 Playback API는 모든 비디오에 대한 정보를 검색할 수 있습니다. 자세한 내용은 mediainfo의 비디오 메타데이터문서.

입증

Playback API 요청은 암호화된 정책 키 문자열을 사용하여 인증됩니다. 아래 CodePen에는 샘플 Brightcove 계정에 대한 정책 키가 포함되어 있으므로 원하는 경우 사용할 수 있습니다. 다음 중 하나를 사용하여 계정에 대한 정책 키를 얻을 수 있습니다. Policy API또는에서 Brightcove 플레이어 중 한 명 .

지침

API를 탐색하려면 아래 CodePen을 사용하십시오.

  1. 결과를 보려면 샘플 Brightcove 계정에 대해 비디오 및 재생 목록을 요청하는 것으로 시작하십시오.
  2. 다음 중 하나를 사용하여 고유한 정책 키를 가져옵니다. Policy API또는에서 Brightcove 플레이어 중 한 명 .
  3. 아래 CodePen 양식에서 계정 ID, 정책 키, 비디오 ID 및 재생 목록 ID를 자신의 값으로 바꿉니다.
  4. CSS를 수정하여 비디오 데이터의 표시를 사용자 정의하십시오.
  5. id 대신 참조 id로 동영상과 재생 목록을 가져오도록 JavaScript를 수정하십시오.
