개요

이 자습서에서는 API 수준에서 라이브 스트리밍 작업을 만드는 방법을 설명합니다. 브라이트코브 라이브 MPEG2-TS 입력을 허용합니다. MPEG2-TS 입력을 사용하는 핵심은 다음을 선택하는 것입니다. protocol MPEG2-TS를 지원하고 인코더가 제공할 수 있습니다. Live API에서 지원하는 TS 지원 프로토콜은 다음과 같습니다.

  • rtp
  • rtp-fec
  • srt

기본 입력 프로토콜은 다음과 같으므로 TS 입력을 사용하려면 이러한 프로토콜 중 하나를 명시적으로 지정해야 합니다. rtmp .

클리핑, SSAI 큐 포인트 또는 ID3 태그에 SMPTE 타임코드를 사용하려면 SEI Picture Timing 메시지의 H.264(AVC) 비디오에 삽입해야 합니다.

이러한 프로토콜은 인코더와 Live 간의 통신에 TCP가 아닌 UDP(User Datagram Protocol)를 사용하므로 방화벽/라우터를 올바르게 구성하고 인코더의 IP에서 인터넷으로 UDP 출력을 허용해야 합니다.

로컬 네트워크에 열린 포트가 있는 경우 Brightcove는 각 SRT 및 RTP 작업에 대해 포트를 동적으로 할당하며, 범위는 10000~65535입니다.

사용하는 경우rtp-fec , FEC 정보는 미디어와 다른 포트에서 전송되므로 예를 들어 포트 2000을 사용하는 경우 포트 2002 및 2004도 열어야 합니다(2D FEC를 사용하는 경우).

이 튜토리얼에서는곱슬 곱슬하다 API 요청을 만들 수 있지만불명 증 , 우편 집배원또는 대신 다른 REST 클라이언트.

인코더 구성(TS+FEC)

여기에 표시된 단계는 다음을 가정합니다. 엘리멘탈 라이브 박스이 자습서의 단계를 테스트하는 데 사용된 인코더입니다. 다른 인코더가 있는 경우 설정이 비슷해야 합니다.

이를 위한 가장 중요한 설정rtp-fec입력은 다음과 같습니다.

  • 타임코드 소스를 "시스템 시계"로 구성
  • "정방향 오류 수정"을 "열 및 행"으로 설정하고 "열 깊이" 및 "행 길이" 값을 설정합니다.
    • 컬럼 깊이: 4("열 깊이" TS 패킷마다 1개의 추가 FEC 패킷이 추가됨)
    • 행 길이: 10("행 길이" TS 패킷마다 1개의 추가 FEC 패킷이 추가됨)
    • 추가된 데이터는 다음과 같이 계산할 수 있습니다. TotalBitrate = MediaBitrate * (1/(ColumnDepth + 1) + 1/(RowLength + 1)
  • 영상 내 "타임코드 삽입" 확인
  • 권장: 프레임 속도 = 소스 팔로우

이것은 이 TS+FEC 샘플에 사용한 구성입니다.
