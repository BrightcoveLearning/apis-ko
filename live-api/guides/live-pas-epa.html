소개

라이브 작업에 대해 PAS 또는 EPA를 활성화하여 DRM에서 제공하는 보호 기능을 확장할 수 있습니다. PAS는 특정 사용자 에이전트로 재생을 제한하거나 IP 주소 변경 또는 총 사용 횟수를 제한하는 제한된 제한 세트를 제공합니다. Live의 경우 고객이 해당 사용자의 DRM CRT를 재정의할 수도 있습니다. 이러한 제한 사항은 재생 권한에 직접 지정됩니다. JWT .

EPA는 훨씬 더 포괄적인 제한 사항을 제공합니다. 이러한 제한 중 하나는 재생 권한 관리 API를 통해 구성되며 재생 권한을 사용하여 참조됩니다. id . 이것id비디오와 연결할 수 있습니다( playback_rights_id아래에 표시된 속성) 또는 JWT . 두 번째 세트에는 동시성 제한 및 장치 수와 같은 사용자 수준 제한이 포함됩니다. 이는 JWT 클레임에 직접 지정됩니다(사용자 ID [v uid주장하다]).

두 모드 모두 작업 요청에는 다음이 포함되어야 합니다. videocloud.video Video Cloud 비디오가 생성되고 라이브 작업에 연결되도록 속성. NS playback_rights_id속성은 Video Cloud 비디오에서 업데이트하여 다음을 통해 Playback Restrictions을 변경할 수 있습니다. CMS API .

당신은 또한 필요합니다 JWT 생성사용 중인 항목에 따라 PAS 또는 EPA에 적절한 청구가 포함됩니다. JWT는 모든 라이선스 서버 요청에 포함되어야 합니다. 자신의 플레이어를 구현하는 경우 이 값을 다음 중 하나에 전달해야 합니다. BCOV-Auth헤더 또는bcov_auth쿼리 매개변수(DRM 토큰 추가).

EPA/PAS로 라이브 작업 만들기

다음 새 필드는 라이브 작업에 EPA 및/또는 PAS를 추가하는 데 사용됩니다.

새로운 분야drm물체:

require_playback_token (부울)
PAS를 활성화합니다. 라이선스를 요청할 때 재생 인증 JWT가 필요합니다.
check_playback_rights (부울)
EPA를 활성화합니다. 라이선스를 요청할 때 재생 인증 JWT가 필요합니다. 동영상 또는 JWT와 연결된 재생 권한 ID를 사용하여 제한을 적용합니다.

새로운 분야videocloud.video물체:

이 필드는 CMS API를 사용하여 비디오에 추가해야 합니다. ~ 후에 비디오를 만들 때 추가할 수 없기 때문에 라이브 작업을 만듭니다.
playback_rights_id (끈)
지정된 EPA 재생 권한을 비디오와 연결합니다.
