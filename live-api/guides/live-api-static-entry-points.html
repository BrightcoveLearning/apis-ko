소개

정적 진입점(SEP) 기능을 사용하면 진입점 URL과 재생 URL을 정적 및 재사용 가능한 상태로 유지하면서 활성화 및 비활성화할 수 있는 장기 실행 라이브 작업을 허용합니다. 이 기능을 통해 고객은 자신의 시설이나 현장에서 인코더를 구성할 수 있고 고객이 라이브 채널 또는 프로그램에 대한 자체 스케줄링 로직을 생성할 수 있습니다.

DVR 및 SEP

DVR 기능은 정적 진입점에 대해 사용할 수 있지만 SEP가활성화 .

정적 진입점 생성

표준작업 생성작업이 사용되지만 정적 진입점을 추가하려면 작업 요청에 속성을 추가해야 합니다.

"static": true

이 작업의 응답 RTMP URL은 정적이며 아래 섹션에 설명된 작업으로 작업을 활성화/비활성화하여 재사용할 수 있습니다. 이 문서의 끝에 나열된 제한 사항을 읽고 이해하십시오.

최대 대기 시간

SEP 작업이 활성화된 후 인코더가 시작되지 않으면 일정 시간이 지나면 자동으로 비활성화됩니다. 규칙은 다음과 같습니다.

  • 만약event_length값이 30분보다 크면 작업은 기본적으로 30분 후에 자동으로 종료됩니다. 대기 시간을 변경하려면 지원팀에 문의해야 합니다. 자동 비활성화는 비활성화할 수 없습니다.
  • 만약event_length 값이 다음보다 작습니다. reconnect_time , 작업은 에 대해 설정된 시간에 종료됩니다. event_length .

SEP 라이브 스트림 활성화

정적 진입점 라이브 스트림을 활성화하려면 Live API에 다음 요청을 보냅니다.

정적 진입점을 수집하려면 다음을 수행해야 합니다. activate API 요청.
활성 SEP 스트림
방법 PUT
끝점 
https://api.bcovlive.io/v1/jobs/{jobID}/activate

요청 본문이 필요하지 않습니다.

SEP 라이브 스트림 비활성화

스트림을 비활성화하기 전에 항상 인코더를 중지하십시오. 비활성화가 즉시 발생하지 않을 수 있으므로 시청자가 콘텐츠 수신을 중단하는 시기를 정확하게 제어할 수 있습니다.

정적 진입점 라이브 스트림을 비활성화하려면 Live API에 다음 요청을 보냅니다.

활성 SEP 스트림
방법 PUT
끝점 
https://api.bcovlive.io/v1/jobs/{jobID}/deactivate
정적 진입점을 비활성화하면 스트림의 마지막 몇 초는 계속 재생할 수 있습니다.

요청 본문이 필요하지 않습니다.

SEP 작업 취소

정적 진입점 작업을 영구적으로 중지하려면 라이브 작업과 마찬가지로 다음을 전송하여 작업을 취소할 수 있습니다. PUT끝점을 요청/jobs/{job_id}/cancel . 참조 API 참조자세한 내용은

제한 사항

  • Live to VOD는 완료 시 스트림을 보관하도록 설정할 수 없습니다.

    대안:

    • 사용하다깎는전체 라이브 이벤트를 VOD로 보관합니다. 클리핑을 위한 창은 이벤트 후 또는 다음 SEP 활성화까지(둘 중 더 짧은 쪽) 표준 기간 동안 사용할 수 있습니다.
  • 작업 설정은 수정할 수 없습니다. 현재 SEP를 처음 생성한 설정에 연결되어 있습니다.
  • 스트림을 활성화하는 데 진입점을 사용할 수 있으려면 최대 60초가 걸릴 수 있습니다.

    모범 사례:

    SEP 작업이 실제로 준비될 때 실시간 업데이트를 받기 위해 작업 상태 변경에 대한 라이브 플랫폼의 알림을 구현합니다.

    대안:

    1. 인코더를 시작하기 최소 60초 전에 스트림을 활성화하십시오.
    2. 인코더에 내장된 재시도(Wirecast, Elemental 등)가 있는 경우 스트림을 즉시 연결할 수 있으며(일부 오류가 있을 수 있음) 인코더는 진입점이 사용 가능한 즉시 다시 연결됩니다.
  • Reconnect_time은 인코더 연결 해제 시 SEP 작업에 계속 적용됩니다. 재연결 시간에 도달하면 SEP 작업이 비활성화된 상태로 돌아갑니다.

    모범 사례:

    작업이 완료되면 항상 API를 통해 스트림을 비활성화하고 재연결 시간을 연결이 끊긴 경우 대기할 수 있는 허용 가능한 창으로 설정합니다(이 대기 기간 동안 입력 시간 비용이 발생함).

  • 각 계정에는 3개의 사용 가능한 SEP 작업만 할당됩니다. 지역별. 한계에 도달하면 다음을 수행해야 합니다. 취소새 작업을 만들기 전에 SEP 작업 중 하나를 제거하십시오.

    최대 SEP 작업 수에 도달하고 다른 작업을 생성하려고 하면 다음 오류가 반환됩니다.

    {
  "error": "You've reached the number of static entry points jobs you can create for your account"
}
  • 한 번에 5개 이상의 SEP 작업을 활성화하려고 하면 다음과 같은 오류가 표시됩니다. 
    {
  “message”:“Allocating additional streaming capacity to the region. Please retry your request.: no_streaming_capacity_in_region”,
  “statusCode”:500,
  “name”:“ApiError”,
  “isApiError”:true
}

    요청을 다시 시도하기 전에 일부 작업의 활성화가 완료될 때까지 기다려야 합니다.

노트

비활성화 후 SEP 작업 클리핑
사용자는 다음 스트림 활성화까지 또는 7일 중 더 짧은 기간 동안 클립을 만들 수 있습니다. 스트림이 다시 활성화되면 이전 녹화 세션이 서버에서 제거되고 더 이상 자를 수 없습니다.
SEP를 사용할 때 stream_start_time 필드의 타이밍은 무엇입니까?
SEP가 활성화되면 시작 시간으로 간주됩니다.
