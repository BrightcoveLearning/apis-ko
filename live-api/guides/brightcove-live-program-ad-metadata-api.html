소개

광고 메타데이터(ad_server_data) 스키마는 중첩 요소가 없는 플랫 JSON 구문을 지원합니다(키-값 쌍만 해당). API를 통해 설정된 광고 서버 데이터는 데이터를 덮어쓰거나 재설정할 때까지 업데이트 후 수신되는 모든 광고 시간에 사용됩니다. 고객은 실시간 스트림 중에 언제든지 데이터를 업데이트 및/또는 재설정할 수 있습니다.

이 메타데이터는 Steam 활성화 간에 지속되지 않습니다. 스트림 활성화 간에 유지해야 하는 광고 메타데이터는 SSAI 애플리케이션 광고 구성에서 설정해야 합니다. API를 통한 데이터 세트는 광고 서버 및 서버 측 비콘 교체에서 매크로로 사용하도록 구성할 수 있습니다.

광고 프로그램 메타데이터는 맞춤 재생 목록 태그를 통해 SSAI 시스템에 전달되고 SSAI 시스템의 기존 DAP(동적 광고 매개변수) 우선 순위를 따릅니다. 큐포인트에서 전송된 광고 서버 데이터는 작업에 할당된 메타데이터보다 우선합니다. 작업의 큐포인트 데이터와 광고 메타데이터 간에 키 충돌이 발생하면 큐포인트 값이 더 높은 우선순위로 간주되어 병합된 컬렉션에 사용됩니다.

큐포인트에 제공된 광고 메타데이터는 해당 큐포인트에만 사용되며 후속 큐포인트 삽입에는 유지되지 않습니다. 프로그램 광고 메타데이터 API를 통해 설정된 광고 메타데이터만 업데이트되거나 재설정될 때까지 이후에 수신되는 모든 광고 시간에 대해 유지됩니다.

끝점

엔드포인트에 대한 자세한 내용은 다음을 참조하세요. 라이브 API 참조 .

추가/업데이트

라이브 프로그램 광고 메타데이터를 추가하거나 업데이트하려면POST요청:

https://api.bcovlive.io/v1/jobs/{job_id}/cuepointdata

필드 추가/업데이트
들 유형 설명 예시
ad_server_data 물체 키/값 쌍을 포함하는 개체입니다. 
{
  "ad_server_data": {
  	"one":"won",
  	"two":2,
  	"three":false
  }
}

삭제

라이브 프로그램 광고 메타데이터를 삭제하려면DELETE요청:

https://api.bcovlive.io/v1/jobs/{job_id}/cuepointdata
