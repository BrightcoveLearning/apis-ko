소개

작업을 완료한 후 스케줄러는 http 메서드를 사용하여 구성된 끝점에 알림을 보냅니다. 우편수행된 작업 및 성공/실패에 대한 세부 정보.

예정된 작업에 대한 사전 알림을 구성할 수도 있습니다. 참조라이브 API 참조알림 URL 및 설정 지정에 대한 자세한 내용과 SEP 작업의 시작/중지 시간 예약 및 클립 예약의 차이점을 확인하십시오.

스케줄러 시작/중지
  • 알림 URL은 요청 본문 개체의 최상위 수준에서 정의됩니다.
  • 알림은활성화그리고비활성화객체(알림을 보낼 이벤트 전의 시간(초))

예시

{
	"job_id": "your Job ID",
	"activation": {
		"time_utc": 1612274100,
		"notification": 600
	},
	"deactivation": {
		"time_utc": 1612281600,
		"notification": 600
	},
	"description": "My Tuesday broadcast",
	"notification_url": "http://example.com/live/jobstartstop/callbacks"
}
클립 스케줄러
  • 알림 URL은 요청 본문 개체의 최상위 수준에서 정의됩니다.
  • 알림은클립객체(작업이 알림 전송을 시작하기 전의 시간(초))

예시

{ "job_id": "작업 ID", "설명": "내 예정된 클립",  "notification_url": "http://example.com/live/clip/callbacks" , "클립": { "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879800, "videocloud": { "video": { "name": "My Scheduled Clip" } } }, "time_utc": 1612879800,  "공고": 600 } }

알림 본문

서버가 사용자에게 알림을 보낼 때알림_url , 요청에는 JSON 본문이 포함됩니다. 필드는 아래 표에 자세히 설명되어 있습니다.

알림 본문 필드
동작 작업 이름입니다. 보다행위
워크플로 작업 완료 후 워크플로가 업데이트되었습니다. 구조는 워크플로 유형에 따라 다릅니다.
오류 작업을 완료하는 동안 오류가 발생한 경우 이 필드에 오류가 포함됩니다. 값은 작업 유형에 따라 다릅니다.

행위

다음 표에서는 워크플로 유형별 작업을 자세히 설명합니다.

행위
이름 워크플로 유형 설명
notify-activate jobstartstop 귀하의 작업이 곧 활성화될 것임을 미리 알려드립니다.
activate jobstartstop 작업 활성화가 완료되었습니다. 만약에 "오류"이 있는 경우 활성화에 실패했으며 비활성화가 취소됩니다.
notify-deactivate jobstartstop 귀하의 작업이 곧 활성화될 것임을 미리 알려드립니다.
deactivate jobstartstop 작업 비활성화가 완료되었습니다. 만약에 "오류"있는 경우 비활성화에 실패했습니다.
notify-clip clip 귀하의 클립이 곧 요청될 것임을 알려드립니다.
clip clip 클립 요청이 완료되었습니다. 만약에 "오류"있는 경우 요청이 실패했습니다.
