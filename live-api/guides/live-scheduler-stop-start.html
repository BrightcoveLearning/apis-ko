요구 사항

  • 계정의 Live API 키입니다.
  • 라이브 계정에 스케줄러 재정의가 활성화되어 있어야 합니다.
  • 기존 라이브 작업 "정적": 참 (보다정적 진입점 작업자세한 내용은).
라이브 모듈에서 활성화 및 비활성화를 예약할 수도 있습니다. 관제실에서 라이브 이벤트 자동 시작/중지자세한 내용은

스케줄러 만들기

에 시작하는 2시간 이벤트를 예약한다고 가정해 보겠습니다. 2021-04-02T14:00:00+00:00 .

타임스탬프는 초 단위의 유닉스 타임스탬프 형식의 UTC여야 합니다. 당신이 사용할 수있는 https://www.unixtimestamp.com/변환을 위해.

활성화 시간 제한이 설명되어 있습니다. 여기예약된 활성화에 계속 적용되므로 원하는 시작 시간보다 최소 60초 전에 시간을 예약하는 것이 좋습니다.

이 예에서는 예정된 시작 시간 5분 전을 선택합니다.

시작: 2021-04-02T13:55:00+00:00 = 1617386100 중지: 2021-04-02T16:00:00+00:00 = 1617393600

스케줄러를 구현하려면 다음을 수행하십시오. POST요구:

https://api.bcovlive.io/v1/scheduler/jobstartstop

헤더

Content-Type: application/json
		X-API-KEY: {your API Key}

요청 본문

{
			"job_id": "your Job ID",
			"activation": {
				"time_utc": 1617386100
			},
			"deactivation": {
				"time_utc": 1617393600
			},
			"description": "My Wednesday broadcast"
		}

스케줄러 업데이트

업데이트할 수 있습니다. 활성화및/또는비활성화작업이보류 중상태 ( "상태": "보류 중" ). 이렇게 하려면놓다요청:

https://api.bcovlive.io/v1/scheduler/jobstartstop/{workflow_id}

헤더는 다음과 동일합니다. 우편위에 표시된 요청.

요청 본문

요청 본문 개체에활성화및/또는비활성화필드.

예를 들어, 예정보다 30분 일찍 이벤트를 중지하려면:

요청 본문

{
	"deactivation": {
		"time_utc": 1617391800
	}
}

샘플 응답

 
{
	"account_id": "1752604059001",
	"activation": {
		"state": "pending",
		"time_utc": 1617386100
	},
	"deactivation": {
		"state": "pending",
		"time_utc": 1617391800
	},
	"description": "My Wednesday broadcast",
	"job_id": "15981d93877e490185e6ebc9af2c79a3",
	"type": "jobstartstop",
	"workflow_id": "d44cf29f4f184757ac0995beb8a0097e"
}

스케줄러 취소

다음으로 활성화 및 비활성화를 취소할 수 있습니다. 삭제요구:

https://api.bcovlive.io/v1/scheduler/jobstartstop/{workflow_id}

활성화 후에 이 작업을 수행하여 비활성화를 취소할 수도 있습니다.

헤더

유일한 필수 헤더는 다음과 같습니다.

X-API-KEY: your API Key

요청 본문

이 요청에 대한 요청 본문이 없습니다.

샘플 응답

 
{
		"account_id": "1752604059001",
		"activation": {
			"state": "cancelled",
			"time_utc": 1617386100
		},
		"deactivation": {
			"state": "cancelled",
			"time_utc": 1617391800
		},
		"description": "My Wednesday broadcast",
		"job_id": "15981d93877e490185e6ebc9af2c79a3",
		"type": "jobstartstop",
		"workflow_id": "d44cf29f4f184757ac0995beb8a0097e"
	}

알림

워크플로를 생성할 때 선택적으로 알림을 구성할 수 있습니다. 서비스에 대한 URL을 제공해야 합니다. 우편에게. 스케줄러가 활성화 및 비활성화 호출을 할 때 알림을 보내드립니다. 알림을 받도록 구성할 수도 있습니다. N활성화 및/또는 비활성화 전 초. 참조라이브 스케줄러 알림자세한 사항은.

샘플 요청 본문

 
{
		"job_id": "your Job ID",
		"activation": {
			"time_utc": 1612274100,
			"notification": 600
		},
		"deactivation": {
			"time_utc": 1612281600,
			"notification": 600
		},
		"description": "My Tuesday broadcast",
		"notification_url": "http://example.com/live/jobstartstop/callbacks"
	}
