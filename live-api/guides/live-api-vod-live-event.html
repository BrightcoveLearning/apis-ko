Live to VOD는 암호화된 스트림과 함께 사용할 수 없으며 라이브 작업 이벤트를 생성할 때 오류를 반환합니다.

소개

짧은 라이브 스트리밍 이벤트의 경우 이벤트를 VOD(주문형 비디오)로 녹화할 수 있습니다. 비디오 클라우드이벤트 종료 후 계정. VOD에 대한 일부 구성을 추가하면 쉽게 이 작업을 수행할 수 있습니다. 당신의 라이브 작업을 생성 .

Fast Live to VOD를 사용하면 세그먼트 정확도로 클립을 빠르게 게시할 수 있으며 고객에게 완전히 투명한 백엔드 프로세스가 클립 경계를 정리하고 다시 게시할 수 있습니다.

그 결과 원하는 클립 경계와 함께 자동으로 다시 게시되는 클립을 사용하여 출시 시간을 단축할 수 있습니다.

중요한 :

  • 라이브 스트리밍 이벤트가 완료되면 인코더를 중지하지만 ~하지 않다취소작업 . 작업을 취소하기 전에 VOD 클립이 생성될 때까지 기다리십시오. 그렇지 않으면 성공적으로 생성되지 않습니다.
  • VOD 생성은reconnect_time라이브 작업이 만료되었습니다.
  • 라이브 이벤트 기간이 24시간보다 긴 경우 VOD에는 마지막 24시간만 캡처됩니다.

빠른 VOD 옵션

빠른 VOD 옵션을 사용하면 라이브 이벤트의 VOD 버전을 더 빨리 얻을 수 있습니다. Fast Live to VOD는 (transmux를 사용하여) 라이브 변환을 게시합니다. 정확한 세그먼트 .

더output빠른 VOD 필드는 다음과 같습니다.

  • live_to_vod로 설정true
  • mode (빠른 VOD에 필요) - 빠른 VOD의 경우 값은instant
  • playlist_label (선택사항) - 인스턴트 모드 VOD에만 적용됩니다. 라이브 작업에서 VOD 출력을 만드는 데 사용할 재생 목록을 나타냅니다. 라이브 작업 생성 시 사용자 지정 재생 목록이 정의된 경우에만 필요합니다.

구현

라이브 작업을 생성할 때

VOD에 라이브를 구현하는 가장 간단한 방법은"live_to_vod": true다음 중 하나에서 작업을 생성할 때 요청 본문의 필드

  • 최상위videocloud개체(VOD를 원격 자산으로 만들기 위해)
  • Video Cloud 또는 S3로 출력하는 출력 객체

보내기비디오 클라우드

2가지 방법이 있으니 참고하세요videocloud개체는 새 라이브 작업을 생성할 때 사용됩니다.

  1. 포함videocloud에서작업 수준 필드에서 동영상을 생성합니다. 비디오 클라우드라이브 스트림을 원격 자산으로 사용하고 Video Cloud 라이브러리에 VOD를 생성하는 계정: 
    {
    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 2400,
    "live_dvr_sliding_window_duration": 30,
    
      "videocloud": {
        "video": {
          "name": "test_live_stream"
        },
        "live_to_vod": true
        }

  2. 포함하는videocloud속성의 개체outputs클립 또는 전체 VOD 개체는비디오 클라우드클립에 대한 비디오 - 클립이 수집됩니다. 비디오 클라우드 VOD 라이브러리의 일부가 됩니다.

		"video": {
			"name": "test_live_stream"
		}

아직 Video Cloud 계정 설정에 대한 자격 증명이 없는 경우 새 자격 증명을 추가할 수 있습니다. API를 통해 .

비디오 메타데이터에 대해 나열된 모든 필드를 포함할 수 있습니다. 비디오 만들기 CMS API의 작동.

수집 프로필은 VOD용으로 생성될 변환을 정의합니다. 을위한valid-ingest-profile-name , 계정에 대한 사용자 지정 또는 표준 수집 프로필의 이름을 대체할 수 있습니다. 보유하고 있는 표준 프로필은 계정이 Dynamic Delivery를 사용할 수 있는지 여부에 따라 다릅니다. 참조동적 전달 및 컨텍스트 인식 인코딩을 위한 표준 수집 프로필아니면 그레거시 트랜스코딩을 위한 표준 수집 프로필 .

참고로profile필드는 선택 사항입니다. 프로필을 지정하지 않으면 계정의 기본 프로필이 사용됩니다.

S3로 보내기

이벤트의 VOD 버전을 S3 버킷에 저장하려면 강조 표시된 버전을 포함하십시오. 산출아래에.

{
  "live_stream": true,
  "region": "us-west-2",
  "reconnect_time": 600,
  "ad_insertion": false,
  "outputs": [
    { OTHER_JOB_OUTPUT_OBJECT },
    { OTHER_JOB_OUTPUT_OBJECT_2 },
    {
      "url":"s3://YOUR_BUCKET/path/filename.mp4",
      "credentials": "YOUR_CREDENTIAL_LABEL",
      "notifications": [
        {
          "url": "http://httpbin.org/post?vodStateChange"
        },
        {
          "url": "http://httpbin.org/post?vodFinished",
          "event": "output_finished"
        }
      ]
    }
  ]
}
S3 버킷의 유효한 사용자 이름+비밀번호에 대한 자격 증명 레이블은 Live 계정이 설정될 때 생성되어야 하며 자격 증명 레이블이 귀하에게 전송됩니다. 없으시다면 부탁드립니다 Brightcove 지원에 문의 .

VOD를 개방형 비디오 클립으로 만들기

라이브 이벤트에 대한 VOD를 얻는 다른 방법은 VOD 클립 만들기종료 시간 없이. 자세한 내용은 클리핑 문서를 참조하십시오.

제한 사항

  • 만약 너라면중지(취소) VOD가 생성되기 전의 라이브 작업은 생성되지 않습니다. 대신 인코더를 중지하여 VOD 생성을 시작하고 작업을 중지하기 전에 완료될 때까지 기다리십시오.
  • 을위한정적 진입점스트림오직 , Live to VOD는 완료 시 스트림을 보관하도록 설정할 수 없습니다.

    대안:

    • 사용하다깎는전체 라이브 이벤트를 VOD로 보관합니다. 클리핑을 위한 창은 이벤트 후 또는 다음 SEP 활성화까지(둘 중 더 짧은 쪽) 표준 기간 동안 사용할 수 있습니다.
