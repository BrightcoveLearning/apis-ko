개요

클립은 라이브 스트림에서 추출한 비디오입니다. S3 버킷, FTP 사이트 또는비디오 클라우드계정. 클립은 MP4 비디오로 생성되며 모든 경우에 대상으로 전송됩니다. 의 경우비디오 클라우드 , MP4는 수집 시스템에 의해 트랜스코딩되며 비디오에 대해 생성되는 변환 종류는 사용되는 수집 프로필에 따라 다릅니다.

클립에 대한 정의는 다음을 사용하여 생성됩니다. /vods끝점.

클립은 여러 가지 방법으로 만들 수 있습니다.

  • 와 함께stream_start_timecode및/또는stream_end_timecode라이브 스트림 이벤트에 대한 SMPTE 타임코드에 정의됨 - 이를 위해서는 인코더가 타임코드 정보를 전송해야 합니다.
  • 와 함께start_time및/또는end_time시작 시간을 기준으로 정의됨( stream_start_time ) 전체 라이브 스트림 이벤트의
  • 와 함께start_time및/또는end_time Epoch(Unix) 시간으로 정의됨(초)
  • 와 함께duration
  • VOD API ~ 할 수있다와 함께 사용하다암호화 또는 DRM 보호일자리. 현재 Live Module은 이를 지원하지 않지만 향후 릴리스에서 지원할 예정입니다.

노트

  1. 가능한 한 빨리 클립을 사용할 수 있도록 하기 위해 먼저 정확한 세그먼트 클립을 만든 다음 사용 가능한 즉시 정확한 프레임 클립으로 교체합니다.
  2. 지정하는 경우duration , 결과 클립은 다음과 같습니다.
    • 작업이 활성 상태이고 여전히 활성 상태인 경우: (요청 시간 - 기간) ~ (요청 시간)
    • 작업이 완료된 경우: ( finished_at - 기간) ~ ( finished_at )
  3. 둘 다 지정하면start_time그리고end_time :
    • 작업이 활성 상태이고 여전히 활성 상태인 경우: Epoch 시간 창이 완전히 범위 내에 있는 한created_at요청 시간, 클립이 만들어집니다
    • 작업이 완료된 경우: Epoch 시간 창이 완전히 범위 내에 있는 한created_at그리고finished_at , 클립이 만들어집니다
  4. 다음을 사용하는 라이브 스트림 클립싸이광고를 포함하지 않습니다.
  5. 클립은 이벤트 후 최대 7일까지 생성할 수 있습니다. 을위한 9월 , 다음 활성화 또는 7일(둘 중 더 짧은 날짜)까지 생성할 수 있습니다.
  6. VOD API는 스트림에 있는 것 이외의 콘텐츠를 추가하지 않습니다. 300초 길이의 라이브 스트림에 350을 지정하면 출력은 300초 길이가 됩니다.
  7. 라이브 스트림은 방송된 그대로 저장되고 이벤트가 끝난 후 즉시 그리고 7일 동안 사용할 수 있기 때문에 클리핑이 작동하기 위해 DVR 지원 라이브 스트림을 사용할 필요가 없습니다.
  8. Brightcove Live 클리핑은 최고 해상도 출력과 동일한 해상도의 클립만 생성합니다. 소스 입력 해상도와 일치하지 않습니다(최고 해상도 출력과 동일하지 않은 경우).

클립을 여러 대상으로 보낼 수도 있습니다.

  • NS 비디오 클라우드계정
  • FTP 서버
  • S3 버킷

클립을 지정하면 출력 ~해야하다포함하다어느 하나 NS url목적지또는 NS videocloud비디오 생성 및 클립 수집을 자세히 설명하는 개체비디오 클라우드 .

참고: 클립라이브 스트림이 실행되는 동안 생성될 수 있습니다. . 이렇게 하려면 Epoch 시간 또는 기준 시간으로 클립의 시작 및 종료 시간을 정의해야 합니다. 시작라이브 스트림의 시간.

예 1: 스트림 시작과 관련된 시작/종료 시간

요청 본문에는 시작 및 종료 시간과 클립을 보낼 위치에 대한 세부 정보가 포함됩니다. 다음은 스트림의 3분 분량의 클립을 생성하여 다음으로 보내는 샘플 요청 본문입니다. 비디오 클라우드계정:

{
  "live_job_id":"PUT-LIVE-JOB-ID-HERE",
  "outputs":[
    {
      "label": "60 secs by stream from min 2 to min 3",
      "stream_start_time": 120,
      "stream_end_time": 180,
      "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL",
      "videocloud": {
        "video": {
        "name": "One Minute Clip",
        "tags": ["live", "clip"]
        },
          "ingest": {
            "capture-images": true
        }
      }
    }
  ]
}

이 예에서는 1분 길이의 클립을 만들어 다음으로 전송합니다. 비디오 클라우드 . 클립 이름과 몇 개의 태그를 지정하지 않고인제스트 프로필계정 기본값이 사용되도록 재변환하고비디오 클라우드트랜스코딩 중에 클립에서 썸네일 및 포스터 이미지를 캡처합니다.

예 2: Epoch 시간의 시작/종료 시간

요청 본문에는 Epoch 시간의 시작 및 종료 시간과 클립을 보낼 위치에 대한 세부 정보가 포함됩니다. 다음은 스트림의 3분 분량의 클립을 생성하여 다음으로 보내는 샘플 요청 본문입니다. 비디오 클라우드계정:

이 예에서는 특정 Epoch 시간(이 경우 2018년 1월 22일 08:24:54 GMT)에 1분 길이의 클립을 생성합니다.

예 3: 스트림 시작을 기준으로 시작 시간이 포함된 기간

요청 본문에는 지속 시간 및 stream_start_time과 클립을 보낼 위치에 대한 세부 정보가 포함됩니다. 다음은 스트림의 3분 분량의 클립을 생성하여 다음으로 보내는 샘플 요청 본문입니다. 비디오 클라우드계정:

요청 본문 - S3

요청 본문에는 시작 및 종료 시간과 클립을 보낼 위치에 대한 세부 정보가 포함됩니다. 다음은 스트림의 3분 분량의 클립을 생성하여 S3 버킷으로 보내는 샘플 요청 본문입니다.

{
  "live_job_id":"{{job_id}}",
  "outputs":[
    {
      "label": "last_30",
      "duration": 30,
      "url": "s3://YOUR_BUCKET_NAME/file_name.mp4",
      "credentials": "s3-credentials",
      "notifications": ["http://myserver.com/api/notification_listener?type=jvod"]
    }
  ],
}

이 예에서는 30초 길이의 클립을 생성하여 S3 버킷으로 보냅니다. 클립의 파일 이름이 포함된 버킷 URL과 저장된 S3 버킷 자격 증명의 이름인 문자열을 제공합니다. 자격 증명은 Brightcove 지원에서 계정에 대해 설정할 수 있습니다.
