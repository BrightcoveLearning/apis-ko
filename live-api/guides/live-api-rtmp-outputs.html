일반 정보

RTMP 출력을 사용하려면 다음 사항에 유의하십시오.

  • 당신은 구매해야합니다이벤트 시간 , 또는 모든 디지털 마케팅 패키지
  • RTMP 출력 시간은 이벤트 시간에 대해 청구됩니다.
  • 라이브 작업당 5개의 출력으로 제한됩니다.
  • 다음을 정의할 수 있습니다. label여기서 레이블은 스트림에 대해 선택된 출력 변환 중 하나를 선택할 수 있습니다. 만약label설정되지 않은 경우 출력은 기본적으로 입력 소스로 설정됩니다.
  • 표준 작업 및 반복 이벤트(SEP)와 함께 사용할 수 있습니다.
  • 이벤트 또는 24/7 스트림에 사용할 수 있습니다.
  • 모든 라이브 스트림과 독립적으로 시작 및 중지할 수 있습니다.
  • RTMP 출력 기능이 구현되기 전에 생성된 SEP 작업의 경우 이 기능이 완전히 작동하지 않습니다. API에서는 RTMP 출력이 실제로 생성되더라도 실패를 표시합니다. 최상의 결과를 얻으려면 RTMP 출력 기능을 사용할 새 SEP 작업을 만드십시오.
  • 표준 작업(비SEP)과 함께 RTMP 출력을 사용하는 경우 RTMP 출력 스트림은 작업이 종료될 때 종료됩니다.
  • RTMP 출력 ~하다지원하다rtmps url이며 소셜 미디어 플랫폼에서 지원하는 경우 사용하는 것이 좋습니다. 또한 2019년 5월 1일부터 Facebook은 RTMPS만 허용합니다.
  • NS RTMP URL에 대해 현재 지원되는 형식만 ~이다rtmps://host:port/application/streamName .

RTMP 출력 생성

먼저 평소와 같이 라이브 작업을 생성한 다음 RTMP 엔드포인트 생성을 위한 추가 요청을 해야 합니다.

참고: 작업 활성화/비활성화와 같은 특정 상황에서 이 API 호출이 오류를 반환할 수 있습니다. 이 경우 일시 중지 후 다시 시도하십시오.

끝점

	https://api.bcovlive.io/v1/jobs/{jobId}/rtmpouts

방법

POST

요청 본문

	{
		"url": "rtmps://host:port/application/streamName",
		"output_label": "480p"
	}
요청 본문 필드
분야 명 필수의 유형 설명
url 예 끈 RTMP 출력용 URL - 예rtmp://live-api-s.facebook.com:80/rtmp/my_stream
output_label 아니요 끈 RTMP 출력의 소스로 사용할 작업 출력의 레이블입니다. RTMP 입력을 대신 사용하려면 이 매개변수를 생략하십시오.

샘플 응답

	{
		"host": "live-api-s.facebook.com",
		"port": 80,
		"application": "rtmp",
		"streamName": "my_stream",
		"rtmp_out_id": "asdfg-lkjh"
	}
