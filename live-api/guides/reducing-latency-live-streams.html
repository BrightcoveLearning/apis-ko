소개

대기 시간은 실제 생활에서 무언가가 녹화되는 시점과 비디오 플레이어에서 보이는 시점 사이의 시간 지연입니다. 물론 소스에서 인터넷을 통해 시청자 플레이어에게 신호를 전송하는 데 약간의 실시간이 필요합니다. 지연은 온라인 비디오가 세그먼트라고 하는 신중한 청크로 전송된다는 사실로 인해 악화됩니다.

비디오 세그먼트의 길이를 줄이면 대기 시간을 줄일 수 있으며 다음 섹션에서는 이를 수행하는 방법을 자세히 설명합니다. 그러나 중단 없는 재생을 보장하기 위해 플레이어는 현재 재생 위치보다 먼저 몇 개의 비디오 세그먼트를 다운로드하여(이를 버퍼링이라고 함) 필요할 때 준비할 수 있도록 절충점이 있다는 점에 유의하십시오. 세그먼트 길이를 줄이면 플레이어가 미리 가져오는 비디오 데이터가 줄어들고 플레이어가 추가 세그먼트를 버퍼링하는 동안 재생이 일시 중지될 가능성이 높아집니다.

추가 제한 사항

감소된 지연 시간을 사용하는 라이브 스트림에는 다음 제한 사항이 적용됩니다.

  • SSAI는 ~ 아니다지원
  • DRM은 ~ 아니다지원
  • DVR 창은 다음보다 작아야 합니다. 5 분
  • 이상 4번역

대기 시간 감소 라이브 프로필 생성

대기 시간을 줄이려면 사용자 지정 라이브 프로필을 만들어야 합니다. 아래 단계를 따르세요.

  1. 스튜디오에 로그인
  2. 이동관리자 > 수집 프로필 :
  3. 다음 중 하나를 선택하십시오. 살다이하의 프로필 수집 4변환:
  4. 프로필을 열려면 링크를 클릭하십시오
  5. 클릭복제하다이 프로필을 기반으로 새 프로필을 만드는 버튼
  6. 새 프로필에서 변경name "낮은 대기 시간 프로필"(또는 원하는 이름)
  7. 각각에 대해renditions , 변경segment_seconds에게2세그먼트의 길이를 2초로 줄이려면

Live API 사용

대기 시간을 줄이고 Live API를 사용하여 라이브 작업을 생성하는 경우 평소와 같이 작업을 생성하되 다음을 포함합니다. 4회 이하 , 설정segment_seconds각 변환에 대해2 . 또한 에 나열된 기타 제한 사항을 준수해야 합니다. 소개 .
