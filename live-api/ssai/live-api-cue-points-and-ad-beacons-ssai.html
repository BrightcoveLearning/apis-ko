개요

서버 측 광고 삽입 (싸이)라이브 스트리밍 이벤트 동안 지정된 시간에 광고를 표시할 수 있습니다. 일반 정보는 다음을 참조하십시오. 라이브 API: 서버 측 광고 삽입 (싸이)문서.

  • 사용하기 전에싸이 , 계정에 대해 활성화되어야 합니다. 자세한 내용은 계정 관리자에게 문의하세요.

큐 포인트

광고 시간은 다음 두 가지 방법으로 지정할 수 있는 큐 포인트에 의해 실행됩니다.

  • 인코더에서 Brightcove로 전송
  • 를 통해 생성된 즉각적인 큐 포인트라이브 API

인코더에서

Brightcove 라이브 전달 시스템은 인코더가 AMF 형식으로 제출한 큐 포인트를 해석할 수 있습니다.

  AMFDataList
  [0]:onCuePoint
  [1]:{Obj[]:
    time: 1.9889, //Difference from PTS of THIS packet to the first PTS of the 1st video frame in the adbreak
    name: "scte35",
    type: "event",
    ad_server_data: "eyJrZXkiOiJ2YWx1ZSJ9",	// optional introduced by Brightcove. It is a base64 encoded json map of parameters e.g. {"key":"value"}
    parameters: {Obj[]:
      type: "avail_in",
      duration: 12.0
    }
  }

노트:

  • 오직avail_in유형 큐 포인트는 현재 RTMP 입력에서 지원됩니다.
  • SCTE-35 큐 포인트는 RTP 및 SRT 입력에서 지원됩니다.
참조아래 부록 Elemental 인코더에 대한 샘플 큐 포인트 설정의 스크린샷.

수동 큐 포인트 삽입

다음을 사용하여 즉각적인 큐 포인트를 만들 수 있습니다. 라이브 API를 보내서POST요구:

방법 POST
URL https://api.bcovlive.io/v1/jobs/Job_ID/cuepoint
헤더 X-API-KEY: your API KEY

다음을 지정하는 요청 본문을 포함합니다.

들 유형 설명
duration 정수 휴식 시간(초)입니다.

삽입되는 큐 포인트의 지속 시간은세그먼트 길이의 최소 2배일에. 참조지속 시간 예 .
timecode SMPTE 형식 선택 과목: SMPTE 형식의 타임코드, HH:MM:SS:FF (FF = 프레임), 변수 세트(키/값 쌍)가 adServer에 전달되어야 하는 시기를 지정합니다.

생략하면 큐 포인트가 즉시 삽입됩니다.

timecode 속성을 사용하는 경우 인코더는 SMPTE 형식( HH:MM:SS:FF )에 저장된 타임코드tc재산을 통해OnFI . 타임코드는 라이브 스트림의 시작 부분부터입니다.
ad_server_data 물체 선택 과목: 전달하는 키/값 쌍은 사용 중인 광고 서버에 따라 다릅니다. 자세한 내용은 광고 서버 설명서 및광고 매크로를 사용하여 광고 타겟팅부분.

기간 예

삽입되는 큐 포인트의 지속 시간은세그먼트 길이의 최소 2배일에.

예를 들어 10초 큐 포인트직장에서"segment_seconds"=4 , 잘 작동합니다. 그러나 다음을 사용하여 작업에 동일한 큐 포인트를 삽입하면"segment_seconds"=6다음 오류가 발생합니다.

  "error": "The parameter duration should be greater than
    or equal to (2 * target duration) of the job"
 

샘플 요청 본문

  {
    "duration": 30,
    "timecode": "15:50:49:16",
    "ad_server_data" : {
    "adbreakid": 12312
    "breaktheme": "fitness"
    }
  }

노트

  1. Wirecast 및 OBS와 같은 소프트웨어 인코더는 다음을 통한 전송 타임코드를 지원하지 않습니다. OnFI RTMP 스트림의 패킷
  2. Elemental 하드웨어 인코더는 다음을 통해 전송 타임코드를 지원합니다. OnFI RTMP 스트림의 패킷

비콘

비콘은 재생 시 광고 재생 여부와 재생 횟수를 추적하기 위해 타사 분석으로 전송되는 데이터 포인트입니다. 이 섹션에서는 다음을 사용하여 설정할 수 있는 비콘 유형을 살펴보겠습니다. 라이브 API , 데이터를 제공하는 데 사용할 수 있는 변수. 다음 섹션에서는 비콘 세트를 만들고 관리하는 데 사용하는 API 요청에 대해 자세히 설명합니다.

SCTE-35 광고 변수

NS 케이블 통신 엔지니어 협회(SCTE)라이브 스트림에 대한 동적 광고 삽입 표준을 정의합니다. 다음 표에는 SCTE-35 광고 구성 변수가 요약되어 있습니다.

SCTE-35 광고 매개변수는 현재 FEC(순방향 오류 수정)가 있거나 없는 RTP(실시간 전송 프로토콜) 및 TS(전송 스트림)를 기반으로 하는 SRT(보안 신뢰할 수 있는 전송) 입력 소스에서 모두 지원됩니다. FEC는 RTP에 적극 권장됩니다.

이러한 광고 매개변수는 RTMP(실시간 메시징 프로토콜)를 사용하는 입력 스트림에서 사용할 수 없습니다.
SCTE-35 광고 구성 변수
변하기 쉬운
설명
scte35_eventID
SCTE35 메시지와 함께 전달된 고유한 이벤트 ID입니다.
scte35_programID
SCTE35 메시지와 함께 전달된 고유한 프로그램 ID입니다.

광고에 사용할 수 있는 특정 연결 시간의 ID로 SCTE35 메시지를 통해 보냅니다.

SCTE35 메시지와 함께 전달된 프로그램의 90kHz 클록 틱으로 표시되는 광고 시간의 중단 시간입니다.
scte35_spliceTime
SCTE35 메시지와 함께 전달된 프로그램의 90kHz 클록의 틱 측면에서 광고 시간을 위한 스플라이스 시간.

      "beacon_url": "https://myserver.com/beaconRX2/play",
      "beacon_type": "Play"
      }]
  }]

id로 설정된 비콘 가져오기

ID로 설정된 단일 비콘을 검색하려면GET요구:

방법 GET
URL https://api.bcovlive.io/v1/ssai/beaconsets/beaconset/beacon_set_id
헤더 X-API-KEY: your API KEY

샘플 응답

  {
      "account_id": "User account ID",
      "beacon_set_id": "Beacon set ID",
      "beacon_urls": [{
        "beacon_type": "Load",
        "beacon_url": "https://myserver.com/beaconRX2/load"
    },
    {
      "beacon_type": "Play",

    }]
  }

방법 DELETE
URL https://api.bcovlive.io/v1/ssai/beaconsets/beaconset/beacon_set_id
헤더 X-API-KEY: your API KEY

샘플 응답

  {
    "beacon_set_id": "Beacon set ID",
    "deleted": true
  }
