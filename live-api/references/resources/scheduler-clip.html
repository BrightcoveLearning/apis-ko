클립

Clip 끝점을 사용하여 기존 라이브 작업의 클립 이벤트를 예약할 수 있습니다. 지정된 시간에 당사 시스템은 귀하를 대신하여 클립 작업을 시작하는 데 필요한 API를 호출합니다.

요구 사항

  • 계정의 Live API 키입니다.
  • 라이브 계정에 스케줄러 재정의가 활성화되어 있어야 합니다.

클립 워크플로 만들기

클립을 생성할 때 클립 시간 경계를 모두 지정해야 합니다( 산출필드) 및 예약된 시간( time_utc필드)를 사용하여 클립을 요청합니다.

NS 산출요청 본문의 필드는 동일한 모든 필드를 지원합니다. 출력필드는 표준 VOD 끝점에 있지만 예약된 클립은 목록이 아닌 단일 출력만 지원합니다. 보다 VOD 클리핑 지원 문서사용 가능한 필드에 대한 자세한 내용은

일반적으로 다음을 설정합니다. time_utc클립의 끝 경계와 일치하도록 합니다. 예약된 시간은 클립의 끝 경계 이후여야 합니다. 그렇지 않으면 예약된 시간에 클립 요청이 실패합니다.

타임스탬프 time_utc필드는 초 단위의 유닉스 타임스탬프 형식의 UTC여야 합니다. 당신이 사용할 수있는 https://www.unixtimestamp.com/변환을 위해.

에 시작하는 15분짜리 클립을 예약한다고 가정해 보겠습니다. 2021-02-09T13:55:00+00:00 . 우리는 사용할 수 있습니다 output.start_time그리고 output.end_time클립 경계를 유닉스 시간(초)으로 지정합니다. (다시, 참조 VOD 클리핑 지원 문서사용 가능한 필드에 대한 자세한 내용은).

시작 시간: 2021-02-09T13:55:00+00:00 = 1612878900 종료 시간: 2021-02-09T14:10:00+00:00 = 1612879800

스케줄러도 클립 요청을 예약하는 데 시간이 필요하므로 다음과 같은 값을 사용할 수 있습니다. end_time초 단위의 유닉스 시간이기도 하기 때문입니다.

우리는 또한 클립에 대한 Videocloud 비디오를 만들고 싶으므로 이를산출 . 워크플로를 예약할 때 Videocloud 비디오가 생성되며 해당 ID가 응답에 반환되므로 클립이 만들어지기 전에 비디오 ID를 알 수 있습니다. 나중에 클립 요청을 취소하거나 예정된 시간에 실패하면 Videocloud Video는 계정에서 제거되지 않습니다.

워크플로를 만들려면 다음 요청을 수행합니다.

POST https://api.bcovlive.io/v1/scheduler/clip 헤더: 콘텐츠 유형: application/json X-API-KEY: API 키 본문: { "job_id": "your Job ID", "description": "My Scheduled Clip", "clip": { "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879800, "videocloud": { "video": { "name": "My Scheduled Clip" } } }, "time_utc": 1612879800 } }

다음과 유사한 응답을 받아야 합니다.

{ "account_id": "your Account ID", "description": "My Tuesday Clip", "type": "clip", "job_id": "your Job ID", "clip": { "state": "pending", "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879800, "videocloud": { "video": { "id": "70702010152202", "name": "My Scheduled Clip" } } }, "time_utc": 1612879800 } "workflow_id": "d44cf29f4f184757ac0995beb8a0097e", }

참고 clip.output.videocloud.video.id새로 생성된 Videocloud Video의 ID를 포함합니다. ~에 time_utc비디오 소스는 수집된 클립으로 업데이트됩니다.

클립 워크플로 업데이트

둘 다 업데이트 할 수 있습니다산출그리고 time_utc당신의 클립의. 다음 경우에만 업데이트할 수 있습니다. "상태": "보류 중" .

이전 예에 따라 클립이 계획보다 5분 짧기를 원합니다. 업데이트할 때산출필드에 변경 사항뿐만 아니라 전체 개체를 제공해야 합니다. 여기에는 위의 생성 호출에서 반환된 Videocloud 비디오 ID가 포함됩니다.

이 예에서는 두 가지 모두 업데이트했습니다. end_time그리고 time_utc 5분 단축을 반영합니다.

PUT https://api.bcovlive.io/v1/scheduler/clip/workflow_id 헤더: 콘텐츠 유형: application/json X-API-KEY: API 키 본문: { "clip": { "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879500, "videocloud": { "video": { "id": "70702010152202", "name": "My Scheduled Clip" } } }, "time_utc": 1612879500 } }

응답에서 업데이트된 워크플로를 받게 됩니다.

{ "account_id": "your Account ID", "description": "My Tuesday Clip", "type": "clip", "job_id": "your Job ID", "clip": { "state": "pending", "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879500, "videocloud": { "video": { "id": "70702010152202", "name": "My Scheduled Clip" } } }, "time_utc": 1612879500 } "workflow_id": "d44cf29f4f184757ac0995beb8a0097e", }

클립 워크플로 취소

다음을 사용하여 클립을 취소할 수 있습니다. 삭제요구.

Clip 워크플로를 취소할 때 예약 시 생성된 Videocloud 비디오는 백엔드에서 제거되지 않습니다. Videocloud 계정에서 분리된 비디오를 수동으로 제거해야 합니다.

삭제 https://api.bcovlive.io/v1/scheduler/clip/workflow_id 헤더: X-API-KEY: API 키

응답에서 업데이트된 워크플로를 받게 됩니다.

{ "account_id": "your Account ID", "description": "My Tuesday Clip", "type": "clip", "job_id": "your Job ID", "clip": { "state": "cancelled", "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879500, "videocloud": { "video": { "id": "70702010152202", "name": "My Scheduled Clip" } } }, "time_utc": 1612879500 } "workflow_id": "d44cf29f4f184757ac0995beb8a0097e", }

알림

워크플로를 생성할 때 선택적으로 알림을 구성할 수 있습니다. 서비스에 대한 URL을 제공해야 합니다. 우편에게. 스케줄러가 클립을 호출하면 알림을 보내드립니다. 알림을 받도록 구성할 수도 있습니다. N클립 전 초. 참조알림 안내자세한 사항은.

POST https://api.bcovlive.io/v1/scheduler/clip 헤더: 콘텐츠 유형: application/json X-API-KEY: API 키 본문: { "job_id": "your Job ID", "description": "My Scheduled Clip", "notification_url": "http://example.com/live/clip/callbacks", "clip": { "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879800, "videocloud": { "video": { "name": "My Scheduled Clip" } } }, "time_utc": 1612879800, "알림": 600 } }
