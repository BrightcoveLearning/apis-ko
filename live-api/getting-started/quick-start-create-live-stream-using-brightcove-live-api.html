필요한 것

이 자습서를 완료하려면 다음이 필요합니다.

  • Live API용으로 설정된 계정
  • Live API용 API 키(계정 설정 시 제공됨)
  • Video Cloud 계정(라이브 스트림 작업을 만드는 데 필요하지 않지만 Brightcove Player에서 재생을 설정하는 데 필요)
  • 사용하실 예정이라면 CMS API Studio가 아닌 Video Cloud 비디오를 생성하려면 비디오/읽기 및 비디오/쓰기 권한이 있는 클라이언트 자격 증명이 필요합니다. 클라이언트 자격 증명을 얻는 프로세스에 익숙하지 않은 경우 다음을 참조하십시오. 입증아래 섹션.
  • 다음 중 하나를 통해 REST API 요청을 수행하는 기능곱슬 곱슬하다또는 다음과 같은 REST 클라이언트불명 증또는우편 집배원 .

    귀하를 돕기 위해 연습에 사용할 수 있는 일부 Insomnia 및 Postman 가져오기를 준비했습니다(Brightcove Learning Services에서는 Insomnia를 선호하고 사용하지만 그것이 있는 경우 Postman도 작동합니다).

    각 zip 파일에는 도움이 되는 추가 정보 페이지가 있습니다.

    • 불면증 작업 공간 가져오기
    • 우편 배달부 컬렉션 및 환경 가져오기
  • 실제 라이브 스트림을 생성하려면 라이브 스트리밍을 위한 소프트웨어 또는 하드웨어 인코더가 필요합니다. 아래 지침에는 Wirecast( 평가판에 등록 ). 다른 인코더를 사용하는 경우 프로세스는 다르지만 Live API 응답에서 필요한 정보 측면에서 유사해야 합니다.

CMS API에 대한 자격 증명 가져오기

사용하려면 CMS API적절한 자격 증명이 필요합니다.

대부분의 경우 자격 증명을 얻는 가장 쉬운 방법은 Studio Admin API 인증 섹션을 사용하는 것입니다(계정에 대한 관리자 권한 필요). 보다 API 인증 자격 증명 관리자세한 내용은. 이 경우 필요한 권한은공유 관계 - 읽기 및 쓰기 권한이 모두 필요합니다.

필요한 권한을 Studio에서 사용할 수 없거나 OAuth API에서 직접 권한을 가져오려면 원하는 권한을 사용하십시오. 클라이언트 자격 증명 가져오기아래 나열된 문서. 어떤 옵션을 선택하든 올바른 작업 권한을 요청해야 합니다. cURL 또는 Postman과 함께 다음을 사용하여 적절한 권한을 얻을 수 있습니다.

    "operations": [
    "video-cloud/videos/read",
    "video-cloud/videos/create",
    "video-cloud/videos/update"]
  • 인증: cURL을 사용하여 클라이언트 자격 증명 가져오기
  • 인증: Postman을 사용하여 클라이언트 자격 증명 가져오기

비디오 클라우드 비디오 설정

이제 라이브 스트림 정보가 있으므로 Video Cloud에서 원격 비디오를 설정하여 Brightcove Player에서 재생할 수 있습니다.

이 단계에서는 다음을 사용하여 비디오를 설정합니다. CMS API , 그러나 원하는 경우 Video Cloud Studio에서도 수행할 수 있습니다. 아래의 대체 단계 .

Insomnia 또는 Postman을 사용하는 경우 클라이언트 자격 증명을 사용하여 OAuth2 인증을 설정해야 합니다. 도움이 필요하면 다음 가이드 중 하나를 참조하십시오.

  • API 요청에 불면증 사용
  • API 요청에 Postman 사용

  1. 먼저 다음을 만들어 새 비디오를 만들어야 합니다. POST요청:

    https://cms.api.brightcove.com/v1/accounts/account_id/videos

  2. 요청 본문에는 여러 메타데이터 조각이 포함될 수 있습니다(참조 API 참조자세한 내용은), 그러나 필요한 유일한 것은name :

    {
    "name": "some name for your live stream"
}

  3. 응답에는 모든 비디오 메타데이터가 포함되지만 중요한 필드는id , 원격 자산을 추가하는 데 필요합니다.

  4. HLS 스트림을 추가하려면 다른POST요청:

    https://cms.api.brightcove.com/v1/accounts/account_id/videos/video_id/assets/hls_manifest

  5. 이번에는 요청 본문은 다음과 같습니다.

    {
    "remote_url": "playback_url for your live stream"
}

플레이어에 비디오 추가

이제 간단한 HTML 페이지를 만들고 방금 만든 비디오를 사용하여 여기에 플레이어를 추가합니다.

여기서 다시 Studio를 사용하여 플레이어 게시 코드를 가져올 것입니다. Player Management API너가 선호한다면.
  1. 라이브 스트림 플레이어를 테스트하기 위해 간단한 HTML 페이지를 만듭니다. 원하는 다른 콘텐츠를 포함하거나 전혀 포함하지 않을 수 있습니다.
  2. Studio Media Module에서 이전 단계에서 만든 비디오를 선택하고게시 및 포함 > 웹 플레이어 :
  3. 게시 대화 상자에서 플레이어를 선택합니다(Brightcove 기본 플레이어는 괜찮음).
  4. 아래로 스크롤하여 플레이어 크기를 원하는 대로 설정한 다음 코드 블록을 클릭하여 표준 플레이어 코드를 선택합니다. 이 코드를 복사하여 html 페이지 본문에 붙여넣습니다.
  5. html 페이지를 탐색할 수 있도록 원격 또는 로컬 웹 서버에 저장합니다.
  6. 웹 브라우저에서 페이지를 엽니다. 스팀을 시작하지 않았기 때문에 미디어 오류가 표시됩니다.

Wirecast를 사용하여 이벤트 스트리밍

이 섹션에서는 Telestream Wirecast 인코더를 구성하고 사용하여 라이브 스트리밍 이벤트를 지원합니다. Telestream Wirecast 소프트웨어는 카메라의 입력을 캡처하고 CDN에서 전달할 수 있는 스트림을 생성하는 데스크톱 응용 프로그램입니다. 라이브 스트리밍 이벤트를 전달하는 데 더 적합할 수 있는 하드웨어 및 기타 소프트웨어 기반 인코딩 솔루션이 있습니다. Wirecast 소프트웨어의 평가판을 다운로드할 수 있습니다. 텔레스트림 웹사이트 .

참고: 계속하기 전에 Wirecast 소프트웨어가 설치되어 있고 카메라가 컴퓨터에 연결되어 있어야 합니다.

라이브 이벤트에 대해 Wirecast를 구성하려면 다음 단계를 따르세요.

  1. 와이어캐스트를 엽니다.
  2. 마우스 오버 +,딸깍 하는 소리캡처 장치카메라를 선택합니다.
  3. 화살표 버튼( ) 카메라가 라이브 샷을 촬영하도록 합니다.
  4. 클릭개울버튼( ). 출력 설정을 구성하라는 메시지가 표시됩니다.
  5. 설정목적지에게 RTMP 서버 . 딸깍 하는 소리좋아요 .
  6. 를 위해주소 , 사용 stream_url Live API에서 반환됩니다(3단계).
  7. 를 위해개울 , 살아 사용합니다.
  8. (선택 과목)추가 출력 스트림을 생성하려면추가하다... 추가 출력 스트림을 생성할 때 출력 비트 전송률을 제어하기 위해 사용자 지정 인코딩 프로필을 생성할 수 있습니다. 다음 사항을 확인하십시오. 개울이름은 새 비트 전송률을 반영합니다.
  9. 딸깍 하는 소리좋아요출력 설정을 저장합니다.
  10. 클릭개울버튼( ). 실시간 스트림이 시작되어야 합니다.
