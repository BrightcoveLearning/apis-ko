일반 오류 메시지

상태 이름 메세지 원인
400 BAD_REQUEST ILLEGAL_PATH: 10개 이상의 동영상 요청은 지원되지 않습니다 기본GET /videos/video_ids요청은 10개 이상의 동영상을 요청할 수 없습니다.
400 INVALID_SORT 잘못된 속성을 기준으로 정렬을 시도했습니다: '[속성 이름]' NS sort유효하지 않거나 존재하지 않는 필드를 가리키는 매개변수
400 INVALID_SEARCH 검색 문자열이 잘못되었습니다. 검색 문자열 구문이 잘못되었거나 검색 문자열을 URIEncode하지 못했습니다.
400 ILLEGAL_QUERY 쿼리 문자열에 문제가 있습니다. 검색 문자열 구문이 잘못되었습니다. 예: 1) ~을 하다tags쉼표로 끝나거나 닫히지 않은 따옴표가 있는 검색
400 BAD_VALUE 제출된 데이터에서 인식할 수 없는 필드 철자 오류 또는 존재하지 않는 필드의 기타 사용
400 AD_CONFIG_NOT_FOUND 광고 구성을 찾을 수 없음 SSAI 요청에 지정된 광고 구성을 찾을 수 없습니다.
400 AD_CONFIG_INACTIVE 광고 구성이 비활성화되었습니다. SSAI 요청에 지정된 광고 구성이 비활성화되었습니다.
400 REFERENCES_EXIST 이 동영상은 하나 이상의 재생목록에서 참조합니다. 하나 이상의 재생 목록에 포함된 동영상을 삭제하려고 합니다.
400 SHARED_VIDEO 공유 동영상을 삭제하지 못했습니다. 공유 동영상 삭제는 아직 지원되지 않습니다.
401 UNAUTHORIZED 권한이 거부되었습니다. OAuth 액세스 토큰이 없거나 잘못되었습니다. 액세스 토큰을 제공한 경우 이 요청에 대한 적절한 범위가 없을 수 있습니다.
403 NOT_AVAILABLE 리소스를 사용할 수 없습니다. 요청한 리소스를 일시적으로 사용할 수 없습니다. 이는 일종의 비디오 처리가 진행되는 동안 일시적인 상태일 수 있지만 메시지가 지속되면 지원팀에 문의하세요.
404 NOT_FOUND 리소스가 존재하지 않습니다. 존재하지 않는 리소스를 요청했습니다. 맞춤법을 확인하세요.
405 METHOD_NOT_ALLOWED 요청에 사용된 HTTP 메서드는 이 리소스에 대해 허용되지 않습니다. 가장 일반적인 원인은 PUT, POST, PATCH 또는 DELETE를 읽기 전용 리소스로 보내는 것입니다.
406 NOT_ACCEPTABLE HTTP Accept 헤더에 허용되지 않는 값이 있습니다. NS Accept헤더에는 값이 있어야 합니다. application/json ; 다음과 같은 다른 값 application/x-www-form-urlencoded이 오류와 함께 요청이 거부됩니다.
409 LIVE_VIDEOS_NOT_SHAREABLE 이 비디오는 라이브 중이므로 공유할 수 없습니다. 라이브 스트리밍 비디오 및 이 비디오에서 만든 클립은 공유할 수 없습니다.
409 REFERENCE_ID_IN_USE 참조 ID가 이미 사용 중입니다. 이미 사용 중인 참조 ID로 동영상을 만들거나 다른 동영상에서 이미 사용 중인 동영상에 참조 ID를 추가하려고 했습니다.
409 CONCURRENT_UPDATE 동시에 다른 업데이트로 인해 업데이트에 실패했습니다. 다시 시도해 주세요. 별도의 API 요청, Studio 사용자 또는 일부 시스템 프로세스가 현재 동영상을 업데이트하고 있습니다. 이는 동일한 비디오에 대해 API 업데이트 요청을 비동기식으로 빠르게 연속적으로 수행하는 경우에도 발생할 수 있습니다.
409 CONFLICT 참조 ID가 이미 사용 중입니다. 참조 ID의 고유성을 보장하기 위해 참조 ID와 관련된 작업은 ID를 최대 3분 동안 잠급니다. 즉, 참조 ID를 생성하거나 변경하는 요청을 만든 다음 해당 참조 ID를 즉시 사용하는 다른 요청을 하면 작업이 409 오류와 함께 실패합니다. 여기에는 다음이 포함됩니다. 1) 비디오를 삭제한 다음 참조 ID를 다른 비디오에 할당하려고 합니다. 2) 503(서비스를 사용할 수 없음) 오류와 함께 실패한 새 비디오를 만들려고 시도하고 동일한 요청을 다시 시도합니다.

이 오류가 발생한 후 작업을 다시 시도하기 전에 3분 이상 기다리십시오. 또한 참조 ID로 비디오를 만들려고 시도했지만 시도가 실패하면(참조 ID와 관련이 없는 이유로) 해당 참조 ID는 30초 동안 잠기고 비디오를 만들려는 반복 시도는 다음을 수행할 때까지 실패합니다. 참조 ID가 잠금 해제될 때까지 충분히 기다립니다.
409 CONFLICT 두 계정 모두 미디어 공유를 위해 활성화되어야 합니다. 마스터 계정이 미디어 공유를 위해 채널에 제휴 계정을 추가하려고 하면, 둘 다 미디어 공유를 위해 계정을 활성화해야 합니다.
409 UNSUPPORTED_MEDIA_TYPE   요청에 헤더가 누락되었을 가능성이 큽니다. Content-Type: application/json .
415 SHARING_DISABLED 마스터 계정은 미디어 공유에 사용할 수 없습니다. 미디어 공유가 활성화되지 않은 계정에서 비디오를 공유하려고 하면 이 오류가 반환됩니다.
422 ILLEGAL_FIELD 제출된 데이터에서 인식할 수 없는 필드 철자 오류 또는 존재하지 않는 필드의 기타 사용
422 VALIDATION_ERROR (JSON 데이터가 유효하지 않음 - 오류 메시지는 문제에 따라 다름) 메시지의 예:
  • 이름: REQUIRED_FIELD(이름이 없는 동영상 만들기, 이름을 빈 문자열로 업데이트)
  • 계정 ID: WRONG_ACCOUNT   (URL의 계정 ID가 JSON의 계정 ID와 일치하지 않음)
  • 유효하지 않은 키: id(id는 수정할 수 없으며 JSON에서 키로 표시되어서는 안 됨)
429 TOO_MANY_REQUESTS 너무 많은 요청 너무 많은 동시 요청 또는 초당 너무 많은 요청을 제출하고 있습니다.
500 UNKNOWN 알 수 없는 내부 오류가 발생했습니다 Brightcove 시스템 문제 - 나중에 다시 시도하십시오.
503 SERVICE_UNAVAILABLE API를 일시적으로 사용할 수 없습니다. 백엔드 문제 - 나중에 다시 시도하세요.
504 TIMEOUT 뭔가 너무 오래 걸렸다 서버가 너무 바쁠 수 있습니다. 나중에 다시 시도하십시오.

미디어 공유 오류

미디어 공유 오류는 API 요청에 대한 별도의 오류 응답으로 반환되는 것이 아니라error_message일반 응답 필드:

  [
  {
  "video_id" : "394872349182374",
  "affiliate_id" : "234987239487",
  "affiliate_video_id" : "30308254055202",
  "status" : "COMPLETE",
  "shared_at" : "2017-12-11T17:57:45.530Z",
  "updated_at" : "2017-12-11T18:03:32.789Z",
  "error_message" : "[{"error_code":"MISSING_CUSTOM_FIELDS","error_message":"Affiliate account is missing custom fields: [whisky]"}]"
  }
  ]

NS error_message json은 오류 객체의 배열이 될 것이며, 객체는error_code그리고error_message .

error_code

오류 코드는 일반적으로 공유에 따라 다르지만 비디오에 대한 PATCH(업데이트) 작업으로 반환될 수 있는 유효한 오류 코드 중 하나일 수도 있습니다. 공유 특정 오류 코드는 아래 표에 나와 있습니다.

미디어 공유 오류 코드
error_code 에러 메시지 원인
MISSING_CUSTOM_FIELDS 제휴 계정에 사용자 정의 필드가 누락되었습니다: [field1, field2] 마스터가 설정한enforce_custom_fields에게true , 그러나 공유되는 비디오는 제휴 계정에 존재하지 않는 필드를 참조합니다.
CONFLICT 제휴 계정이 지역 제한에 대해 구성되지 않았습니다. 마스터의 force_geo가 true로 설정되어 있고 해당 계정이 지역 필터링에 대해 구성되어 있지만 제휴 계정은 지역 필터링에 대해 구성되어 있지 않습니다.
SHARING_DISABLED 제휴 계정은 미디어 공유에 사용할 수 없습니다. 마스터가 미디어 공유가 활성화되지 않은 제휴사와 비디오를 공유하려고 시도했습니다.
NOT_FOUND 리소스가 존재하지 않습니다. 뭔가 부족했습니다. 일반적으로 이것은 마스터 비디오가 사라졌음을 의미합니다. 이는 공유 요청 직후 마스터 비디오가 삭제되는 경우 발생할 수 있습니다. 이는 마스터 계정에 아직 채널이 없거나 제휴 계정이 존재하지 않거나 채널의 유효한 구성원이 아님을 의미할 수도 있습니다. 다음으로 마이그레이션하는 데 문제가 있는 경우 발생할 수 있습니다. CMS API미디어 공유.
UNKNOWN 알 수 없는 오류 error_id INSERT-UNIQUE-ERROR-ID로 인해 공유에 실패했습니다. 예상치 못한 일이 발생했습니다. error_message에는 devops에서 근본 원인을 식별하는 데 사용할 수 있는 오류 ID가 포함됩니다. 공유 시도를 다시 시도하면 많은 경우에 효과가 있습니다.
VALIDATION_ERROR :{field} ILLEGAL_VALUE 만약에{field} "경제"인 경우 문제는 마스터 비디오가 "광고 지원"이지만 제휴 계정에 광고가 활성화되어 있지 않다는 것입니다. 광고에 대해 계정이 비활성화되었을 때 (비공유 비디오의) 비디오 메타데이터에서 지원되는 광고로 광고를 설정한 경우에도 발생합니다. 만약에{field}다른 것이라면 조사가 필요합니다.
REFERENCE_ID_IN_USE 사용한 참조 ID는 이미 사용 중입니다. 참조 ID는 마스터 계정에서 유효했지만 이미 제휴 계정에서 사용 중입니다.
