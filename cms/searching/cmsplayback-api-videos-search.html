API 사용

검색 기능은 다음 중 하나와 함께 사용할 수 있습니다. CMS API또는 Playback API.

두 API에 대한 검색 문자열은 최소 3자를 포함해야 합니다.

CMS API

CMS API로 검색을 사용하는 경우 다음이 적용됩니다.

  • 모든 요청(검색 포함)에는 액세스 토큰이 포함된 인증 헤더가 필요합니다. 클라이언트 자격 증명을 얻고 이를 사용하여 액세스 토큰을 검색하는 방법에 대한 자세한 내용은 다음을 참조하십시오. 브라이트코브 OAuth 개요 .
  • 검색에서 반환되는 최대 동영상 수에는 제한이 없지만 속도 제한은 적용됩니다.
  • 검색 결과에는 재생 가능 여부 및/또는 지역 제한 여부에 관계없이 계정의 모든 동영상이 포함됩니다.

API 요청/응답에 대한 자세한 내용은비디오 가져오기섹션 CMS API 참조 .

Playback API

Playback API로 검색을 사용할 때 다음 사항이 적용됩니다.

  • Playback API를 사용한 검색 요청에는 다음과 같은 정책 키가 필요합니다. 검색 가능 .
  • 이있다한계검색에서 반환된 최대 동영상 수
  • 검색 결과는 재생 가능한 비디오( state:inactive무시됩니다).
  • 검색은 결과에서 지리적으로 제한된 동영상을 생략하는 것과 같은 재생 정책 제한을 적용합니다.
  • 결과 캐싱은 더 높은 요청 비율과 더 빠른 응답을 제공하며 비율 제한이 없습니다.
Playback API 검색은 요청당 100개의 동영상으로 제한되며 최대 1000개의 동영상으로 제한됩니다. 하는 법을 배우다검색 결과를 통한 페이지 . 1000개 이상의 동영상을 반환해야 하는 경우 CMS API를 사용해야 합니다.

API 요청/응답에 대한 자세한 내용은비디오 가져오기섹션Playback API 참조 .

기본 검색

미디어 라이브러리에서 용어 검색을 수행하려면q매개변수.

      q={search terms}

지정하는 검색어는 공백으로 구분된 URL로 인코딩된 검색어 목록이어야 합니다.

검색 지원 형태소 분석. 어간 추출은 단어의 어근과 같은 어근에서 파생된 다른 단어에 대한 매핑입니다. 예를 들어, "run"에 대한 검색은 지정된 필드에 "running" 또는 "run"이 있는 비디오와 일치해야 합니다. 그것은 것 ~ 아니다 "run"은 루트로 "run"이 없기 때문에 "rune"과 일치합니다.

다음과 같이 검색어에 대한 한정자를 제공하지 않는 경우q=bird , 요청은 다음 필드에서 해당 값을 검색합니다.

  • id [1-1]
  • name
  • description
  • long_description
  • text (실제 메타데이터 필드가 아니라 검색에 사용할 수 있는 의사 필드name , description , 그리고long_description - 예q=text:bird )
  • tags
  • reference_id
  • custom_fields (모든 사용자 정의 필드 검색)
  • custom_field_name (특정 명명된 사용자 정의 필드 검색)[1-2]

[1-2] 목록 유형 사용자 정의 필드가 있고 여러 값 중 하나가 있는 비디오를 반환하려는 경우 다음과 같이 할 수 있습니다.

라는 필드가 있다고 가정해 보겠습니다. color값을 취할 수 있습니다. red , green , yellow , 또는blue . 해당 필드가 값으로 설정된 비디오를 찾고 싶습니다. green또는blue :

인용 검색어

기본적으로 검색은 검색어와 유사한 단어를 찾습니다. 여러 단어를 일치시키려면 해당 용어를 따옴표로 묶으면 됩니다.

대부분의 브라우저 및 기타 에이전트는 리터럴 따옴표( "..." )는 정확하지만 인용된 검색어가 올바른 결과를 반환하지 않는 것으로 보이는 경우 인용 부호를 다음으로 대체해 보십시오. %22 ( %22...%22 )

            
              q="foo" or q=%22foo%22
              q="foo%20bar" or q=%22foo%20bar%22
따옴표는 일치하는 비디오에 모든 용어가 포함되어야 하지만 해당 용어와 반드시 정확히 일치하는 것은 아닙니다. 예를 들어"home run"단어가 포함된 동영상을 반환할 수 있습니다. homeless그리고running제목에.

이는 특정 필드를 검색할 때도 작동합니다.

            
              q=name:"home"
              q=name:"home%20run"

여러 단어

예시: 이 요청은 다음 값 중 하나를 갖는 동영상을 반환합니다. sea또는mammal에서tags들.

      https://cms.api.brightcove.com/v1/accounts/921483702001/videos?q=tags:sea,mammal

그러나 다음 요청은 태그가 있는 동영상만 반환합니다. sea,mammal .

      https://cms.api.brightcove.com/v1/accounts/921483702001/videos?q=tags:"sea,mammal"
태그는 여러 태그를 쉼표로 구분된 목록으로 검색할 수 있는 특별한 경우입니다. 이는 이름, 설명 등과 같은 다른 필드에는 해당되지 않습니다. 다른 종류의 다중 용어의 경우 용어를 다음과 결합하십시오. +다음과 같이 서명하십시오. q=tags:"bird"%20name:"owl" .

사용자 정의 필드

비디오에 대해 정의한 모든 사용자 정의 필드에서 검색할 수 있습니다.

      q=my_field:foo
      q=my_field:"foo"

참고: 모든 사용자 정의 필드 값은 문자열로 처리됩니다. 예를 들어 값을 사용할 수 있는 목록 유형 사용자 정의 필드가 있는 경우true또는false , 검색은 부울 값이 아닌 해당 문자열을 찾습니다(많은 프로그래밍 언어에서1그리고0와 교환하여 사용할 수 있습니다. true그리고false부울 값으로 검색하지만q=my_boolean_field:1다음이 있는 동영상은 반환하지 않습니다. my_boolean_field로 설정true ).

예시: 이 요청은 다음 값을 가진 동영상을 반환합니다. 동물에서subject사용자 정의 필드.

      https://cms.api.brightcove.com/v1/accounts/921483702001/videos?q=subject:animal

기간

날짜 필드에서 검색하는 경우 시작 날짜와 종료 날짜( q=updated_at:2018-01-01..2018-02-01 ).

이렇게 하면 다음이 포함된 모든 동영상이 검색됩니다. updated_at 2012년 8월 1일에서 2012년 10월 8일 사이의 값입니다. 여기에서는 UTC 형식으로 각 날짜를 지정합니다.

      q=updated_at:2012-08-01T00:00:00Z..2012-10-08T23:59:59Z

시간 구성 요소를 삭제하여 이 검색을 단순화할 수 있습니다. 다음은 위의 검색과 동일합니다.

      q=updated_at:2012-08-01..2012-10-08

상대적 날짜

상대 날짜의 경우 방향( +또는- ) 다음에 숫자, 기간이 옵니다. 상대 날짜는 항상 현재 시간부터 측정됩니다. 법적 기간은 분, 시간, 일입니다.

NS +기호는 다음과 같이 URI로 인코딩되어야 합니다. %2B .

예:

상대 날짜 샘플
날짜에 대한 q 매개변수 의미
q=updated_at:-1일..지금 1일 전부터 현재 날짜까지 업데이트된 동영상
q=created_at:-2일 2일 전에 추가된 동영상
q=updated_at:-4시간..지금 4시간 전에서 현재 시간으로 업데이트된 동영상
q=created_at:-60분.. 60분 전부터 현재 시간까지 추가된 동영상
q=created_at:2016-01-01..-1d 2015년 1월 1일부터 하루 전까지 만든 동영상
q=updated_at:-14d..지금 지난 2주 동안의 동영상

개방형 범위

지정된 시간까지의 모든 날짜를 일치시키거나 지정된 시간 이후의 모든 날짜를 일치시키려면 범위의 한쪽 끝을 생략하십시오.

예시: 지난 2일 동안 수정된 모든 동영상 검색

      q=updated_at:-2days..

예시: 2013년 8월 11일 이후에 수정된 모든 동영상 검색:

      q=updated_at:2013-08-11T00:00:00Z..

NOW일정 날짜에 대한 연산자

을위한schedule.starts_at그리고schedule.ends_at , 당신이 사용할 수있는NOW날짜 값으로. 이것은 현재 날짜-시간을 기반으로 동적 쿼리를 설정할 수 있는 편리한 연산자입니다. 몇 가지 예:

일정 데이터 예
매개변수에서/까지 URI 인코딩 의미
?q=schedule.starts_at:..지금 ?q=schedule.starts_at:..지금 starts_at은 시간의 시작부터 이 순간까지입니다.
?q=schedule.starts_at:지금 ?q=schedule.starts_at:지금 starts_at는 이 순간부터입니다.
?q=schedule.ends_at:지금.. ?q=schedule.ends_at:지금.. end_at는 이 순간부터 시간이 끝날 때까지입니다.
?q=+schedule.starts_at:..NOW +schedule.ends_at:NOW.. ?q=%2Bschedule.starts_at:..NOW%20%2Bschedule.ends_at:NOW.. 이 순간 이전의 starts_at 및 이 순간 이후의 끝_at(비디오는 이 순간에 예정되어 있음)

검색 기준 결합

복잡한 검색에 대한 기준을 결합할 수 있습니다.

예시: 이 요청은 다음이 포함된 동영상을 검색합니다. name가치잡담 , 2010년 8월 1일과 2010년 10월 8일 사이에 업데이트되었습니다. 그런 다음 응답 데이터를 다음과 같이 정렬합니다. updated_at내림차순으로 날짜.

      q=%2Bname:gossip%20%2Bupdated_at:2010-08-01..2010-10-08&sort=-updated_at

용어 결합

사용필수/제외 구문다음이 있는 동영상을 반환하려면모두지정된 조건의.

뒤에 오는 여러 검색어의 모든 예에서, 텍스트 검색 용어 뒤에 추가 용어가 추가된 용어가 첫 번째 용어의 일부로 해석되지 않도록 첫 번째 용어 뒤에 URI로 인코딩된 공백(%20)이 있습니다. .

예를 들어 다음을 검색하는 경우:

      q=name:foo +tags:bar (URI-encoded: q=name:foo%20%2Btags:bar)

응답에는 'bar' 태그가 있는 동영상이 포함됩니다.' 그리고 또한 가질 수 있습니다foo이름에. 다음이 있는 비디오만 반환하려면foo이름과 태그 'bar'에서 다음을 검색해야 합니다.

      (unencoded) q=+name:foo +tags:bar (URI-encoded) q=%2Bname:foo%20%2Btags:bar

마찬가지로 다음이 있는 동영상만 반환하려는 경우foo이름으로 하지만 ~ 아니다 'bar' 태그가 있으면 다음을 검색합니다.

      (unencoded) q=+name:foo -tags:bar (encoded) q=%2Bname:foo%20-tags:bar

시료: 결합 용어
인코딩되지 않은 검색 문자열 URI로 인코딩된 검색 문자열 의미
q=foo bar q=foo%20bar 반환된 항목에는 "foo" 또는 "bar"가 있습니다.
q=foo +bar q=foo%20%2Bbar 반환된 항목에는 "bar"가 있어야 하고 "foo"가 있을 수 있습니다.
q=+foo bar NS =%2Bfoo%20bar 반환된 항목에는 "foo"가 있어야 하고 "bar"가 있을 수 있습니다.
q=+foo +bar q=%2Bfoo%20%2Bbar 반환된 항목에는 "foo" 및 "bar"가 있어야 합니다.
q=-foo +bar q=-foo%20%2Bbar 반환된 항목에는 "bar"가 있어야 하고 "foo"가 없어야 합니다.
다중 태그 검색
q=tags:bee,bop q=tags:bee,bop "bee" 또는 "bop" 태그가 있는 비디오를 반환합니다.
q=tags:bee tags:bop q=tags:bee%20tags:bop "bee" 또는 "bop" 태그가 있는 비디오를 반환합니다.
q=+tags:bee tags:bop q=%2Btags:bee%20tags:bop 반환된 모든 비디오에는 "bee" 태그가 있어야 합니다. 태그 "밥"도 있을 수 있습니다.
q=+tags:bee +tags:bop q=%2Btags:bee%20%2Btags:bop 반환된 모든 비디오에는 "bee" 태그와 "bop" 태그가 있습니다.

특정 동영상 검색

특정 비디오 세트로 검색을 제한하려면 다음에서 검색할 수 있습니다. id :

예시: 이 요청은 ID가 있는 동영상을 반환합니다. 123456789 , 987654321그리고48376387

      q=id:123456789%20id:987654321%20id:48376387

주별 검색

다음 매개변수를 사용하여 비디오 상태별로 검색을 수행하거나 필터링할 수 있습니다.

      q=state:ACTIVE( | INACTIVE | PENDING | DELETED)[3]

노트

  • [삼] DELETED 동영상 검색은 지난 10일(현재 시간에서 10일을 뺀)에 삭제된 동영상에 대해서만 사용할 수 있습니다. CMS API (Playback API 아님).

형태소 분석

스테밍이 지원되지만 ~ 아니다부분 단어 검색. 예를 들어, q=name:ban "라는 이름의 동영상을 반환해야 합니다. Parking Ban Announced " 또는 " Parking to be Banned " 또는 " City Banning Parking " 하지만 " Bank Holiday " 또는 " Bandit Captured ".

공백/특수 문자

NS CMS API일반적으로 몇 가지 예외를 제외하고 검색 문자열의 특수 문자를 처리합니다.

  • 공백은 허용되지 않으며 다음과 같이 인코딩해야 합니다. %20 . (인코딩되지 않은+기호는 공백을 대체할 수도 있지만 이는 다음과 같이 쿼리에 혼란을 초래할 수 있습니다. +용어가 필요함을 나타낼 수도 있습니다. 보다필수/제외 구문 )

    예를 들어, 이름에서 "내가 가장 좋아하는 비디오"를 검색하려면:

    q=name:"my%20favorite%20video"

    따옴표를 사용하면 반환된 비디오에 모든 용어가 포함됩니다. 그것들이 없으면 세 가지 용어 중 하나라도 포함된 비디오가 반환됩니다. 자세한 내용은 아래에서 확인하세요.
  • 리터럴을 검색하려면+서명하거나 사용+반환된 동영상이 ~해야하다용어를 포함하려면 인코딩해야 합니다. +같이%2B :

    다음을 포함해야 하는 동영상 검색"two+two"이름 필드에

    q=name:two%2Btwo

    다음을 포함해야 하는 동영상 검색"heron"이름 필드에

    q=%2Bname:heron

  • 일부 에이전트는 리터럴 따옴표를 올바르게 처리하지 못할 수 있으므로 인코딩하는 것이 더 안전합니다. "foo"같이%22foo%22

다중 단어 태그 예
인코딩되지 않은 검색 문자열 인코딩된 검색 문자열

따옴표를 사용하면 반환된 비디오가 모든 용어와 일치하지만 반드시 정확한 용어는 아닙니다. 형태소 분석여전히 적용됩니다. 예를 들어"new house"다음과 같은 동영상도 반환합니다. name또는description처럼news about housing .

일회성 요청의 경우 Brightcove Learning의문자열 인코더검색 문자열을 인코딩합니다. 앱의 경우 사용 중인 언어로 URI 인코딩 기능을 찾아야 합니다.

별표( * )는 일반 특수 문자로 처리되며, ~ 아니다와일드카드.

클립 검색어

클립은 다른 비디오의 섹션에서 만든 비디오입니다. 클립은 다음으로 생성할 수 있습니다. 브라이트코브 소셜 , 그리고 다른 수단은 미래에 사용할 수 있습니다. 계정에서 생성된 클립을 찾는 데 사용할 수 있는 몇 가지 특수 검색어가 있습니다.

  • q=%2Bis_clip:true - 클립만 반환
  • q=%2Bis_clip:false - 클립이 아닌 것만 반환
  • q=%2Bclip_source_video_id:video_id - 지정된 비디오에서 생성된 클립을 반환합니다.
