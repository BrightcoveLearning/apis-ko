소개

에서 사용하는 비디오 검색 버전 2 CMS API구문을 단순화하고 사용하기 쉽게 만듭니다.

노트:
  • v2 검색 구문은 다음에서 지원됩니다. CMS API오직 너 Playback API와 함께 이 구문을 사용할 수 없습니다. .
  • 원래 검색 구문은 CMS와 Playback API 모두에서 계속 지원됩니다.

사용할 구문을 선택하는 것은 적절한 URL 매개변수를 선택하는 간단한 문제입니다.

  • 새로운 v2 검색을 사용하려면: 
        .../videos?query={search_string}
  • 원래 검색을 사용하려면: 
        .../videos?q={search_string}

기초

검색 문자열의 기본 요소는 검색어, 필드 이름이 접두사로 붙을 수 있습니다. 필드 이름이 포함된 경우 해당 메타데이터 필드만 검색됩니다. 그렇지 않으면 여러 필드(아래 나열)가 검색됩니다.

예를 들면 다음과 같습니다.

기본 검색
검색 문자열 무엇을 반환할 것인가
bird "라는 단어가 나오는 동영상bird " 아래 나열된 필드에
name:bird "라는 단어가 포함된 동영상bird "에서name (제목)이(가) 반환됩니다.

검색할 필드 이름을 제공하지 않으면 요청은 다음 필드에서 해당 값을 검색합니다.

  • id
  • name
  • description
  • long_description
  • text (실제 메타데이터 필드가 아니라 검색에 사용할 수 있는 의사 필드name , description , 그리고long_description - 예text:bird )
  • tags
  • reference_id
  • custom_fields (모든 사용자 정의 필드 검색)
  • custom_field_name (특정 명명된 사용자 정의 필드 검색)

검색에 지원되는 필드는 다음과 같습니다.

지원되는 검색 필드
들 법적 가치
name 문자열 또는 따옴표 붙은 문자열
텍스트 문자열 또는 따옴표로 묶인 문자열(검색name , description , 그리고long_description )

custom_fields 문자열 또는 따옴표로 묶인 문자열(모든 사용자 정의 필드 검색 - 특정 사용자 정의 필드를 사용할 수도 있습니다. 내부의 이름)

위에 표시된 두 예에서 ""라는 단어가 없는 동영상은bird " 관련 필드에서 여전히 반환될 수 있습니다. 다음 섹션에서는 지정된 용어가 있는 비디오로만 검색 결과를 제한하는 방법을 설명합니다.

고급 검색

검색 결과를 정확히 원하는 비디오로 제한하는 데 도움이 되는 몇 가지 수정자가 있습니다.

검색 수정자
수정자 설명 예
+ 검색어 앞에 더하기(+)가 붙으면 반환된 동영상이 ~해야하다지정된 용어를 가지고
  • +bird (보고 오직 "가 포함된 동영상bird " 위에 나열된 필드)
  • +tags:bird (보고 오직 "가 포함된 동영상bird "에서tags )
-또는NOT 마이너스(-) 기호로 검색어 접두사 또는NOT반환된 비디오가해서는 안 된다지정된 용어를 가지고
  • -birds또는NOT birds (다음을 수행하는 비디오만 반환 ~ 아니다가지다 " bird " 위에 나열된 필드)
  • -name:birds또는NOT name:birds (다음을 수행하는 비디오만 반환 ~ 아니다가지다 " bird "에서name )
(term) AND (term)
또는
(term) OR (term)		 논리적AND그리고OR연산자를 사용하면 복잡한 쿼리에 대해 여러 검색어를 결합할 수 있습니다.
  • (+name:heron) AND (+tags:bird) ( 둘 다 " heron "에서name그리고 " bird "에서tags )
  • (+name:heron) OR (+tags:bird) ( 어느 하나 " heron "에서name그리고 " bird "에서tags )
  • ((+name:heron) AND (+tags:bird)) AND (NOT tags:internal) ( 둘 다 " heron "에서name그리고 " bird "에서tags , 하지만 ~ 아니다태그 " internal ")

구문 검색

따옴표를 사용하여 한 단어가 아닌 구를 검색할 수 있습니다.

  • "blue heron"
  • name:"blue heron"

날짜/시간

다음을 사용하여 날짜-시간 간격으로 검색할 수 있습니다.

[{start} TO {end}]

단일 날짜/시간으로 검색하려면start그리고end같은 값으로:

[2019-09-30T00:00:00.000Z TO 2019-09-30T00:00:00.000Z]

날짜-시간 값은 ISO 8601 형식을 사용하여 지정됩니다.

날짜/시간 형식
날짜 시간 체재 예시
날짜 시간 yyyy-MM-ddThh:mm:ss.sssZ 2019-09-30T14:24:33.512Z
와일드카드(시작 또는 종료 날짜/시간에 사용할 수 있음) *
  • 2019-09-30T14:24:33.512Z TO *
  • * TO 2019-09-30T14:24:33.512-4:00Z
필수 참고Z시간대 표시기에서. 현재 값은 비어 있어야 합니다(UTC를 시간대로 나타냄).

다음은 몇 가지 샘플 날짜/시간 검색 문자열입니다.

샘플 데이터/시간 검색
검색 문자열 설명
+updated_at:[2019-09-30T00:00:00.000Z TO 2019-10-07T00:00:00.000Z] 2019년 9월 30일에서 2019년 10월 7일 사이에 업데이트된 비디오
+created_at:[2019-09-30T00:00:00.000Z TO 2019-09-30T00:00:00.000Z] 2019년 9월 30일에 추가된 동영상
+created_at:[2019-09-30T14:00:00.000Z TO 2019-09-30T16:30:00.000Z] 2019년 9월 30일 오후 2시~오후 4시 30분(UTC)에 추가된 동영상
+created_at:[* TO 2019-09-30T00:00:00.000Z] 2019년 9월 30일 이전에 추가된 동영상
