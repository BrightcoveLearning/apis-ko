소개

사용자 정의 필드는 미디어 모듈의 비디오에 추가할 수 있지만, 하나씩 추가해야 합니다. 여기에 있는 샘플 앱을 사용하면 한 번에 최대 10개의 비디오에 사용자 정의 필드를 추가할 수 있습니다(또한 해당 숫자를 늘리거나 계정의 모든 비디오에 사용자 정의 필드를 추가하도록 쉽게 수정할 수 있음). 앱은 상당히 직관적입니다. 사용자 정의 필드에 대한 UI를 만드는 데 있어 유일한 문제는 값에 대해 간단한 문자열을 사용하거나 값 목록에서 선택할 수 있다는 것입니다. 그러나 필드 메타데이터에서 필드 유형을 쉽게 식별하고 적절한 입력 필드를 표시할 수 있으므로 관리하기가 그리 어렵지 않습니다.

이 샘플 및 관련 코드는 프로덕션 개발을 위한 가이드로 제공됩니다. 코드는 프로덕션 용도로 인증되지 않았으며 공식적으로 지원되지 않습니다.

자격 증명 가져오기

사용하려면 CMS API적절한 자격 증명이 필요합니다.

대부분의 경우 자격 증명을 얻는 가장 쉬운 방법은 Studio Admin API 인증 섹션을 사용하는 것입니다(계정에 대한 관리자 권한 필요). 보다 API 인증 자격 증명 관리자세한 내용은. 대부분의 경우 모든 권한을 얻기를 원할 것입니다. CMS API작업:

필요한 권한을 Studio에서 사용할 수 없거나 OAuth API에서 직접 권한을 가져오려면 원하는 권한을 사용하십시오. 클라이언트 자격 증명 가져오기아래 나열된 문서. 어떤 옵션을 선택하든 올바른 작업 권한을 요청해야 합니다. 다음을 cURL 또는 Postman과 함께 사용하여 CMS API에 대한 모든 권한을 얻을 수 있습니다.

          "operations": [
            "video-cloud/video/all",
            "video-cloud/playlist/all",
            "video-cloud/sharing-relationships/all",
            "video-cloud/notifications/all"
          ]
  • 인증: cURL을 사용하여 클라이언트 자격 증명 가져오기
  • 인증: Postman을 사용하여 클라이언트 자격 증명 가져오기
