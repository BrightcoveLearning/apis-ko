제한 사항

이 샘플 및 관련 코드는 프로덕션 개발을 위한 가이드로 제공됩니다. 코드는 프로덕션 용도로 인증되지 않았으며 공식적으로 지원되지 않습니다.

  1. 콘텐츠 보안을 위해 Brightcove API에서 반환하는 비디오 및 자산의 URL은 다음과 같습니다. 서명 , 이는 짧은 시간(보통 몇 시간) 동안만 유효한 토큰을 포함하지만 이는 계정 구성에 따라 다를 수 있음을 의미합니다. 즉, 이 도구로 생성된 MRSS 코드의 URL은 일정 기간 동안만 유효합니다.

    다음 두 가지 방법 중 하나로 이 문제를 해결할 수 있습니다.

    • 계정의 미디어 URL에 연결된 토큰의 "수명"을 늘리도록 Brightcove 지원에 요청할 수 있습니다. 값은 최대 1년의 기간으로 설정할 수 있습니다. 그러나 이는 계정 수준 설정이며 변경 사항은 모든 동영상에 적용됩니다.
    • 또는 이 앱의 고유한 버전을 만들고 토큰 수명 내에서 정기적으로 MRSS 피드의 새 버전을 만들도록 할 수 있습니다.
  2. 출처가 없는 동영상은 피드에 포함되지 않습니다.

자격 증명 가져오기

사용하려면 CMS API적절한 자격 증명이 필요합니다.

대부분의 경우 자격 증명을 얻는 가장 쉬운 방법은 Studio Admin API 인증 섹션을 사용하는 것입니다(계정에 대한 관리자 권한 필요). 보다 API 인증 자격 증명 관리자세한 내용은. 대부분의 경우 모든 권한을 얻기를 원할 것입니다. CMS API작업:

필요한 권한을 Studio에서 사용할 수 없거나 OAuth API에서 직접 권한을 가져오려면 원하는 권한을 사용하십시오. 클라이언트 자격 증명 가져오기아래 나열된 문서. 어떤 옵션을 선택하든 올바른 작업 권한을 요청해야 합니다. 다음을 cURL, Postman 또는 온라인 앱과 함께 사용하여 CMS API에 대한 모든 권한을 얻을 수 있습니다.

          "operations": [
            "video-cloud/video/all",
            "video-cloud/playlist/all",
            "video-cloud/sharing-relationships/all",
            "video-cloud/notifications/all"
          ]
