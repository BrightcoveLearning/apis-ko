소개

이 샘플 및 관련 코드는 프로덕션 개발을 위한 가이드로 제공됩니다. 코드는 프로덕션 용도로 인증되지 않았으며 공식적으로 지원되지 않습니다.

Video Cloud를 사용하면 비디오 트랙이 없는 비디오 형식의 오디오 전용 콘텐츠를 사용할 수 있습니다. 그러나 이러한 비디오를 "오디오 전용"으로 플래그 지정하지 않으므로 오디오 및 비디오 혼합 콘텐츠의 대규모 라이브러리가 있는 경우 오디오 전용 콘텐츠에 대한 보고서를 만드는 방법이 명확하지 않을 수 있습니다.

변환에 오디오 전용인지 알려주는 속성이 있지만 레거시 수집 시스템이나 Dynamic Delivery 시스템을 사용하여 수집된 비디오의 속성은 다릅니다.

  • 레거시 인제스트 비디오의 경우 각 변환에는audio_only부울인 속성
  • Dynamic Delivery 비디오의 경우 각 변환에는media_type로 설정될 속성audio또는video

여기 샘플 앱에서는 다음 중 하나를 확인하는 함수를 사용합니다.

        function isAudio(rendition) {
          if (rendition.hasOwnProperty('audio_only') && rendition.audio_only === false) {
            return false;
          } else if (rendition.hasOwnProperty('media_type') && rendition.media_type === 'video') {
            return false;
          }
          return true;
        }

이 샘플에 대한 참고 사항

  • Video Cloud의 모든 비디오에는 오디오 전용 변환이 있을 수 있습니다. 이 샘플은 다음이 포함된 콘텐츠를 식별합니다. 오직오디오 변환을 수행하므로 모든 변환을 확인합니다. 하나 이상의 오디오 전용 변환이 있는 콘텐츠에 대한 보고서를 생성하도록 코드를 수정하는 것은 간단합니다. 다음은 샘플 코드에서 수행해야 하는 유일한 변경 사항입니다.

    기존 코드(오디오 전용 콘텐츠에서 찾기 위해):

            function processRenditions(video, renditions) {
            var i,
              iMax = renditions.length,
              audioRenditions = 0;
            if (video.id === '5831706803001') {}
            // separate renditions by type
            for (i = 0; i < iMax; i++) {
              if (isAudio(renditions[i])) {
                audioRenditions++;
              } else {
                // if any non-audio renditions, stop
                break;
              }
            }
            // check to see if all renditions are audio
            if (audioRenditions === renditions.length) {
              video.renditionCount = audioRenditions;
              audiosArray.push(video);
              return;
            } else {
              return;
            }
          }

    오디오 전용 변환이 있는 콘텐츠를 식별하는 기능 버전:

            function processRenditions(video, renditions) {
            var i,
              iMax = renditions.length,
              audioRenditions = 0;
            if (video.id === '5831706803001') {}
            // separate renditions by type
            for (i = 0; i < iMax; i++) {
              if (isAudio(renditions[i])) {
                audioRenditions++;
              }
            }
            // check to see if any renditions are audio
            if (audioRenditions > renditions.length) {
              video.renditionCount = audioRenditions;
              audiosArray.push(video);
              return;
            } else {
              return;
            }
          }

자격 증명 가져오기

사용하려면 CMS API적절한 자격 증명이 필요합니다.

대부분의 경우 자격 증명을 얻는 가장 쉬운 방법은 Studio Admin API 인증 섹션을 사용하는 것입니다(계정에 대한 관리자 권한 필요). 보다 API 인증 자격 증명 관리자세한 내용은. 대부분의 경우 모든 권한을 얻기를 원할 것입니다. CMS API작업:

필요한 권한을 Studio에서 사용할 수 없거나 OAuth API에서 직접 권한을 가져오려면 원하는 권한을 사용하십시오. 클라이언트 자격 증명 가져오기아래 나열된 문서. 어떤 옵션을 선택하든 올바른 작업 권한을 요청해야 합니다. 다음을 cURL 또는 Postman과 함께 사용하여 CMS API에 대한 모든 권한을 얻을 수 있습니다.

          "operations": [
            "video-cloud/video/all",
            "video-cloud/playlist/all",
            "video-cloud/sharing-relationships/all",
            "video-cloud/notifications/all"
          ]
