페이지 내용

    미디어 공유

    이 주제에서는 CMS API를 사용하여 하나의 Video Cloud 계정에서 다른 계정으로 비디오를 공유하는 방법을 배웁니다.

    서문

    미디어 공유는 Video Cloud의 기능으로, 게시자가 다른 게시자와 비디오를 공유해서 여러 계정에서 비디오를 더욱 쉽게 관리하도록 지원합니다. 예를 들어 게시자는 비디오 콘텐츠의 마스터 계정을 유지 한 다음 비디오를 조직의 다른 부서 또는 자회사와 공유 할 수 있습니다.

    모든 미디어 공유 작업은 Studio에서도 수행 할 수 있습니다. 보다미디어 공유 설정 관리 .

    • 미디어 공유는 여러 계정을 가진 Video Cloud Enterprise 및 Pro 고객 만 사용할 수 있습니다. Video Cloud 계정 업그레이드에 대한 자세한 정보는 브라이트코브에 문의하십시오.
    • 마스터 및 각 제휴 계정에 대해 CMS API 미디어 공유 작업을 활성화해야합니다. Brightcove 지원에 문의공유가 필요한 경우.
    • 미디어 공유는 양방향 일 수 있습니다. 각 계정은 마스터가 될 수 있으며 다른 계정은 제휴사가 될 수 있습니다.

    공유 미디어 및 과금

    공유 미디어에 대한 청구 방식에 대한 정보는 다음을 참조하십시오. 미디어 모듈을 사용한 미디어 공유 .

    분류: 용어

    미디어 공유에서는 마스터 계정 (비디오 공유)과 하나 이상의 제휴 계정 (공유 비디오 수신)간에 다음과 같은 관계가 있습니다.

    미디어 공유 용어
    계정 설명
    석사 원본 동영상을 만든 계정입니다.

    마스터는 콘텐츠를 소유하고 있으며, 콘텐츠를 설정, 관리 및 제휴사에 제공할 책임이 있습니다.
    제휴 비디오를 수신하는 계정입니다.

    제휴사는 마스터로부터 공유된 콘텐츠를 수락할 수 있습니다.
    채널 콘텐츠가 마스터에서 여러 제휴사로 공유되는 파이프 라인입니다. 미디어 공유가 활성화되면default계정에 채널이 생성됩니다.
    관계 마스터와 계열사 간의 상호 작용을 설명합니다.

    관계는 콘텐츠를 공유하는 마스터, 콘텐츠를 공유하는 채널, 콘텐츠 수락 계약 및 콘텐츠를 수신하는 제휴사로 구성됩니다.
    계약 마스터와 계열사 간의 공유 관계를 설명합니다.

    마스터가 계약을 생성 한 다음 공유를 활성화하려면 수락해야합니다. 제휴자는 공유 된 비디오를 자동으로 수락할지 또는 하나씩 승인해야하는지 여부를 지정할 수도 있습니다.

    무엇을 공유합니까?

    이 섹션에서는 공유되는 항목과 비디오에 대한 후속 변경 사항을 처리하는 방법에 대해 설명합니다.

    동영상이 공유되는 경우

    대부분의 비디오 메타 데이터 필드는 비디오가 공유 될 때 마스터에서 제휴 계정으로 복사됩니다. 주목할만한 예외는 다음과 같습니다.

    • id -동영상은 제휴 계정에서 고유 한 ID를 갖습니다.
    • 다음과 같은 날짜 필드created_at과updated_at

    모든 비디오 자산 (변환, 이미지, text_tracks 등)은 재생을 위해 제휴 계정에서 사용됩니다.

    동영상이 공유 된 후

    비디오가 공유 된 후 마스터 계정의 비디오에 대한 일부 변경 사항은 자동으로 제휴 계정에 상속되고 일부는 상속되지 않습니다.

    비디오 자산

    이미지 제외 , 비디오 자산에 대한 마스터 변경 사항은항상계열사에 의해 상속됩니다. 계열사자산을 변경할 수 없습니다렌 디션, 매니페스트, 텍스트 트랙 또는 디지털 마스터와 같은

    마스터에 의한 이미지 변경 사항은 계열사에 상속됩니다. 제휴사가 이미지를 대체하지 않는 한 . 제휴사가 이미지를 변경하면 해당 이미지는 더 이상 마스터로부터 상속되지 않습니다.

    비디오 메타데이터

    모든 비디오 메타 데이터 (예 : 이름, 설명 및 참조 ID)는 제휴자가 수정할 수 있으며 마스터 비디오에 대한 변경 사항은 다음과 같습니다. 아니계열사에 의해 상속됩니다.

    동영상 다시 공유

    그러나 마스터가재 공유동영상 (Studio가 아닌 CMS API를 통해서만 가능), 모든 자산 및 메타 데이터 (데이터 / 시간 필드 제외)가 제휴 자에게 공유됩니다. 계열사가 변경 한 사항을 덮어 씁니다. .

    다시 공유하는 방법에는 결과가 다른 두 가지가 있습니다.
    석사공유 해제그리고재 공유비디오
    이 경우 이전에 공유 한 동영상은 공유 해제시 모든 제휴 계정에서 삭제됩니다. 마스터가 비디오를 다시 공유하면새로운비디오새로운고유 ID가 각 제휴 계정에 추가되어 마스터 계정의 메타 데이터 및 자산을 원래 공유와 같이 가져옵니다.
    석사재 공유비디오없이공유 해제
    이 경우 동영상 마스터 사본의 모든 메타 데이터와 이미지가 제휴 버전으로 푸시되어 제휴 변경 사항을 덮어 쓰지만 제휴 동영상 ID는아니변화.
    비디오를 다시 공유하기 전에 이것이 당신이하고 싶은 일이고 제휴 자들이 무슨 일이 일어날 지 알고 있는지 확인하십시오!

    제한 사항

    현재 미디어 공유에는 다음과 같은 제한 사항이 있습니다.

    • DRM : 미디어 공유CMS API현재 DRM 사용 계정에는 지원되지 않습니다. DRM이 활성화되지 않은 계정에서 DRM이 활성화 된 계정으로 비디오를 공유하는 것은 지원되지만 공유 된 비디오는아니 DRM 용으로 패키지화됩니다.

    • 마스터 계정에서 정의한 채널이 설정된 경우enforce_custom_fields ...에true , 그런 다음 제휴 계정에서 허용하지 않는 값을 가진 사용자 정의 필드가있는 비디오를 공유하면 해당 공유 시도가 실패합니다. 공유 상태는 다음과 같은 오류 메시지로 업데이트됩니다.

            [{"error_code": "ILLEGAL_CUSTOM_FIELD_VALUE", "error_message": "Illegal value for custom fields: [topic]"}]

      마스터 계정에서 정의한 채널이 설정된 경우enforce_custom_fields ...에false , 그런 다음 제휴 계정에서 허용하지 않는 값을 가진 사용자 정의 필드가있는 비디오를 공유하면 공유 시도가 작동하지만 잘못된 값을 가진 필드는 비디오의 제휴사 사본에 포함되지 않습니다.

    • SSAI를 사용하여 공유 비디오를 재생할 때 SSAI는 하위 비디오 대신 상위 비디오의 메타데이터를 사용합니다. SSAI는 상위 동영상이 다음과 같이 표시된 경우에도 광고 조회를 건너 뜁니다. Advertising='Free' , 하위 동영상이Ad Supported .
