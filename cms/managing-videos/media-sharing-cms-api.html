소개

미디어 공유는 게시자가 다른 게시자와 비디오를 공유할 수 있도록 하는 Video Cloud의 기능으로, 여러 계정에서 비디오를 보다 쉽게 관리할 수 있습니다. 예를 들어 게시자는 비디오 콘텐츠의 마스터 계정을 유지한 다음 조직의 다른 부서 또는 자회사에 비디오를 공유할 수 있습니다.

모든 미디어 공유 작업은 Studio에서도 수행할 수 있습니다. 보다미디어 공유 설정 관리 .

또한 참고:
  • 미디어 공유는 여러 계정을 가진 Video Cloud Enterprise 고객만 사용할 수 있습니다. Video Cloud 계정 업그레이드에 관심이 있는 경우 Brightcove에 자세한 내용을 문의하십시오.
  • 마스터 및 각 제휴 계정에 대해 CMS API 미디어 공유 작업을 활성화해야 합니다. Brightcove 지원에 문의공유를 활성화해야 하는 경우.
  • 미디어 공유는 양방향일 수 있습니다. 각 계정은 다른 계정의 마스터 및 제휴사가 될 수 있습니다.

공유 미디어 및 청구

공유 미디어에 대한 청구 방식에 대한 정보는 다음을 참조하십시오. 미디어 모듈을 사용한 미디어 공유 .

술어

미디어 공유에서 마스터 계정(동영상 공유)과 관련된 하나 이상의 제휴 계정(공유 동영상 수신) 사이에는 다음과 같은 관계가 있습니다.

미디어 공유 용어
계정 설명
주인 원본 동영상을 만든 계정입니다.

마스터는 콘텐츠를 소유하고 콘텐츠를 설정, 관리 및 계열사에 제공할 책임이 있습니다.
제휴하다 비디오를 수신하는 계정입니다.

계열사는 마스터로부터 공유된 콘텐츠를 수락할 수 있습니다.
채널 콘텐츠가 마스터에서 원하는 수의 계열사로 공유되는 파이프라인입니다. 미디어 공유가 활성화된 경우default계정에 채널이 생성됩니다.
관계 마스터와 계열사 간의 상호 작용을 설명합니다.

관계는 콘텐츠를 공유하는 마스터, 콘텐츠를 공유하는 채널, 콘텐츠를 수락하는 계약, 콘텐츠를 수신하는 계열사로 구성됩니다.
계약 마스터와 계열사 간의 공유 관계를 설명합니다.

마스터가 계약을 생성한 다음 공유를 활성화하려면 수락해야 합니다. 제휴사는 공유 비디오를 자동으로 수락할지 아니면 하나씩 승인해야 하는지 지정할 수도 있습니다.

기본 URL

모두에 관해서 CMS API요청에서 아래에 설명된 작업의 기본 URL은 다음과 같습니다.

      https://cms.api.brightcove.com/v1

아래에 설명된 모든 엔드포인트는 요청 시 기본 URL에 추가됩니다.

입증

요청에 대한 인증에는 Authorization 헤더가 필요합니다.

          Authorization: Bearer {access_token}

NS access_token Brightcove OAuth 서비스에서 가져와야 하는 임시 OAuth2 액세스 토큰입니다. 클라이언트 자격 증명을 얻고 이를 사용하여 액세스 토큰을 검색하는 방법에 대한 자세한 내용은 다음을 참조하십시오. 브라이트코브 OAuth 개요 .

주변의 모든 작업에 유의하십시오. 관계필요한 새 권한:

      video-cloud/video/all
      video-cloud/sharing-relationships/read
      video-cloud/sharing-relationships/create
      video-cloud/sharing-relationships/update
      video-cloud/sharing-relationships/delete

또는 다음을 사용할 수 있습니다.

      video-cloud/sharing-relationships/all

Studio API 인증 관리 페이지에는 두 가지 권한이 표시됩니다.

  • 읽기 공유 (에 해당video-cloud/sharing-relationships/read )
  • 읽기/쓰기 공유 (에 해당video-cloud/sharing-relationships/all )

  • 제휴 계정에는 마스터 계정의 비디오에 값이 설정된 사용자 정의 필드가 없습니다.
  • 마스터 계정에는 지역 필터링이 활성화되어 있지만 제휴 계정에는

사용자 정의 필드 일치

채널에 대해 맞춤 필드 일치를 적용할 수 있습니다. 즉, 동영상 공유가 실패합니다. 비디오에 제휴 계정에 없는 사용자 정의 필드 값이 있는 경우 . 비디오에 일치하지 않는 사용자 정의 필드 값이 없는 경우 비디오는 계속 성공적으로 공유됩니다.

기본적으로 사용자 정의 필드 일치는 ~ 아니다시행.

일치하지 않는 맞춤 필드로 인해 동영상 공유가 실패하면 응답에 다음과 같은 오류가 표시됩니다.

지역 필터링 일치

채널에 대해 지역 필터링 일치가 활성화된 경우 마스터 계정에 지역 필터링이 활성화되어 있고 제휴 계정에는 활성화되어 있지 않으면 비디오를 공유할 수 없습니다.

기본적으로 지역 필터링 일치 ~이다시행.

무엇을 공유합니까?

이 섹션에서는 공유되는 항목과 비디오에 대한 후속 변경 사항이 처리되는 방법에 대해 설명합니다.

동영상을 공유할 때

대부분의 비디오 메타데이터 필드는 비디오가 공유될 때 마스터에서 제휴 계정으로 복사됩니다. 주목할만한 예외는 다음과 같습니다.

  • id - 비디오는 제휴 계정에 고유한 ID를 갖습니다.
  • 다음과 같은 날짜 필드created_at그리고updated_at

모든 비디오 자산(변환, 이미지, text_tracks 등)은 재생을 위해 제휴 계정에서 사용됩니다.

영상이 공유된 후

비디오가 공유된 후 마스터 계정의 비디오에 대한 일부 변경 사항은 자동으로 제휴 계정에 상속되고 일부는 그렇지 않습니다.

동영상 자산

이미지 제외 , 비디오 자산에 대한 마스터 변경 사항은언제나계열사 상속. 계열사자산을 변경할 수 없습니다변환, 매니페스트, 텍스트 트랙 또는 디지털 마스터와 같은

마스터에 의한 이미지 변경 사항은 제휴사에 상속됩니다. 제휴사가 이미지를 교체하지 않는 한 . 제휴사가 이미지를 변경하면 해당 이미지는 더 이상 마스터로부터 상속되지 않습니다.

동영상 메타데이터

모든 비디오 메타데이터(예: 이름, 설명 및 참조 ID)는 제휴사가 수정할 수 있으며 마스터 비디오에 대한 변경 사항은 ~ 아니다계열사가 상속합니다.

동영상 다시 공유

그러나 마스터의 경우다시 공유비디오(이는 Studio가 아닌 CMS API를 통해서만 수행할 수 있음), 모든 자산 및 메타데이터(데이터/시간 필드 제외)가 계열사에 공유됩니다. 계열사가 변경한 사항 덮어쓰기 .

다시 공유하는 방법에는 두 가지가 있으며 결과가 다릅니다.
주인 공유 취소 그리고 다시 공유 비디오
이 경우 이전에 공유된 동영상은 공유 해제 시 모든 제휴 계정에서 삭제됩니다. 마스터가 동영상을 다시 공유하면 새로운 비디오 새로운 고유 ID가 각 제휴 계정에 추가되어 원래 공유에서와 같이 마스터 계정에서 메타데이터와 자산을 가져옵니다.
주인 다시 공유 비디오없이공유 취소
이 경우 비디오의 마스터 사본에 있는 모든 메타데이터와 이미지가 제휴 버전으로 푸시되어 제휴 변경 사항을 덮어쓰지만 제휴 비디오 ID는 ~ 아니다변화.
비디오를 다시 공유하기 전에 이것이 당신이 하고 싶은 일이고 당신의 제휴자들이 무슨 일이 일어날지 알고 있는지 확인하십시오!

제한 사항

현재 미디어 공유에는 다음과 같은 제한 사항이 있습니다.

  • DRM: 미디어 공유 CMS API현재 DRM 사용 계정에서는 지원되지 않습니다. DRM이 활성화되지 않은 계정에서 DRM이 활성화된 계정으로 비디오를 공유하는 것은 지원되지만 공유된 비디오는 ~ 아니다 DRM용으로 패키징해야 합니다.

  • 마스터 계정으로 정의된 채널이 설정되어 있는 경우enforce_custom_fields에게true , 그런 다음 제휴 계정에서 허용하지 않는 값을 가진 사용자 정의 필드가 있는 비디오를 공유하면 해당 공유 시도는 실패합니다. 공유 상태는 다음과 같은 오류 메시지와 함께 업데이트됩니다.

          [{"error_code": "ILLEGAL_CUSTOM_FIELD_VALUE", "error_message": "Illegal value for custom fields: [topic]"}]

    마스터 계정으로 정의된 채널이 설정되어 있는 경우enforce_custom_fields에게false , 그런 다음 제휴 계정에서 허용하지 않는 값을 가진 사용자 정의 필드가 있는 비디오를 공유하면 공유 시도는 작동하지만 잘못된 값을 가진 필드는 비디오의 제휴 사본에 포함되지 않습니다.

  • SSAI로 공유 비디오를 재생할 때 SSAI 매크로 대체는 하위 비디오 대신 상위 비디오의 메타데이터를 사용합니다. 상위 비디오가 다음으로 표시된 경우 SSAI는 광고 조회도 건너뜁니다. Advertising='Free' , 하위 동영상에 다음과 같은 라벨이 지정되어 있어도Ad Supported .
