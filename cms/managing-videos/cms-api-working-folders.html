소개

비디오 라이브러리에서 비디오를 구성하는 데 사용되는 폴더를 만들 수 있습니다. 주제, 작성자, 제품, 지역 등으로 비디오를 구성하도록 선택할 수 있습니다. 폴더에 비디오가 있으면 수백 또는 수천 개의 비디오가 있는 계정에서 더 쉽게 관리할 수 있습니다.

노트:
  • 이 주제에서 논의된 모든 작업은 다음에서도 수행할 수 있습니다. 사진관대신 CMS API를 통해.
  • NS CMS API모든 요청에 대해 클라이언트 자격 증명과 액세스 토큰이 필요합니다. 자격 증명을 얻는 데 도움이 필요하면 다음을 참조하십시오. 아래의 자격 증명 가져오기 섹션 .
  • 이 항목에서는 CMS API에 대해 가장 일반적으로 사용되는 폴더 작업에 대해 설명하지만 다음에서 찾을 수 있는 몇 가지가 더 있습니다. CMS API 참조 .

CMS API에 대한 자격 증명 가져오기

사용하려면 CMS API적절한 자격 증명이 필요합니다.

대부분의 경우 자격 증명을 얻는 가장 쉬운 방법은 Studio Admin API 인증 섹션을 사용하는 것입니다(계정에 대한 관리자 권한 필요). 보다 API 인증 자격 증명 관리자세한 내용은. 이 경우 필요한 권한은비디오 - 읽기 및 쓰기 권한이 모두 필요합니다.

필요한 권한을 Studio에서 사용할 수 없거나 OAuth API에서 직접 권한을 가져오려면 원하는 권한을 사용하십시오. 클라이언트 자격 증명 가져오기아래 나열된 문서. 어떤 옵션을 선택하든 올바른 작업 권한을 요청해야 합니다. cURL 또는 Postman과 함께 다음을 사용하여 적절한 권한을 얻을 수 있습니다.

           "operations": [
           "video-cloud/videos/read",
           "video-cloud/videos/create",
           "video-cloud/videos/update"]
  • 인증: cURL을 사용하여 클라이언트 자격 증명 가져오기
  • 인증: Postman을 사용하여 클라이언트 자격 증명 가져오기

다음은 새 폴더를 만드는 데 필요한 정보입니다.

      }

폴더에 비디오 추가

폴더에 비디오를 추가하려면PUT요청:

      https://cms.api.brightcove.com/v1/accounts/account_id/folders/folder_id/videos/video_id

요청 본문

이 작업에 대한 요청 본문이 없습니다.

응답

이 요청에 대한 응답은204 NO CONTENT .

폴더에서 비디오 제거

폴더에서 비디오를 제거하려면DELETE동영상을 추가하는 데 사용하는 것과 동일한 URL로 요청:

      https://cms.api.brightcove.com/v1/accounts/account_id/folders/folder_id/videos/video_id

요청 본문

이 요청의 경우 요청 본문에 폴더 이름도 포함해야 합니다.

      {
        "name": folder_name
      }

폴더에서 비디오 가져오기

폴더에 비디오를 가져오려면GET요청:

      https://cms.api.brightcove.com/v1/accounts/:account_id/folders/folder_id/videos

요청 매개변수

한계그리고오프셋

기본적으로 이 요청은 가장 최근에 폴더에 추가된 20개의 비디오를 반환합니다. 다음 매개변수 중 하나 또는 둘 다를 요청에 추가하여 이를 변경하고 많은 비디오가 포함된 폴더에 대한 결과를 페이지로 넘길 수 있습니다.

  • limit : 반환할 동영상 수(기본값: 20, 최대값: 100)
  • offset : 건너뛸 동영상 수 (기본값: 0)

예를 들어 이 요청은 가능한 결과의 비디오 21-30을 반환합니다(폴더에 많은 비디오가 포함되어 있다고 가정).

https://cms.api.brightcove.com/v1/accounts/:account_id/folders/folder_id/videos?limit=10&offset=20

종류

다음을 사용하여 반환된 비디오를 정렬할 수도 있습니다. 종류매개변수, 다음 필드 중 하나와 동일하게 설정:

  • 이름
  • 참조 ID
  • 업데이트됨_at
  • 만든_에
  • 게시된_at
  • schedule_starts_at
  • schedule_ends_at
  • 상태
  • play_total
  • play_trailing_week

내림차순으로 정렬하려면 필드 값 앞에 마이너스 기호( -created_at ).

에 대한 기본값종류 ~이다 -updated_at .
