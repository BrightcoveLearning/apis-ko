소개

재생 목록을 사용하면 관련 비디오 세트를 함께 그룹화하여 플레이어에 로드하고 그룹으로 시청할 수 있습니다. NS CMS API재생 목록을 생성, 업데이트 또는 삭제하는 데 사용할 수 있는 쓰기 방법 세트가 포함되어 있습니다.

이러한 작업은 Studio Media 모듈에서도 수행할 수 있습니다.

입증

요청 CMS API액세스 토큰이 포함된 인증 헤더가 필요합니다. 클라이언트 자격 증명을 얻고 이를 사용하여 액세스 토큰을 검색하는 방법에 대한 자세한 내용은 다음을 참조하십시오. 브라이트코브 OAuth 개요 .

재생 목록 유형

하나가있다EXPLICIT (수동) 비디오 ID 배열로 포함할 비디오를 지정하는 재생 목록 유형입니다. 또한 7가지 스마트 재생 목록 유형이 있습니다. 스마트 재생 목록 유형은 검색 문자열을 사용하여 동적으로 선택된 비디오 집합을 정렬하는 방식이 다릅니다. 아래 표는 모든 재생 목록 유형을 보여줍니다.

재생 목록 유형
유형 설명
EXPLICIT 수동 재생 목록. 포함된 비디오와 표시 순서는 비디오 ID 배열로 정의됩니다.
ACTIVATED_OLDEST_TO_NEWEST 선택한 동영상이 활성화 날짜(오름차순)별로 정렬된 스마트 재생 목록입니다.
ACTIVATED_NEWEST_TO_OLDEST 선택한 동영상이 활성화 날짜(내림차순)별로 정렬된 스마트 재생 목록입니다.
ALPHABETICAL 선택한 동영상이 이름의 알파벳순으로 표시되는 스마트 재생 목록입니다.
PLAYS_TOTAL 선택한 동영상을 전체 재생 횟수(내림차순)로 정렬한 스마트 재생 목록입니다.
PLAYS_TRAILING_WEEK 선택한 동영상을 전주 재생 횟수(내림차순)로 정렬한 스마트 재생 목록입니다.
START_DATE_OLDEST_TO_NEWEST 선택한 동영상이 예정된 시작 날짜(오름차순)에 따라 정렬되는 스마트 재생 목록입니다.
START_DATE_NEWEST_TO_OLDEST 선택한 동영상이 예정된 시작 날짜(내림차순)에 따라 정렬되는 스마트 재생 목록입니다.

검색 필드

모든 스마트 재생 목록 유형에 대해 비디오 컬렉션은search재생 목록의 필드 값입니다. 검색 필드 값은 태그에 대한 유효한 검색 문자열이어야 합니다. 검색 문자열 CMS API용. 다음은 반환할 동영상에 대한 설명이 포함된 몇 가지 예입니다.

현재 재생 목록은 하나 이상의 태그(쉼표로 구분) 검색을 통해서만 구성할 수 있습니다. 또한 다음을 수행하는 비디오를 검색할 수 없습니다. ~ 아니다특정 태그가 있습니다.
샘플 검색 문자열
검색 문자열 설명
+tags:bird "bird" 태그가 있는 비디오를 반환합니다.
+tags:bird,woodland "bird"와 "woodland" 태그가 모두 있는 비디오를 반환합니다.
tags:bird,woodland 다음이 있는 동영상을 반환합니다. 어느 하나 "bird" 또는 "woodland" 태그 중(참고: +검색 문자열에서 로그아웃하면 차이가 있음)

동영상 수 가져오기

다음을 사용하여 재생 목록(스마트 또는 수동)의 비디오 수를 확인할 수 있습니다. counts끝점:

    https://cms.api.brightcove.com/v1/accounts/account_id/playlists/playlist_id/videos
스마트 재생 목록의 경우 반환된 결과는 재생 목록에 대해 지정된 검색 기준에 의해 현재 반환된 비디오 수를 나타냅니다.

재생 목록 만들기

새 재생 목록을 만들려면POST요청:

요청 본문

요청에 비디오 메타데이터 필드의 많은 부분(전부는 아님)을 포함할 수 있습니다. 너 ~해야하다적어도name그리고type재생 목록을 위해. 다음은 두 가지 예입니다. 하나는 EXPLICIT 재생목록을 만들고 다른 하나는 두 개의 태그가 있는 동영상을 포함하여 ACTIVATED_NEWEST_TO_OLDEST 유형의 스마트 재생목록을 만듭니다.

EXPLICIT 재생 목록의 경우

    {

재생 목록 업데이트

재생 목록을 업데이트하는 것은 재생 목록을 만드는 것과 매우 유사합니다. 유일한 차이점은 요청 방법( PATCH POST 대신) 및 URL에 추가된 재생 목록 ID. 요청 본문의 필드는 정확히 동일합니다.

참고하세요할 수 없다변경EXPLICIT재생 목록을 다른 유형으로 변경하거나 다른 유형의 재생 목록을EXPLICIT .
