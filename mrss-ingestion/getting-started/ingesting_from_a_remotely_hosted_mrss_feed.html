소개

미디어 RSS(MRSS)를 사용하여 Brightcove가 게시하는 MRSS 피드에서 자체 시스템의 콘텐츠를 가져오도록 할 수 있습니다. Video Cloud는 MRSS 피드에서 메타데이터를 가져와 원격 자산을 생성하거나 다운로드(호스팅되는 위치에서)하고 비디오 및 관련 자산(썸네일, 자막 파일 등)을 처리합니다.

원격으로 호스팅된 MRSS 피드에서 수집 베타에 참여하려면 계정 관리자에게 문의하세요.

MRSS 수집 개요

MRSS 소개

미디어 RSS(MRSS)는 RSS 피드에서 멀티미디어 파일을 신디케이트할 수 있는 RSS의 확장입니다. 기술적인 세부 사항은 다음을 참조하십시오. 미디어 RSS 사양 .

MRSS 설정

계정 관리자에게 연락하여 MRSS를 사용하고 싶다고 표시하십시오. MRSS 피드는 공개 URL에서 호스팅되어야 하며 여기에 포함된 비디오 및 자산도 공개 URL에 있어야 합니다. URL에는 http:// 또는 https://가 포함될 수 있습니다.

MRSS 사용

MRSS를 사용하려면 웹 서버에서 메타데이터 및/또는 비디오 및 기타 파일(썸네일, 자막)을 호스팅하고 XML 매니페스트 파일해당 콘텐츠를 참조합니다.

MRSS 피드는 Brightcove에서 5분마다 가져옵니다.

MRSS를 사용하여 메타데이터만 가져오기

  1. 에 설명된 대로 XML 매니페스트 파일을 만듭니다. XML 매니페스트 파일 . 당신은 사용해야합니다ooyala:remoteasset메타데이터 전용 수집용 태그입니다.
  2. 제공한 MRSS 위치 중 하나에서 XML 매니페스트 파일을 호스팅합니다. 파일 이름과 경로가 Brightcove에 제공한 경로 중 하나와 정확히 일치하는지 확인하십시오..

MRSS를 사용하여 비디오, 축소판 및 자막 파일 가져오기

  1. 웹 서버에서 하나 이상의 파일(비디오, 축소판 및 자막 파일)을 호스팅합니다. URL은 공개적으로 액세스할 수 있어야 합니다.
  2. 생성 XML 매니페스트 파일 . 파일의 항목이 서버의 파일을 참조하는지 확인하십시오. 특히, 가치<media:content>집단href기인하다~해야하다다음 예와 같이 서버의 파일에 대한 HTTP URL이어야 합니다. 
    <media:content url="http://mysite.com/upload/lacrosse_70.mov" />
  3. 제공한 MRSS 위치 중 하나에서 XML 매니페스트 파일을 호스팅합니다. 파일 이름과 경로가 Brightcove에 제공한 경로 중 하나와 정확히 일치하는지 확인하십시오..
    메모: 불행히도 "클라이언트 로깅" 개념은 MRSS 피드 기반 처리에 적용되지 않습니다. 피드가 성공적으로 처리되고 있는지 확인하는 가장 좋은 방법은 다음 중 하나를 사용하여 자산의 모양을 확인하는 것입니다. 비디오 클라우드 스튜디오또는 비디오 클라우드 CMS API .
